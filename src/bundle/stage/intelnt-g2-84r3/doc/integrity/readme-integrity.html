What is Integrity

Integrity is a real-time, integrated and extensible network management and root-cause intelligence platform that maintains continuous availability in voice and data networks by detecting, diagnosing, and correcting problems before they impact services.

Integrity delivers continuous service availability through:

  • Proactive network, applications, service level, and performance monitoring

  • Automation of the labor-intensive, time-consuming, and reasoning-intensive steps required to diagnose and resolve network problems

  • Rapid determination of the root causes of problems

  • Automated testing and diagnosis

  • Business impact analysis

  • Flexible GUI

  • Alarm filtering

  • Correlation capabilities that present critical troubleshooting information

  • Automated recovery actions

Its combination of object technology, extensive intelligent systems technologies, and proven reliability, scalability and performance make Integrity unique in its ability to address complex network, system and application availability problems.

System Requirements

To use Integrity, you need:

  • Processor speed/RAM:

    • Windows

      • Windows XP Pro, 2003, or 2000.

      • Pentium processor (at least 1 GHz Pentium 4 processor recommended for development).

      • 1 GB memory.

    • UNIX:

      • Minimum: 512 Mbytes of RAM. Recommended: 1024 Mbytes of RAM or more.

  • 1 GB disk space, including documentation.

  • 14.1" TFT display for laptops or 17" desktop monitor with a minimum screen resolution of 1024x768

  • TCP/IP protocol installed and configured

  • 10 Mbps local area network (LAN) connected to the server

Windows Operating Systems Considerations

Integrity installation differs depending on the Windows operating system:

  • For Windows, the installation differs depending on your privileges when you install the product. If you are installing with administrative privileges, the system, without prompting, installs for all users. To install as the current user, temporarily remove your administrative status.

  • For Windows XP, 2003, and 2000, you must use administrator privileges to install. If you do not, the system generates the error, "Setup failed to launch install engine: Class not registered."

  • For Windows XP, 2003, and 2000, after installation, a user without administrator privileges cannot run Integrity until the file permissions on the installed files have been changed. The recommended way to do this is to have the administrator follow these steps after installation:

    1. Choose Properties on the folder that contains the Integrity bundle software, g2-8.3r1, by default.

    2. Click the Security tab, then click the Advanced button.

    3. Enable the "Reset permissions on all child objects and enable propagation of inheritable permissions" option.

Enabling this option allows the user to run Integrity without giving administrator privileges. You can now run Integrity.

What's in this Release?

Integrity is installed in the following directory, by default: 

    C:\Program Files\Gensym\g2-8.3r1\

Integrity contains the following directories and files in the installation directory. This table lists only those directories and files that you need to know about for building and running Integrity applications.

File Description 
integrity

Integrity application files. 

    bin

Icon file and Ping Manager executable. 

    examples

Demo files for Integrity. 

    kbs

Integrity KB files. The top-level KB is integrity.kb. 

    logs

G2 log files. 

    resources

Text resources for Integrity and SymCure. 

    deprecated

KBs that existing application may require as part of their module hierarchy but are no longer part of Integrity. 

g2i

Support KBs for Integrity. This directory also contains an example KB named cdg-modguide.kb, which provides two SymCure applications for fault diagnosis. 

g2mibparser

Java class files used to parse MIB files. See the setup wizard described in the Integrity User's Guide. 

jsnmp

Java class files for the Java SNMP bridge. See the setup wizard described in the Integrity User's Guide. 

sockman

Java class files for the Java Socket Manager Bridge. See the setup wizard described in the Integrity User's Guide. 

doc

Top-level directory for all online documentation for the Integrity bundle.

    integrity

Subdirectory that contains Integrity online documentation. For a description of each document and information on how to access them, see Online Documentation.

Online Documentation

The Integrity documentation is available in PDF format. The typical installation directory is: 

    C:\Program Files\Gensym\g2-8.3r1\doc\integrity

You can access online documentation from the Windows Start menu by choosing:

    Start > Programs > Gensym G2 8.3r1 > Documentation > G2 Integrity

Integrity is bundled with the G2 Bundle, which means the complete G2 Bundle documentation is also available. For a description of the G2 Bundle documentation, see the readme-g2.html file.

Note: The Integrity documentation has been partially updated for Version 5.0 Rev. 0. However, the dialogs and figures have not been updated to reflect the new user interface. Updated documentation will be provided in a future release.
