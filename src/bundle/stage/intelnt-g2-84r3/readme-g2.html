G2 Bundle
Version 2011
Readme

The G2 Bundle provides a basic suite of products for developing and deploying intelligent, knowledge-based, real-time applications that can communicate with dynamic external systems.

This readme file contains these sections:

  • Installing on Windows Operating Systems describes how to install the G2 Bundle software on Windows platforms.

  • Modifying Your Installation describes how to add components to an existing installation, using a new license key.

  • G2 Bundle Products briefly describes each product in the Development version of the G2 Bundle.

  • Change to Authorization Files describes important changes to the G2 authorization file.

  • Change to G2 Utilities and Components describes how to access the
    sys-mod.kb file from G2 utilities and components.

  • Batch File and Shell Script for Starting the G2 Server describes how to use a batch file or shell script for starting the server with a default module search path and user-specified command-line options.

  • Important Bridge Product Information contains important information on how to run your bridge products.

  • Online Documentation provides information about the documentation included with this version of G2.

Installing on Windows Operating Systems

These instructions provide information for installing G2 on the Windows Vista/Windows 7/XP/2003/2008/2000 operating systems.

Note: G2 no longer supports Window 95, 98, Me, or NT 4.0. Windows 95, 98, and Me were supported in offline use only.

Before Installing

  1. Please read this file for additional installation information and for last-minute updates to the software or documentation. The contents of this file are included as a hardcopy document in the G2 Bundle and are located in the top-level directory of the G2 installation. The hardcopy version is always the most up-to-date.

  2. If you have a previous installation of G2 on your machine, make a backup copy of the g2.ok (OK) file and save it to another directory. This should be done each time that you modify the installation or uninstall.

  3. Uninstall any previous installation of the G2 Bundle prior to G2 Version 6.1 Rev. 0 and reboot your machine. This removes all previous installation information from the Windows registry. If left in the registry, this information will interfere with the new installation.

    To uninstall G2, you can use the setup.exe program located on the G2 Bundle CD and follow the instructions for uninstalling the software. You can also uninstall the software by using Add/Remove Programs on the Windows Control Panel. Either technique is equivalent.

    Any user-defined files remain in the existing directory structure of the uninstalled program. For example, user-defined KBs remain in the kbs directory. Only subdirectories with user-defined files remain; all other subdirectories under the installation directory are deleted.

  1. Uninstall any previously installed stand-alone versions of G2 JavaLink and reboot your machine. After uninstalling the products, you must also remove from the PATH and CLASSPATH environment variables any entries that support the uninstalled products. You must also remove these entries if you installed G2 JavaLink as part of the G2 Classic Version 5.2 Rev. 2 bundle. See the readme-javalink.html file for further information.

System Requirements

  • G2 Standard: Windows XP, Windows 7, Windows 2003, Windows 2008R2

    G2 Enterprise: Windows 7, Windows 2008R2, Red Hat Enterprise Linux 6

  • Pentium processor (at least 1 GHz Pentium 4 processor recommended for development).

  • 128 Mbytes memory (512 Mbytes memory recommended for development).

  • 1 Gbytes disk space without online documentation (900 Mbytes for full installation).

  • Display with resolution of 1024 x 768, 16-bit color, and 2 Mbytes of video RAM.

Windows Operating Systems Considerations

G2 installation differs depending on the Windows operating system:

  • For Windows, the installation differs depending on your privileges when you install the product. If you are installing with administrative privileges, the system, without prompting, installs for all users. To install as the current user, temporarily remove your administrative status.

  • For Windows 2000 and XP, you must use administrator privileges to install. If you do not, the system generates the error, "Setup failed to launch install engine: Class not registered."

  • For Windows 2000 and XP, after installation, a user without administrator privileges cannot run G2 until the file permissions on the installed files have been changed. The recommended way to do this is to have the administrator follow these steps after installation:

    1. Choose Properties on the folder that contains the G2 bundle software, g2-2011, by default.

    2. Click the Security tab, then click the Advanced button.

    3. Enable the "Reset permissions on all child objects and enable propagation of inheritable permissions" option.

Enabling this option allows the user to run G2 without giving administrator privileges. You can now run G2.

Installing the G2 Bundle Software

To install the G2 Bundle software:

  1. If the installer has not started automatically, run the setup.exe file that is located in the top-level directory of the CD.

  2. Follow the installation instructions on your screen. When prompted for the license key, enter the 25-digit license key printed on the CD case.

Determining Version from the Executable

On Windows platforms, you can determine the G2 version without having to run G2 by choosing Properties on the g2.exe executable file and clicking the Version tab. The File Version indicates the version, where the first slot is the major version, the second slot is the minor version, the third slot indicates whether the release is an Alpha (16), Beta (32), or release (128), and the fourth slot indicates the revision and patch. For example, 8.4.128.200 corresponds with the G2 Version 8.4 Rev. 2 release, and Version 8.4 Rev. 2 Patch 1 is 8.4.128.201.

You can also determine the version of the G2 bridges by choosing Properties on the executable, for example, g2-odbc.exe.

Modifying Your Installation

On Windows, if you have already installed the G2 Bundle and want to modify your installation later by adding new components such as a bridge, you can do so by using the same CD with a new license key.

To modify your installation, insert or mount the CD and choose the Modify option, then select the components whose installation you wish to modify, including the new components as well as the existing components that are already installed.

Note: Ensure that existing components are still selected; otherwise, they will be removed from the installation.

G2 Bundle Products

By default, the G2 Bundle installer installs the software components in these directories:

    Windows: C:\Program Files\Gensym\g2-2011\

The software component directories listed in this section are subdirectories immediately below g2-2011 or immediately below your custom installation directory path. From this point forward, this document refers to all directories using a forward slash (/).

On Windows, you can access the G2 Bundle products from the Start menu, as follows:

    Start > Programs > Gensym G2 2011

G2 Version 2011

The G2 executable is Gensym's core product. It is an object-oriented graphical environment for building and deploying expert applications, and is the foundation for Gensym's layered products.

The G2 executable is in the g2 subdirectory.

G2 Utilities Version 2011

The following G2 Utility applications are part of the G2 Bundle. All of their knowledge bases, except G2 ProTools, are located in /g2/kbs/utils/. (See G2 ProTools below.)

  • G2 Developer's Interface (GDI) enables you to develop a consistent Microsoft Windows-like graphical user interface for G2. The top-level knowledge base is g2cuidev.kb.

  • G2 Dynamic Displays (GDD) gives you the ability to add dynamically-generated graphical attribute readouts to icons in your applications. The top-level knowledge bases are gdddev.kb and gddlib.kb.

  • G2 Foundation Resources (GFR) provides tools for implementing multiple-module knowledge bases. The top-level knowledge base is gfr.kb.

  • G2 GUIDE (GUIDE) enables you to create graphical user interfaces for G2 applications, and the G2 GUIDE User Interface Library (GUIDE/UIL) provides an application programmer's interface (API) to procedures that perform basic operations on the dialogs and UIL controls that you use with GUIDE to create a user interface. The top-level knowledge bases are guide.kb and uil.kb.

  • G2 Menu System (GMS) enables you to create menus similar to the menus on PCs. The top-level knowledge base is gms.kb.

  • G2 OnLine Documentation (GOLD) enables you to distribute and access online documentation within your G2 application. The top-level knowledge base is goldui.kb.

  • G2 ProTools helps you, as a G2 developer, to create, debug, profile, document, and deploy G2 knowledge-base applications. This product only ships with the Development version the G2 Bundle. G2 ProTools is located in the protools subdirectory under the G2 installation directory, and the top-level KB is protools.kb.

  • G2 XL Spreadsheet (GXL) allows you to create, display, and edit tabular data in G2 in a familiar spreadsheet style. The top-level knowledge base is gxl.kb.

G2 Bundle Components

In addition to G2 core and G2 utilities, the G2 Bundle includes the following components:

Telewindows Version 2011

Telewindows allows you to open multiple local and remote windows into a running G2 in order to view and work with the contents of the current knowledge base. Your G2 application can offer each Telewindows user a distinct view of the knowledge base depending on such factors as functional privilege levels and natural-language interfaces. Telewindows is located in the g2 subdirectory, and the executables are twng.exe (Windows only), and tw.exe (Windows) .

G2 Gateway (GSI) Version 2011

G2 Gateway, also known as GSI, allows you to develop two-way communication bridges between your G2 applications and dynamic external systems such as database management systems, C/C++ programs, and external simulation software. G2 Gateway gives your G2 application the real-time data it needs, and the ability to update the state of external systems.

On Windows, the G2 Bundle ships with one G2 Gateway directory, called gsi, which contains the GSI libraries and examples compiled with Microsoft Visual Studio.

G2 JavaLink Version 2011

G2 JavaLink allows you to write G2 bridges entirely in Java. G2 JavaLink is located in the javalink subdirectory.

G2 ActiveXLink Version 2011

G2 ActiveXLink enables you to establish communications between G2 and a COM-compliant application running under Windows. G2 ActiveXLink is located in the activexlink subdirectory. G2 ActiveXLink is available on Windows operating systems only.

G2 WebLink Version 2011

G2 WebLink enables you to access your G2 applications using a Web browser. The G2 WebLink executable and knowledge bases are located in the gw subdirectory.

G2 JMail Bridge Version 2.3 Rev. 1

The G2 JMail Bridge allows G2 applications to send and receive email via JMail. The G2 JMail Bridge executable and knowledge base is located in the jmail subdirectory.

G2 Bridge Products

The G2 Bundle allows you to install two of the following Gensym Bridge Products (or two deployments of one bridge product):

  • G2-CORBALink Bridge Version 2011 is the Common Object Request Broker for G2, providing G2 access to objects that do not reside in G2's address space. The executable is located in the corbalink subdirectory and the KBs are located in the kbs subdirectory under corbalink. The top-level KB is named g2idl.kb.

  • G2-OPC Client Bridge Version 2011 gives G2 access to plant and process data from any server that is compliant with OPC (OLE for Process Control). This bridge product is located in the opcclient subdirectory.

  • G2-PI Bridge Version 2011 provides G2 communication with Pi. This bridge product is located in the pi subdirectory.

  • G2-ODBC Bridge Version 2011 provides G2 communication with relational databases on platforms that provide an ODBC driver interface. This bridge product is located in the odbc subdirectory. The G2-ODBC Bridge is available on the Windows operating systems only.

  • G2-ODBC-Unicode Bridge Version 2011 is based on the G2-ODBC Bridge with support for UNICODE characters. This bridge is located in the odbc-unicode subdirectory. The G2-ODBC-Unicode Bridge is available on the Windows Operating System only.

  • G2-MS-SQL Bridge Version 2011 provides G2 communication with MS SQL Server. This bridge is located in the mssql subdirectory. The G2-MSSQL Bridge is available on Windows Operating System only.

  • G2-Oracle Bridge Version 2011 provides G2 access to Oracle databases. The G2-Oracle executables and the g2-database.kb are located in the oracle subdirectory. In previous releases of G2, only one Oracle bridge (g2-oracle) was needed. When installing from the G2 Bundle, you must use a different Oracle bridge depending on which version of the Oracle Client software that you are using. The G2 Bundle contains the g2-ora7 bridge for use with the Oracle 7 Client software; the g2-ora80 bridge for use with the Oracle 8.0 Client software; the g2-ora81 bridge for use with the Oracle 8.1 Client software; the g2-ora90 bridge for use with the Oracle 9.0 Client software; the g2-ora92 bridge for use with the Oracle 9.2 Client software; and the g2-ora10 bridge for use with the Oracle 10g Client software.

  • G2-Sybase Bridge Version 2011 provides G2 access to Sybase databases. The G2-Sybase executables and the g2-database.kb are located in the sybase subdirectory. The G2 Bundle contains the g2-syb11 bridge for use with Sybase 11.1 libraries, and the g2-syb125 bridge for use with Sybase 12.5 libraries.

  • G2 JMSLink Version 2.3 Rev. 1 allows G2 applications to communicate with messaging systems via JMS. The G2 JMSLink executable is located in the jms subdirectory. G2 JMSLink supports some implementations of JMS on Windows platform.

  • G2 Java Socket Manager Version 4.2 Rev. 1 allows G2 applications to communicate with Java sockets. The G2 Java Socket Manager executable is located in the sockman subdirectory. G2 Java Socket Manager is only supported on Windows platform.

  • G2-SNMP Bridge Version 4.0 Rev. 1 allows G2 applications to communicate with devices that support SNMP (Simple Network Management Protocol). The G2 Java SNMP Bridge executable is located in the snmp subdirectory.

  • G2-HLA Bridge Version 2.0 Rev. 1 provides an interface to the Modeling and Simulation (M & S) High Level Architecture (HLA) and located in the hla subdirectory.

In addition to version-specific release notes for each of the database bridges, the G2 Bundle includes the G2 Database Bridge User's Guide, which supports each of the database bridges.

G2 Developers' Utilities Version 2.3 Rev. 1

G2 provides the following developers' utilities located in g2i subdirectory:

  • Business Process Management System (BPMS) provides a user interface, classes, methods, and built-in services that are based on the G2 Graphical Language (G2GL) (bpms.kb).

  • Business Rule Management System (BRMS) provides a user interface for creating, managing, and analyzing complex business rules (brms.kb).

  • G2 Reporting Engine (GRPE) provides a consistent approach for creating for creating reports and charts. It also includes an API to ease Microsoft Excel integration (grpe.kb).

  • G2 Web (GWEB) provides out-of-the-box support for displaying Web pages from within G2 and provides an API for creating Web servers, making SOAP requests, and creating HTML pages (gweb.kb).

  • G2 Event and Data Processing (GEDP) is a multi-purpose graphical language composed of graphical blocks that can be connected together to express a flow of data, perform calculations, execute functions, and generate messages and events (gedp.kb).

  • G2 Run-Time Library (GRTL) provides foundation libraries for publish/subscribe, Meta Model, and XML (grtl.kb, grtl-icons.kb, grtl-utils.kb).

  • G2 Event Manager (GEVM) provides tools that support highly scalable, distributed operator-advisory applications by providing an event "black board" and alarm management capabilities, as well as associated message queues, message browsers, and logging (gevm.kb, gqs.kb, and glf.kb).

  • G2 Dialog Utility (GDU) extends the custom Windows dialog functionality that G2 provides to enable the rapid building and deployment of native Windows dialogs (gdu.kb). This module also includes the G2 Dialog Conversion Utility (GDUC), which generates custom Windows dialog specifications from GUIDE/UIL dialogs (gduc.kb) and the G2 Dialog Configuration Editor (GDUE), which provides a native Windows editor for a native Windows dialog specification (gdue.kb).

  • G2 Data Source Manager (GDSM) provides tools for managing and pooling network connections to improve throughput in large-scale applications, including UIL and native configuration dialogs (gdsm.kb, gdsm-agent.kb, gdsm-db.kb, gdsm-jmail.kb, gdsm-jms.kb, gdsm-opc.kb, gdsm-pi.kb, gdsm-snmp.kb, gdsm-web.kb).

  • G2 Data Point Manager (GDPM) provides functionality to configure, log, replay, and simulate datapoints, typically related to external sensors such as temperature, pressure, and flow. These external values are represented in GDPM as external datapoints and obtain their values typically via an OPC or PI interface and bridge (gdpm.kb).

  • G2 Engineering Unit Conversion (GEUC) provides a way of specifying the engineering units for entering and displaying values, as well as a large number of synonyms for those conversions in both the English and metric systems (geuc.kb).

  • G2 Error Handling Foundation (GERR) provides tools for error handling (gerr.kb).

  • G2 Relation Browser (GRLB) provides tools for displaying related items in a graphical layout (grlb.kb).

G2 Diagnostic Assistant (GDA) Version 5.1 Rev. 1

GDA is a layered knowledge-base application product that provides a visual programming environment for creating diagnostic, alarm-management, and advanced control applications that maintain the availability and performance of continuous and batch production processes. GDA is located in the gda subdirectory, and the top-level KB is gda.kb.

Change to Authorization Files

Beginning with the G2 Bundle Version 8.0 Rev. 0, G2 requires new authorization files. Authorization files from a Version 7.x release will not work in this release, and prior releases cannot use authorization files from this release. This change includes the g2.ok, tw.ok, and gsi.ok files.

Change to G2 Utilities and Components

Beginning with G2 Version 6.1 Rev. 0, G2 components no longer include the sys-mod.kb file as part of the directory. Therefore, when you launch a component such as GDA or G2 ProTools, you must use the -module-search-path argument followed by a pathname to the directory that contains this KB, which is kbs\utils on Windows and kbs/utils on UNIX. You can provide a relative pathname, as shown above, or a complete pathname. For example, on Windows: 

    "C:\Program Files\Gensym\g2-2011\g2\g2.exe" 
          -module-search-path kbs\utils

You can start G2 from a batch file to specify a default module search path. See Batch File and Shell Script for Starting the G2 Server.

Batch File and Shell Script for Starting the G2 Server

G2 provides a batch file called StartServer.bat, which sets up a module search path that points to all available KBs in the bundle, including the \g2\kbs\utils or /g2/kbs/utils directory, respectively. This file accepts all G2 command-line arguments, up to a maximum of nine arguments. If you need to specify more arguments, copy the batch file or shell script, and edit the command that starts G2.

Starting the G2 server on Windows platforms from the Start menu executes this batch file. We recommend that you create a shortcut to this batch file or shell script to start your G2 application at startup. For example, this command starts G2 with the default module search path and loads the specified KB on port 1112:

  • Windows: 

    install-dir\g2\StartServer.bat -kb ..\kbs\apps\my-app.kb 
    -tcpport 1112

Important Bridge Product Information

Bridges Requiring gsi.ok Files

Certain of the bridge products require an authorization file, called a gsi.ok file, to run. G2's installer automatically sets up the gsi.ok file in the directory of each bridge. When the bridge is run, it has to know where to find its gsi.ok file. The Start menu shortcuts that launch a bridge on Windows handle this automatically.

However, to run a bridge from the command line, you must explicitly set your GSI_ROOT environment variable to point to the bridge gsi.ok file. On Windows operating system, supply the directory path as the value of the GSI_ROOT environment variable. For example:

  • Windows:

      C:\Program Files\Gensym\g2-2011\oracle\

Refer to your bridge documentation for further information on how to run the bridge.

Installing the Oracle Client

The G2-Oracle bridge will not run unless the Oracle Client software or an Oracle server is installed. Refer to the Requirements section of the G2-Oracle Bridge Release Notes for information.

Bridge KBs Installed with the G2 Bundle

The G2 Bundle installs the KBs of all bridges so that you can build bridge solutions, even when the bridge is not authorized. Bridge authorization installs the necessary bridge executable and the bridge OK file.
