Building with Microsoft Visual C++

IDE

The BullseyeCoverage setup program inserts a command into the Microsoft Visual Studio Tools menu to enable and disable Coverage Build. With Coverage Build enabled, build your project using command Build Rebuild.

If you do not see this menu entry, you can verify this change using the Visual Studio menu command Tools External Tools. For Visual C++ 6, the command is Tools Customize, then select the tab Tools. The settings should appear as shown in the image alongside. The Command setting is c:\Program Files\BullseyeCoverage\bin\CoverageBrowser.exe.

The setup program adds the BullseyeCoverage\bin to the top of the Executable Directories list. If you find that Coverage Build does not take effect, you can verify this setting using the Visual Studio menu command Tools Options then navigate to the VC++ Directories settings in the image alongside.

Command Line

You can build with BullseyeCoverage from the command prompt just as you normally do. Enable coverage build with the command cov01 -1 before building.

If you build with one of devenv, msbuild.exe or vcbuild.exe, you must have the BullseyeCoverage/bin in the Visual Studio Executable Directories as described above.

If you build with Microsoft build.exe, verify the BullseyeCoverage/bin directory occurs first in your PATH environment variable.

Microsoft Visual C++ 2005 Express

BullseyeCoverage requires user32.lib which is not supplied with Microsoft Visual C++ 2005 Express. You must obtain it yourself and place it in a directory where Visual C++ looks for libraries. You can obtain this file from www.bullseye.com/pub/.
