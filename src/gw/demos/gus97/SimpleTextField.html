Updating Field Widgets Example

This is an example of dynamically updated data feeds between WebLink and Java. The applet can be supplied multiple field names and data sources. The data can either be static (default) or dynamically updated at specified refresh rate.

Note: Due to yet-to-be-determined bugs in certain Java implementations you may need to do a "Reload" to jumpstart the updates. Go figure.

This applet was instantiated with the following HTML code: 

<applet code="SimpleTextField.class" codebase=applets width=450 height=200>
<param name=refreshRateInMS value="5000">
<param name=fontname value="TimesRoman">
<param name=fontslant value="italic">
<param name=fontweight value="bold">
<param name=fontsize value="24">
<param name=data0Name value="Who: ">
<param name=data0URL  value="http://siam:8080/IndividualAttribute?who=bob&attr=name">
<param name=data0Width value ="40">
<param name=data1Name value="Saying: ">
<param name=data0URL  value="http://siam:8080/IndividualAttribute?who=bob&attr=favorite-saying">
<param name=data1Width value ="40">
<param name=data2Name value="Activity: ">
<param name=data0URL  value="http://siam:8080/IndividualAttribute?who=bob&attr=activity">
</applet>
