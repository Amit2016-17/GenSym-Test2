Authorizing G2 WebLink

 Before running G2 WebLink, you must authorize your machine to do so. This requires a product authorization code sheet.

If you do not have a product authorization code sheet, call the Gensym Product Distribution Administrator at (617) 547-2500; if you have a touch-tone phone and are in the U.S.A., you can choose option 6, or you can ask the operator for Production and Licensing. Be prepared to supply your machine identification number.

You can obtain your machine identification number by attempting to run G2 WebLink before it is authorized. This results in an error message that includes your machine identification number, provided that the environment variable GSI_ROOT exists, and you have created an empty gsi.ok file.

When you have a product authorization code sheet, you can authorize the bridge.

To authorize the bridge:

  1. Set the following environment variable or logical, depending on your system:

    • For UNIX (and MS-DOS) systems, if the environment variable GSI_ROOT does not exist, you must set the value of the GSI_ROOT environment variable to the pathname of the GSI.OK file; otherwise, put the GSI.OK file in GSI_ROOT. On UNIX platforms, you must specify the GSI_ROOT file name in capital letters.

    • For VMS systems, you must set the logical GSI$ROOT to the pathname of the directory containing the GSI.OK file. On VMS systems, GSI$ROOT is defined for you if you have already installed GSI.

    • On Windows NT, if GSI_ROOT is not defined or G2 WebLink has not been properly authorized, the bridge process window will be closed immediately (seen as a brief flash), thereby preventing you from seeing any messages corresponding to the problem.

      In order to see the messages, open an MSDOS window and rerun the bridge by typing the name of the bridge executable, such as c:gensymbridgesgw. When the bridge process stops, the MSDOS window will remain open, allowing you to view the messages.

      To define GSI_ROOT, double-click control panels within the main program group. Double-click on system within the control panels group. Click anywhere within the System Environment Variables box. At the bottom of the dialog, replace the contents of Variable: with GSI_ROOT (do not click on the delete button). Replace the contents of Value: with the fully qualified path to your GSI.OK file, such as c:gensymbridges. Do not click on the delete button. Do not include the filename gsi.ok in the definition of GSI_ROOT.

  2. Enter a line in the GSI.OK file exactly as it appears on the product authorization code sheet. This line must contain the following fields, in order, separated by spaces or tabs:

    name


    		 machine ID


    		 product


    		 version


    		 time spec


    		 codes 1- 5


    name
    The name of the machine. Consult your system administrator or networking software manual to obtain the name.

    machine ID
    The identification number of the machine. (Include hyphens if they are part of your machine's identification number.) To obtain a machine's identification number: (1) create a GSI.OK file that contains nothing other than one blank line, and set the appropriate logical or environment variable; (2) run the bridge. A message will state that "there is no line containing a machine ID equal to machine ID" (your machine identification number).

    product
    The name of your bridge, G2-WebLink.

    version
    The version of the product. The syntax is: Vn where V is an upper or lower case "V" and n is an integer which represents the major version number (minor version numbers are ignored).

    time spec
    An optional field that only appears if the license is temporary. If the license is temporary, specify a time period as follows: 

      from day month year to day month year
    where:

      day
      A two-digit integer between 01 and 31.

      month
      One of the following three-letter abbreviations: jan, feb, mar, apr, may, jun, jul, aug, sep, oct, nov, or dec.

      year
      A four-digit integer, such as 1996.

      For example: 

      From 01 jan 1996 to 31 feb 1996
    codes 1-5
    The five product authorization codes specific to your machine and bridge license. In most cases, these codes are on your product authorization code sheet.

Sample Authorization File

 The following is an example of a GSI.OK file that contains a temporary authorization for one bridge and permanent authorization information (no From and to dates) for another.

    Line 1: 

    bridge-mach 80352734 G2-WebLink v1 From 01 may 1996 to 
31 oct 1996 8813869 5920288 414971 6937199 371245905
    Line 2: 

    myvax 08-00-2b-04-1a-10 G2-WebLink V1 11111 33375 555586 
77777 99999
Blank lines in authorization files are ignored.

Validating Your Authorization File

An authorization is valid if all of the following are true:

    • The GSI.OK file can be located by the G2 WebLink executable.

    • Some line within the GSI.OK file is in the proper format and contains the correct number of fields.

    • That same line specifies a machine ID that matches the machine-ID of the machine currently executing G2 WebLink.

    • That same line specifies the G2 WebLink product name that matches the product name of the executing G2 WebLink.

    • That same line, if it contains a time span, contains an unexpired time span.

    • That same line specifies five authorization integers (authorization codes) that the G2 WebLink executable recognizes as valid.
