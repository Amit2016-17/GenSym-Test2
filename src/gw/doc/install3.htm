Loading G2 WebLink from the Distribution Medium

On UNIX Systems

 To load G2 WebLink software from the distribution medium:

  1. Create a directory.

  2. Make that directory the current working directory (or specify the logical name for the directory).

  3. Use the tar utility to transfer files from the distribution medium.

    The following code sequence is a sample installation for UNIX operating systems. In this example, the pound sign symbol (#) is the shell prompt.

    For a directory named g2-weblink and a tape device named /dev/rmt/0m: 

    #mkdir g2-weblink

    #cd g2-weblink

    #tar -xvf /dev/rmt/0m
    Note: If you do not know the tape device name, see your system administrator.

On OpenVMS Systems

 To install G2 WebLink software, use the VMSINSTAL utility.

To load software using VMSINSTAL:

  1. Login as system.

  2. Enter the commands:

    $set def sys$update

    $@vmsinstal
  3. If prompted, reply yes to the following:

    Do you want to continue anyway? y

    Are you satisfied with the backup of your system 
disk[yes]? y
  4. To specify where you want the distribution volume mounted, enter the name of the device, for example:

    Where will the distribution volume be mounted? mka500

    If you do not know which device to use, see your system 
administrator.
  5. Type in the name of the product at the next prompt.

    Enter the products to be processed from the first 
distribution set. 

    * Products: G2WebLink
    where G2WebLink is the product name of the bridge, with a suffix giving the major version (2 digits) and minor version (1 digit).

  6. Accept the default installation options by pressing the Return key.

    Enter the installation options you wish to use [none]:
  7. If you are using a tape device, mount the tape in the drive and respond to the readiness prompt.

    Please mount the first volume of the set in MKA500.

    Are you ready? y

    The following products will be processed:

    G2-WebLink
    where G2-WebLink is the product name of the bridge, with a suffix giving the major version (2 digits) and the minor version (1 digit).

  8. The install program determines whether a prior version of the product exists. For example:

    * Prior G2WebLink found - replace [N]?

    If you reply n or if this is a first-time installation, 
the install program will prompt for the name of another 
directory.

    g2weblink directory
    
[BOSTON$DUA1:[G2.GSI30R1.G2WebLink]]:

    To accept the default directory, press the Return key. 
Otherwise, enter the name of the directory where you 
want the install program to store the files.
  9. The install program prompts for the location to place the specific bridge start-up command procedure:

    Place startup call in G2WebLink.COM [Y]:

    To accept the default, press the Return key. If you do 
not want to include the bridge start-up command 
procedure in GSI-STARTUP.COM, you must invoke: 
sys$startup:G2WebLink-startup.com from some other location 
before you can run the bridge.
  10. Press the Return key at the Products prompt to leave VMSINSTAL.

    Enter the products to be processed from the next 
distribution volume set.

    * Products:

    VMSINSTAL procedure done at [time]

On Windows NT Systems

 To install the G2 WebLink software under Windows NT, you must use the Gensym Bridge Installer. To use the Bridge Installer, follow these steps:

  1. Select Run from the File pulldown menu of the Program Manager or File Manager.

  2. Enter <drive-letter>:setup where <drive-letter> is the drive containing the distribution medium, and click on OK. For example:

    a:setup
  3. Read the instructions provided in the dialog boxes and respond to any questions. If you need help, click on the ? icon at the lower right of the dialog box for context-sensitive help.

  4. Refer to the release notes installed by the Gensym Bridge Installer for any post-installation instructions by clicking on the release notes icon.

If you choose to authorize the bridge with the Gensym Bridge Installer, you can skip the section "Authorizing G2 WebLink" below.
