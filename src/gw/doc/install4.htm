Installing G2 WebLink

 To install G2 WebLink on any platform, you must follow these steps:

  1. Create a subdirectory into which you can install G2 WebLink.

    On VMS systems, you install the product using the VMSINSTAL utility, which creates required subdirectories automatically.

  2. Download the G2 WebLink archive file into that subdirectory, following the steps listed in "Downloading G2 WebLink by FTP".

  3. Unpack G2 WebLink files. The following table summarizes the unpacking commands used on the platforms where G2 WebLink is supported:

    Platform Unpack Command
    Unix


    		tar xvf <filename>.tar


    Windows


    		pkunzip <filename>.zip


    VMS


    		$ @sys$update:vmsinstal <product>-<version> <pathname>


The following sections list the steps for installing G2 WebLink on Unix, Windows, and VMS systems.

Installing on Unix Systems

To install G2 WebLink on a Unix system, follow these steps:

  1. Create a target directory, using a command of the following form: 

    % mkdir /usr/product 

    For example: 

    % mkdir /usr/gwl0
  2. Set the current working directory to the target directory, using a command of the following form: 

    % cd /usr/product
    For example: 

    % cd /usr/gwl0
  3. Download the product, following the steps listed in "Downloading G2 WebLink by FTP".

  4. Unpack the downloaded archive file, which is in Unix tar format: 

    tar xvf <product><rev>.tar

Installing on Windows Systems
To install G2 WebLink on a Windows system, follow these steps:

  1. Create a target directory: 

    mkdir gensymproduct
  2. Set the current working directory to the target directory: 

    > cd gensymproduct
  3. Download the product, following the steps listed in "Downloading G2 WebLink by FTP".

  4. Unpack the downloaded archive file, which is in PKZIP format. Utilities which pack and unpack this format are widely available. For example: 

    > pkunzip <product><rev>.zip

Installing on VMS Systems
To install G2 WebLink on a VMS system, follow these steps:

  1. Login as system.

  2. Run the VMSINSTAL utility, providing as parameters the name of the downloaded product archive (without the .a extension) and the full path of the directory into which it was downloaded. Use a command of the following form:

    $ @sys$update:vmsinstal <product>-<version> <pathname>
    For example: 

    $ @sys$update:vmsinstal gw-10b4-c18
    
mydisc:[myhome.ftpdir]
  3. When queried about whether you are satisfied with your system disk backup, accept the default, Yes, by pressing the Return key: 

    Are you satisfied with the backup of your system 
disk[Yes]?
    The program responds with: 

    The following products will be processed:
    
GW V1.0

    Beginning installation of GW V1.0 at 10:09

    VMSINSTAL-I-RESTORE, Restoring product save set A ...
    
GW V1.0
  4. The program determines whether a directory location already exists for G2 WebLink files and sample knowledge base (KB) files. The default directory for G2 WebLink is <system-disk>:[gw10], where <system-disk> is the name of your system disk.

    If the directory exists, the program queries whether it should replace the existing contents of the directory. If you want to overwrite the old contents with G2 WebLink files from the tape, enter Yes. 

      Prior GW found - replace[n]?
    If you answer No, the program displays the name of the default directory for version 1.0 and prompts you to enter the name of another directory.
    If you are installing G2 WebLink for the first time, or the directory does not exist, the program displays the name of the default directory: 

      <system-disk>:[gw10]
    and prompts you either to accept the default directory or to enter a new directory location. In this example, the system disk name is dka300: 

      *GW directory [DKA300:[gw10]]:
    To accept the default directory, press the Return key. Otherwise, enter the name of the directory where you want G2 WebLink executable files to be stored.

  5. For the next several minutes the program continues, placing the correct disk name in G2 WebLink startup file, gw10-startup.com, and storing the file in sys$startup.

  6. The program checks the sizes of the VMS parameters PAGEFILE, WSMAX, and VIRTUALPAGECNT. If changes are required, the installation issues one or more of the following prompts, and builds a file called gensym_modparams.dat. By default, gensym_modparams.dat is stored in sys$specific:[sysexe].

    1. If the size of the PAGEFILE parameter does not meet or exceed the minimum size required by G2 WebLink, you are prompted to specify a new page file size or accept the default (50000). 

        PAGEFILE size to use [50000]:
    2. If the size of the WSMAX parameter does not meet or exceed the default minimum (2000) required by G2 WebLink, you are prompted either to accept the default by pressing the Return key or to specify a new size. If the new size is below the minimum, you are prompted again until the size specified meets or exceeds the minimum: 

        WSMAX size to use [2000]:
    3. If the size of the VIRTUALPAGECNT parameter does not meet or exceed the minimum, you are prompted either to accept the default (30000) by pressing the Return key or to specify a new size. If the new size is below the minimum, you are prompted again until the size specified meets or exceeds the minimum: 

        VIRTUALPAGECNT size to use [30000]:
    4. The program records the following action items concerning modifying PAGEFILE, WSMAX, and/or VIRTUALPAGECNT in a file called
      installation_action_items.com. The file is stored in gw10$dir. 

        $! Increase PAGEFILE size to <pagefile-size>

        $ sysgen create pagefile-name/size=<pagefile-size>

        $ rename SYS$SPECIFIC:[SYSEXE]GENSYM_MODPARAMS.DAT 
MODPARAMS.DAT.

        $!Run AUTOGEN from SYS$UPDATE: 

        $ @AUTOGEN GETDATA REBOOT NOFEEDBACK.

        <pagefile-size> is the page file size you 
specified at the prompt, or the default if you 
accepted the default. pagefile-name is the 
directory specification of your system's page file.
      After you complete the installation, look at the file:

        installation_action_items.com
      to complete each recommended action that may be listed there (further instructions appear in step 12 of this procedure). Note that the file is created even if no actions are recorded in it; the file is empty if you do not need to reset any of the parameters.

  7. The program prompts for the proper location to place the call to G2 WebLink start-up command procedure. Gensym recommends that you accept the default when installing G2 WebLink for the first time, sylogin.com, which is located in sys$manager. To accept the default, press the Return key:

      *Place startup call in SYLOGIN.COM [Y]:
  8. The next few prompts concern the building of a standard G2 WebLink user account, with the correct set of quotas, privileges, and permissions. The account has the default home directory of gw10$dir. If you do not have an account correctly set up to run G2 WebLink, Gensym recommends that you let the installation program create this standard account. Answer the prompts as follows:

    1. When asked whether you want to create the account, press the Return key to accept the default, YES: 

        Create a GW user account [Y]?
    2. When prompted for the name of the user account, press the Return key to accept the default, GWUSER, or specify another name: 

        Name of account [GWUSER]:
    3. When prompted for the UIC for the account, press the Return key to accept the default ([200, 110]), or specify your own limits: 

        UIC of account [[200,110]]:
    4. When prompted for the password of the account, press the Return key to accept the default (gensym), or specify another password: 

        Password of account [gensym]:
      If messages are displayed outside the Motif window that push the window down the screen, you can enter a Control/F2 key sequence to pull the window back into position.

  9. The following applies if:

    ¯ You did NOT create a new account in step 8, and if

    ¯ You specified a new, non-existent directory in step 6.

    If so, you will need system privileges to change the directory and file ownership. You will see the following message, which describes what to do once the installation is complete: 

      When the installation script is complete, you should 
set the ownership of the installed files 
appropriately. Use the "set file/owner=" command to 
do this. For example, if you have installed the files 
in dka300:[gensym], type both of the following 
commands, replacing <owner> with the account of your 
choice. 

      $ set file/owner=<owner> dka300:[000000]gensym.dir

      $ set file/owner=<owner> dka300:[gensym...]*.*
    The message is followed by the prompt: 

      Press RETURN to acknowledge [OK]?
    Respond by pressing the Return key.

  10. The program displays on the screen the action items, if any, that it recorded in the installation_action_items.com file, preceded by this message:

      The following action items file was produced. If the 
file is not empty (there are items to perform), you 
can type the file out and perform the actions 
manually or you can run the file when you are 
finished with VMSINSTAL.

      To type the file, from the "$" prompt, type

      $type gw10$dir:installation_action_items.com

      To perform the actions automatically, from the "$" 
prompt, type

      $ @gw10$dir:installation_action_items

      ****NOTE: running the file will REBOOT YOUR 
MACHINE****

      Installation complete
      

      
*Products:
    Remove the tape from the tape drive. If you want to terminate the installation program, press the return key.

  11. You have now installed G2 WebLink.

    Look at the installation_action_items.com file and perform any tasks that it lists, either manually or automatically, as explained in step 10 of this procedure. For information about how to run the autogen utility, see the Installation Guide for G2, Telewindows, GSI, and G2 GUIDE/UIL.
