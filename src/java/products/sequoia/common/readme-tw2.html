What is Telewindows2 Toolkit?

Telewindows2 Toolkit (abbreviated as TW2) is a set of G2-aware UI components that you can use to implement a graphical user interface for any G2 application. The Telewindows2 Toolkit components are delivered as Java Beans that can be used in a variety of development environments to assemble a user interface.

Telewindows2 Toolkit can be installed as a stand-alone bundle, or it can be installed into an existing G2 bundle.

Software Requirements

You need the following software correctly installed on your machine to use Telewindows2 Toolkit:

  • Java 2 (JDK Java Development Kit):

    • Windows, Solaris, Linux: Sun's Java 2 (JDK 1.4 Java Development Kit).

    • HP-UX: HP's Java 2 (JDK 1.4 Java Development Kit).

    • AIX: IBM's Java 1.4.1 (IBM Developer Kit for AIX, Java Technology Edition).

    • HP Tru64 UNIX: HP's Java 1.4 (HP Java 2 SDK (J2SDK).

  • G2 JavaLink Version 8.2 Rev. 1, which is available from your Telewindows2 Toolkit Bundle or your G2 Bundle.

  • G2 Version 8.2 Rev. 1 or higher, which is available from your G2 Bundle, is recommended. At least G2 Version 5.1 Rev. 4 is required.

    Note that with old versions of G2, you may experience difficulty in downloading classes that have changed in later G2 versions, such as g2-window.

We recommend that you also see the G2 JavaLink Version 8.2 Rev. 1 readme-javalink.html file for additional information.

System Requirements

Note: The operating system requirements on some UNIX platforms have changed for G2 Version 8.0 Rev. 0. Please review the minimum requirements carefully.

To use Telewindows2 Toolkit, you need:

  • Operating system (minimum version):

    • Windows XP, Windows 2003, Windows 2000, or Windows NT Version 4.0 with Service Pack 6 or 6a, all running on a Pentium 4 computer.

    • Solaris 2.6 running on a Sun Sparc computer.

    • HP-UX 11.0 running on an HP9000 computer with the PA 2.0 architecture.

    • AIX 5.1* running on an IBM RS/6000 computer.

    • Red Hat Linux or Red Hat Enterprise Linux running Linux kernel 2.4.26 or greater

    • HP Tru64 UNIX 5.1A running on a HP Alpha computer.

  • Minimum of 64 MB of RAM.

  • Approximately 100 MB of disk space to install the software and documentation.

* G2 JavaLink requires AIX 5.1; G2 only requires AIX 4.3.3.

Special Notes

Applets

Applets based on Telewindows2 Toolkit components require the Java Plug-in, which is included in the JRE Version 1.4.

Support for Telewindows2 Toolkit ActiveX Controls

Unfortunately, Microsoft has chosen not to upgrade their Java Virtual Machine past Version 1.1.8. The latest version of Telewindows2 Toolkit requires Java 1.2 or later, preferably Version 1.4. Microsoft's decision limits Gensym's ability to continue support for newer versions of the Telewindows2 Toolkit ActiveX controls. The Telewindows2 Toolkit ActiveX control is no longer included in the product.

We will continue to support the ActiveX controls provided in Telewindows2 Toolkit Version 1.1 Rev. 0. If you experience problems with this version, please continue to report these through HelpLink and use the escalation procedures for serious issues, as appropriate.

With the introduction of the G2 Bundles, we generally recommended that the components from a release be used together since they have been tested together. In this case, however, Gensym's G2 developers say that Telewindows2 Toolkit Version 1.1 should be largely compatible with the G2 6.x component with the following known exceptions:

  • One configuration that we know does not work well is an older version of TW2/JavaLink on Windows trying to connect to G2 Version 6.0 on UNIX. Note that the appropriate knowledge base files need to be recompiled, as described in the release notes.

  • You must run Telewindows2 Toolkit and G2 on the same Windows machine; unfortunately, running TW2 Toolkit and G2 on different Windows machines does not work.

Item Configuration Changes

If you edit the item configuration of an item in the G2 server, you must redownload the workspace view to see the changes.

Setup Instructions

Telewindows2 Toolkit is available as a stand-alone bundle. You can install it as a stand-alone bundle or into an existing G2 bundle.

Installing Telewindows2 Toolkit

  1. If you are running TW2 Toolkit in a development environment, install the correct version of Java 2 (JDK 1.4 Java Development Kit) for your platform. If you are running TW2 Toolkit in a deployment environment, you do not need to install the JDK. Instead, you can use the version of JRE located in the jre directory of your TW2 Toolkit product directory.

    For details, see Software Requirements.

    The default install directory depends on your platform.

  1. A minimum of G2 Version 5.1 Rev. 4 is required.

  2. Ensure that G2 is authorized to connect to G2 JavaLink and Telewindows2 Toolkit clients. Your G2 is authorized to use G2 JavaLink when it has the code "jl" listed in the "authorized-products" attribute of its .ok product authorization file. G2 is authorized to use Telewindows2 Toolkit when it has the keyword maximum-number-of-concurrent-floating-tw2-allowed in the authorization file. Please contact Gensym Customer Support if you encounter any authorization problems.

  3. Install G2 JavaLink from the Telewindows2 Toolkit Bundle or the G2 Bundle according to the installation instructions.

  4. Ensure that the following environment variables are set properly to reflect this installation:

    All platforms: JAVA_HOME, G2V8_OK, PATH, and CLASSPATH
    Solaris, HP-UX, and LINUX: LD_LIBRARY_PATH
    AIX: LIBPATH

    For details, see Setting Environment Variables.

Setting Environment Variables

To run Telewindows2 Toolkit, you must explicitly set the following environment variables:

  • JAVA_HOME must point to the JDK directory. If you are running in a deployment environment, JAVA_HOME must point to the jre directory, which is installed as part of the G2 bundle.

  • G2V8_OK must point to the location of the g2.ok file.

  • PATH, as well as platform-specific library path variables, must point to the Java and G2 JavaLink bin directory.

  • CLASSPATH must include the necessary Java, G2 JavaLink, and Telewindows2 Toolkit class files.

To create and modify these environment variables, you can run a batch file or shell script, depending on your platform, as follows:

Platform Command

Windows: 

\tw2\bin\envvars.bat

UNIX: 

/tw2/bin/set-envvars

Running this batch file or shell script sets the environment variables for a single command-prompt session; it does not set them permanently. We recommend that you copy and modify the batch file or shell script to include your own environment variables, then call this file each time you start your application, for example, from a shortcut. For convenience, the envvars file also sets the JAVALINK_HOME, SEQUOIA_HOME, and SEQUOIA_G2 environment variables for a single session to point to the javalink, tw2, and g2 directories, respectively.

To run the demos, you must run the batch file or shell script in the same command window or shell from which you run the demo. On Windows platforms, choosing one of the demos from the Start menu automatically runs the batch file. For more information, see How Can I Begin to Use Telewindows2 Toolkit?.

You can also explicitly set these variables. On Windows platforms, you set the environment variables on the Advanced tab of the System Properties dialog, which you access Programs > Settings > Control Panel > System. On UNIX platforms, you set the environment variables in a csh shell by entering setenv VARIABLE value.

How Can I Begin to Use Telewindows2 Toolkit?

Note: Before you can run the demos, run the default application shell, or create your own application, ensure that environment variables are set up correctly, as described in Setting Environment Variables.

In the descriptions that follow, the directories are located in the tw2 subdirectory of your Telewindows2 Toolkit or G2 installation directory.

After installing Telewindows2 Toolkit and G2 JavaLink you can:

  • Run the Demos

    This release ships with a collection of demos. Each of these can be found in the following directories:

      Windows: \tw2\classes\com\gensym\demos
      UNIX: /tw2/classes/com/gensym/demos

    They include either a rundemo.bat file to run the demo and/or a readme file that describes how to run it. You can run the basic demo by:

      Windows: Choosing Start > Programs > Gensym G2 8.2r1 > Examples > Telewindows2 Toolkit > Telewindows2 Toolkit Demo.
      UNIX: Running the /tw2/bin/t2demo script from a shell.

    You can run the additional demos by:

      Windows: Choosing Start > Programs > Gensym G2 8.2r1 > Examples > Telewindows2 Toolkit > Internationalization Demo or Item Access Demo or Palette Demo .
      UNIX: Running the /tw2/bin/rundemo.sh scripts from a shell.

  • Run the prototype Telewindows2 Toolkit Application Shell

    • Run the shell by either:

      • Choosing Start > Programs > Gensym G2 8.2r1 > Telewindows2 Toolkit > Default Application Shell.

      • Using the command: java com.gensym.shell.Shell

    • View the source code that implements the shell under:

        Windows: \tw2\classes\com\gensym\shell
        UNIX: /tw2/classes/com/gensym/shell

      There is a batch file to launch the shell and the demo in the bin directory.

  • Make your own stand-alone application that contains Telewindows2 Toolkit components, using Java or a third-party IDE such as Visual Cafe.

  • Make your own Telewindows2 Toolkit components in Java and use them in Java, third-party IDEs, or a Web Browser.

  • Query for the current list of bugs to make sure that you are not encountering known problems.
