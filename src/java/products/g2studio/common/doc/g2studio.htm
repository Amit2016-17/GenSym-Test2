Establishing a Connection to a G2 Server

You can establish a G2 connection within G2 Studio by using the Location field or the Open Connection dialog.

To connect to a G2 Server using the Location field:

    > In the main application window, type a url in the Location text field located on the second toolbar.

    For example, to connect to a G2 server located on another platform with host-name babs at port 1112, enter g2://babs:1112/ as shown in this example:

To connect to a G2 Server using the Open Connection dialog:

  1. Select File > Open > Open Connection.

  2. Enter a host and a port. You can also optionally enter a middle-tier url, your user name and mode, and your password.

When G2 Studio locates a G2 server, it displays a Login dialog where you enter your user name, user mode, and password if you have not already entered this information on the Open Connection dialog.

You can change the default values of the Login dialog by using the Preferences dialog, which is accessible through Edit > Preferences. You can also change your default HomePage to either a file url, an http url, or a G2 url.

To close a connection to a G2 host:

    > Select File > Close Connection.

Establishing Connections to Multiple G2 Servers

G2 Studio can be concurrently connected to multiple G2 servers.

To connect to other G2 hosts:

    > Repeat one of the connection methods outlined in the previous section.

G2 Studio can have only one current connection at a time. By default, a connection is made current when it is first opened; however, the other connections are kept open, and you can easily make one of them the current connection.

To make another connection the current connection:

    > Select the connection url from the Location text field drop-down list,

    or

    Select the tab for a connection from the bottom tab bar.

Changing KB Focus When Connected to Multiple G2 Servers

When you select the tab for an individual connection on the connection toolbar, G2 Studio displays the visible workspaces for the KB in that connection only. When you select the All tab, G2 Studio displays the visible workspaces for the KBs in all the open connections simultaneously; however, only one KB is in current focus.

G2 Studio tells you which KB/connection is the current one by showing its url in the Location text field and by displaying the host name and port on the bottom status bar, as shown in the example above. In addition, the knowledge-base browser in the left window displays the module, class, or workspace hierarchy for the current KB/connection.

When you have multiple KBs displayed, you can change KB focus by selecting a workspace window from any of the visible workspaces. The current KB/connection then switches to the KB/connection that has the workspace.

G2 Studio's toolbars and main menu choices are specific to the current connection. For example, the G2 system tables accessible from the G2 main menu choice display the system tables for the knowledge base in the currently connected G2 server, and the Save menu only saves the knowledge base in the currently selected G2.

Loading, Saving, and Clearing Knowledge Bases

From the File menu, you can load, save, and clear knowledge bases in the G2 process that is currently selected.

To load a knowledge base:

    > Select File > Open > Open KB.

From the Open KB dialog, you can browse the file system for KBs, or you can type a file location directly into the Path text field. Note that if the currently selected G2 process has a resident KB, G2 deletes it before loading another KB.

When the KB is loaded, its module hierarchy appears in the Module Browser pane, which is docked on the left-hand side of the main application window. Any modules that have notes appear in red to alert you that items in the module have inconsistencies.

To clear the current KB:

    > Select File > Close > Close KB.

To save the current KB:

    > Select File > Save | Save As | Save As All.

To merge a KB into the current KB:

    > Use the Merge Module dialog launched by selecting File > Modules > Merge.

    The named module and its required modules are loaded without deleting the current KB.

To delete a module from the current KB:

    > Use the Delete Module dialog launched by selecting File > Modules > Delete.

    or

    Right click a module in the Module Browser to bring up its menu and select Delete.

    The Delete-associated-workspaces option on the Delete Module dialog is preselected.Deselect that option if you do not want the module workspaces to be deleted.

Creating Items

To create a workspace:

    > Select File > New > New Workspace

You can interact with a workspace in many ways: by zooming it, by changing its colors and dimensions, and by creating new workspace items, moving them around, resizing and rotating them, and editing their properties.

You hide a workspace from view by clicking the x at the right corner of the workspace window. To permanently delete a workspace, either right-click on the workspace to bring up its popup menu and select Delete, or choose Workspace > Delete. Note that a workspace may disappear from view when the KB issues a workspace-hide command, or when the connection associated with the workspace is closed.

To create workspace items from the Item Palette:

  1. Right-click on a workspace to bring up its menu.

  2. Select New Item to bring up the Item Palette:

  3. Select a palette tab.

  4. Click an item icon and transfer the item to any workspace.

To create workspace items from class-definitions:

  1. Right-click the class-definition of a defined class to display its popup menu.

  2. Select Create Instance.

The instance is created on the workspace of the class-definition. You can cut and paste the item to another workspace.

Customizing the Item Palette

Use the Class Chooser dialog to add items to the Item Palette or to delete items from the Palette.

To launch the Class Chooser dialog:

  1. Right-click the Item Palette background to bring up its popup menu.

  2. Select the Customize Palette menu choice.

You add item icons to the palette by selecting them from the class hierarchy tree in the left pane of the dialog and clicking the right arrow. You delete item icons from the palette by selecting them from the Selected Classes list and clicking the Remove button.

Launching the Icon Editor

To launch the icon editor for a class:

  1. Right-click a class-definition for a defined class.

  2. Select Edit Icon from the popup menu to bring up the icon editor:

Classes without iconic representations have no Edit Icon menu choice; for example, message classes.

To launch an Item Properties dialog:

    or

Finding Items

Finding Items Through the Location Field

The Location text field in the G2 Studio main application window offers a convenient way to locate and display items.

To access a named item:

    > Add /named/<item-name> to a G2 connection url in the Location text field.

    Entering g2://localhost:1111/named/ship establishes a connection to localhost at port 1111, if it is not already connected, and downloads and displays the selected method-declaration named ship as shown in this example:

To access an unnamed item:

    > Add /with-uuid/<uuid> to a G2 connection url in the Location text field.

    For example, entering

      g2://localhost:1111/with-uuid/420f35c5b71611d39aea000000000000

    establishes a connection to localhost at port 1111, if it is not already connected, and downloads and displays the selected item identified with that uuid.

To access a previously viewed item:

    > Select the item's url from the Location text field drop-down list.

Accessing Items Using the Find Items Dialog

G2 Studio's Find facility is similar in design to the Find application in Windows. It allows you to search for items by name, class, or by using a G2 expression. You can also display sets of attributes and go to a found item from its popup menu.

To find items and attributes:

  1. Type Ctrl + F or select Tools > Find to launch the Find Items dialog.

Retrieving Workspaces with the Open Workspace Dialog

To retrieve a named workspace from the Open Workspace dialog:

  1. Select File > Open > Get Workspace

  2. Select a workspace from the dialog.

The workspace is displayed and its url is added to the Location text field.

Navigating to Items From an HTML Document

G2 Studio supports clicking on G2 url hyperlinks within an html document. This facility gives you a good way to present your KB knowledge because urls provide convenient navigation throughout the KB hierarchy.

Navigating to Knowledge-Base Views

When you have many workspaces and documents open and displayed, it is sometimes difficult to move fluidly between them. For this reason, G2 Studio has a navigation toolbar with arrow buttons for moving forward and backward to different views, as well as for returning to your home url. These navigational options are also available through the Go menu, as well as through these key accelerators: Alt + Left for moving backward and Alt + Right for moving forward.

You can also navigate to next and previous window views using Ctrl + Tab and Ctrl + Shift + Tab, respectively, or select from the list of open windows displayed under the Window main menu option, which also has the standard options for tiling and cascading windows.

Accessing Knowledge-Base Status Information

KB information can be accessed from the top-most toolbar, from the KB's set of system tables, and from the Logbook and Message Board.

The top-most toolbar displays Start, Reset, Restart, and Pause buttons. These choices are also available through the G2 menu choices. What selection of choices is available depends on G2's current run state. The top-most toolbar also has a drop-down text field that displays the user mode for the current connection.

The View menu gives you access to the Logbook and the Message Board; and G2 > Server System Tables gives you a menu of the system tables for the current connection.

Browsing Knowledge-Base Hierarchies

G2 Studio offers an explorer-style interface for browsing the various hierarchies within your knowledge base. These hierarchy browsers can be found under the Tools main menu.

The left-hand side of a browser has a tree view of the specified hierarchy. On the right side is a tabular viewing component, displaying the sub-items of the currently selected tree node, as well as any instances of that node.

For the module hierarchy, the required modules for the selected module node are displayed along with the system tables and top-level workspaces for the module. For a workspace hierarchy, all subworkspaces, as well as all the items on that workspace, are displayed.

Using the Toolbars and Status Bar

G2 Studio offers many convenient toolbar buttons for performing the most frequently selected operations, such as getting workspaces, cutting and pasting items, starting and pausing G2, and changing the scale of the current workspace. In addition, there is an extra item toolbar on the bottom of the application main window, that contains simple buttons for nudging and aligning items on a workspace, changing item layering positioning, or manually modifying the width, height, or position of an item.

Try exploring the toolbars yourself and see what else is available. If you aren't quite familiar with the icons, hover the mouse over a button and a Tool Tip is displayed describing its functionality. All toolbar-button operations are also available from the main menu.

Just below the Item toolbar is the Status bar, located on the bottom-most portion of the main application window. Here are various status regions, displaying such information as the current G2 connection, that connection's run state, and a region for displaying the table header of a selected item. Located at the left-most portion of the status bar is an animated wait bar, which moves left and right when G2 Studio is busy doing an operation, such as trying to establish a connection to a remote G2 server.
