Gensym BeanXporter 1.0 Rev 2

Gensym BeanXporter automatically converts an ActiveX control to a JavaBean component by wrapping the control to meet JavaBean standards. Once converted, the new Java Bean is ready for use in your Windows-based Java applications as you would any other Bean. Since the new Java Bean bridges to the original ActiveX control, you get the assurance that you are working with the original functionality of the ActiveX control.

BeanXporter also lets you use Microsoft Office documents as Java Beans. With BeanXporter your Windows-based Java projects can take advantage of ActiveX controls and Office documents such as those for spreadsheets, charts, calendars, word processors, reports, specialized graphics, and many more.

Requirements

  • Hardware: Intel-based PC
  • Operating Systems: Windows 95/98, Windows NT 4.0
  • Java: Sun’s Java Development Kit (JDK) 1.1.6 or later, including Sun's JDK 1.2 Beta 4
  • Memory for Runtime:8 Meg of RAM
  • Memory for Export: Minimum: 16 Meg of RAM, Preferred: 32 Meg of RAM
  • Storage for Runtime: 3 Meg
  • Storage for Export: Minimum 10 Meg, Preferred: 30 Meg

Installation

If you have not done so already - download the software by visiting the BeanXporter home page at "http://www.gensym.com/java"
  • Check you have installed Sun's Java Development Kit (JDK) 1.1.6 or greater, available from the JDK Download Page.
  • Check that the environment variable JAVA_HOME refers to the installation of the JDK.
  • Run the self extracting executable which will guide you through installing the software and documentation.

Setting Environment Variables

BeanXporter requires environment variables that specify the locations of your Java Development Kit (JDK) and BeanXporter installations. The following table describes the environment variables that must be created or appended to:

Variable Description
JAVA_HOME Specifies the absolute path to your installation of the JDK. The beanxporter.bat file uses the JAVA_HOME variable to locate a Java interpreter to launch BeanXporter.
BEANXPORTER_HOME Specifies the absolute path of the BeanXporter installation.
PATH Specifies the absolute path to the directory containing the beanXporter.bat file and the JavaContainer.dll library, so that the Java interpreter can locate the library at runtime.
CLASSPATH Specifies the absolute path to directories used by BeanXporter, which enables the Java interpreter to:
  • Load the Java classes required by BeanXporter. These classes are located in the ax2jbeans.jar file.
  • Load classes that were emitted by BeanXporter.

Setting Environment Variables on Windows NT

On Windows NT, all the environment variables are automatically set during installation, except for the JAVA_HOME variable. Setting the JAVA_HOME variable is described in this section. If you want, you can modify all the environment variables using the Control Panel.

The following table shows examples of the environment variables defined for Windows NT:

Variable Example Value on Windows NT
JAVA_HOME C:\jdk-1.1.6
BEANXPORTER_HOME C:\gensym\bx-10r2
PATH C:\WINNT\system32;
C:\WINNT;C:\JDK-1.1.6\bin;
%BEANXPORTER_HOME%\bin
CLASSPATH .;C:\jdk-1.1.6\lib\classes.zip;
%BEANXPORTER_HOME%\classes;
%BEANXPORTER_HOME%\classes\ax2jbeans.jar

To set the JAVA_HOME variable on Windows NT:

  1. Open the System icon in the Control Panel.
  2. Click the Environment tab.
  3. Create the JAVA_HOME variable, or, if one exists, check to see that it is correct.
  4. Set the value of the JAVA_HOME variable, for example:
    C:\JDK-1.1.6

Setting Environment Variables on Windows 95/98

On Windows 95, you must edit the autoexec.bat file to set the environment variables.

To set environment variables on Windows 95/98

  1. Open a text editor, such as NotePad or WordPad;
  2. Open the autoexec.bat file on drive C;
  3. Set the value of the JAVA_HOME variable, for example:
    SET JAVA_HOME=C:\JDK-1.1.6
  4. Set the value of the BEANXPORTER_HOME variable, for example:
    SET BEANXPORTER_HOME=C:\GENSYM\BX-10R2
  5. Modify the value of the PATH variable, for example
    SET PATH=C:\WINDOWS;C:\WINDOWS\COMMAND;C:\JDK-1.1.6\BIN;
    %BEANXPORTER_HOME%\BIN
  6. Modify the value of the CLASSPATH variable, for example:
    SET CLASSPATH=.;C:\JDK-1.1.6\lib\classes.zip;%BEANXPORTER_HOME%\classes;
    %BEANXPORTER_HOME%\classes\ax2jbeans.jar
  7. After completing all the edits to the AUTOEXEC.BAT file change the environment variables by executing the following in a DOS shell:
    C:> autoexec.bat
Note - Beta users on Windows 95/98 who are upgrading to 1.0 Rev 2, only need to edit the BEANXPORTER_HOME environment variable.

Configuration

Configuration of BeanXporter is controlled by the .com.gensym.properties file. During installation this file will be added to the directory specified by the HOME environment variable. If a HOME environment variable is not defined the properties file is added to the classes directory of the BeanXporter install. The .com.gensym.properties file defines the following properties:
  • com.gensym.emitdir - the default root location where BeanXporter will emit files
  • com.gensym.emitpackage - the default root package into which BeanXporter will emit classes
See the Users Guide for a more detailed discussion of these properties.

Installed Contents

The following contents can be found in the BeanXporter 1.0 Rev 0 installation:
  • bin\JavaContainer.dll - DLL that provides native support for BeanXporter conversion and runtime
  • bin\beanXporter.bat - Batch file for launching BeanXporter
  • classes\ax2jbeans.jar - the Java classes that implement BeanXporter and provide the com.gensym.com API.
  • classes\axtools.jar - A jar containing the ActiveXGenericComponent and ActiveXTraceTool beans.
  • classes\com\gensym\demos\axbeanbox - Files for using BeanXporter with Sun's BeanBox.
  • classes\com\gensym\demos\inetbrowser - Example using Internet Explorer 4.0.
  • classes\com\gensym\demos\tablebrowser - Example using Excel 98. Requires Swing 1.0 to be installed.
  • docs - API and other documentation.

Bug fixes

See the on-line list of known problems for information on which bugs have been fixed since Beta 0.

BeanXporter 1.0 Rev 2 also includes the following changes that were made in Rev 1, Rev 0, Beta 1 and Beta 2:

Runtime Error reporting

BeanXporter 1.0 Rev 1 includes better error reporting. Failures to dispatch ActiveX methods are now propagated to Java. If the "emit exceptions" option is chosen in the BeanXporter wizard these failures will result in an ActiveXDispatchException being thrown at runtime. The ActiveXDispatchException will contain the error code of the failure and additional information when provided by the ActiveX component. If the "emit exceptions" option is not chosen in the BeanXporter wizard the error will appear in a trace that can be retrieved by using the ActiveTraceTool class.

BeanInfo classes

BeanXporter 1.0 Rev 1 will create one BeanInfo class for every Java Bean that is emitted. The BeanInfo class simplifies the method, property and event interfaces presented to a Beans editing tool.

Events

If an ActiveX component has an event source interface an analogous Java Listener interface and event class is emitted by BeanXporter. 1.0 Rev 1 makes it possible to add listeners to events taking place in ActiveX components. See the InternetExplorer example in classes\com\gensym\demos\inetbrowser for an example use of the Events facility.

Frequently Asked Questions

Q. How does BeanXporter work?

A. For developers, BeanXporter provides a conversion tool that first reads the type library associated with the ActiveX control, and then emits the appropriate Java code needed to export the control for use as a JavaBean component with all the original properties, methods, and events. The Java code is automatically compiled into Java bytecode and added to Java Archive Files (.JAR). The end result is that the original ActiveX control is wrapped with the appropriate Java code. For runtime use, BeanXporter provides a lightweight Dynamic Linked Library (DLL) and a small set of Java classes that enable runtime communication between Java and ActiveX controls.

Q. What platforms does BeanXporter support?

A. BeanXporter's developer conversion tool and runtime files run on Intel-based Microsoft Windows 95 and NT platforms. At this time, Gensym does not plan to support platforms other than those based on Microsoft Windows.

Q. Does BeanXporter convert ALL ActiveX controls to JavaBean components?

A. There are thousands of commercially available ActiveX controls. The vast majority of these controls will be convertible with the BeanXporter. Gensym has tested the BeanXporter with the most popular ActiveX controls and OLE documents.
