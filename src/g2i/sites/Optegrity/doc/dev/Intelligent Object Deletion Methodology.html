Intelligent Object Deletion Methodology

 

Deleting an Intelligent Object requires some significant application cleanup.  Related event messages, event objects, and event scheduler lists must be detected and modified or deleted for the deleted Object and all its sub-Objects (i.e. heater cell & pass objects) in addition to deleting the graphical object and its subworkspaces.  The primary tasks that must be performed for deleting an Object are:

 

Delete all Object (& sub-Object) event objects.

Delete all Object (& sub-Object) event messages.

Remove the Object and sub-Object names from all event scheduler lists.

Delete the Object.

 

While there are many approaches to performing these tasks, it will probably be easiest to make use of existing methods to minimize the development time.  Therefore, the following procedure should be considered:

 

Set the Object availability to NOT-AVAILABLE.

Call the event scheduler(s) for the Object, which:

Clears current messages for the Object and its sub-Objects.

Deletes any event objects for the Object and its sub-Objects.

Create a temporary event selection maintenance object.*

Event selection maintenance object defaults to all events disabled.*

Call the Object event selection method, which:*

Clears the Object or sub-Object name from each event scheduler symbol list.*

Deletes each event attribute object.*

Delete each sub-Object.

Delete the Object.

 

 

* These tasks that can be greatly simplified or eliminated if the symbol-list scheduling approach is replace by related event attribute object detection in the Intelligent Object architecture.
