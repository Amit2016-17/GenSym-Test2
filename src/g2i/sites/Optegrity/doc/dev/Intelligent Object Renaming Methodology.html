Intelligent Object Renaming Methodology

 

Renaming an Intelligent Object requires some event by event manipulation.  Related event messages, event objects, and event scheduler lists must be detected and modified or deleted for the renamed Object and all its sub-Objects (i.e. heater cell & pass objects).  The primary tasks that must be performed for renaming an Object are:

 

Stop current event processing for the Object.

Remove Object (& sub-Object) names from scheduler lists.

Rename Object (& sub-Objects).

Replace occurrences of former name in any assigned messages.

Rename any assigned custom messages that use the former name as part of the message object name.

Insert new names into the scheduler lists.

Restart event processing for the Object.

 

While there are several approaches to performing these tasks, it will probably be easiest to make use of existing methods to minimize the development time.  Therefore, the following procedure should be considered:

 

Set the Object availability to NOT-AVAILABLE.

Call a procedure (new) to remove the Object and sub-Object names from the event scheduler lists.*

Call a procedure (new to rename both the Object and sub-Objects).

Call a method for each Object and sub-Object event attribute object that:

Replaces the former name with the new name in the names of any assigned ESIS Rule Objects

Replaces the former name with the new name in the following parameters for each assigned ESIS Rule Object:

Operator Message

Next Best Action

Explain

Message Cause

Call a procedure (new) to put the Object and sub-Object names into the appropriate event scheduler lists.*

Set the Object availability to AVAILABLE to start processing.

 

 

* These tasks that can be greatly simplified or eliminated if the symbol-list scheduling approach is replace by related event attribute object detection in the Intelligent Object architecture.
