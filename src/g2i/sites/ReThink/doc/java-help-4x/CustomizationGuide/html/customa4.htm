Topic - Customizing Instruments

Common Customization Attributes of Instruments

The common attributes of instruments available in Developer mode are:

Attribute
Description
gfr-uuid

The internal identification number of the object.
notes

The current status of the instrument.
names

A symbol that represents the internal name of the instrument. ReThink uses the label attribute rather than the names attribute to identify instruments and avoid naming conflicts between ReThink models.
reset-procedure-name

The procedure name the instrument uses when the simulation is reset. The default value is bpr-reset-instrument.

stop-procedure-name

Available for backward compatibility only. See note below.
animation-subtable

Subobject that specifies the default colors the instrument uses when it is in an active, inactive, or error state. The default value is an instance of a bpr-probe-animation-subtable or a bpr-feed-animation-subtable, depending on the type of instrument.

For additional information, see:

G2 Reference Manual.
Customizing the Default Behavior of Blocks or Instruments.
Editing Subobjects
Caution: The attribute stop-procedure-name for instruments is available for backward compatibility only. When you upgrade your model to ReThink Version 3.1, you should migrate from procedures to methods because support for this named procedure will be removed in a future release of ReThink.
