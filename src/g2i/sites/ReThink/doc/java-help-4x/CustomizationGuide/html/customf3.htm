Topic - Customizing the User Interface

Customizing Properties Dialogs

When subclassing ReThink blocks, instruments, resources, and work objects, you often define class-specific attributes. By default, ReThink creates a new tab on the properties dialog named User Tab on which it places all the class-specific attributes of the object.

When subclassing objects, you might define a combination of public and private attributes. You might want to display only the public attributes on the properties dialog, and you might want to control the organization of the information on the User Tab or on different tabs.

ReThink determines the appropriate user interface control for displaying and editing the attribute value, based on the attribute type. For example, if the attribute is a numeric value it will display an editable field with spin buttons to increment or decrement the value. If the attribute is a color, ReThink creates a dropdown list of all available colors. You might want to use a different control for a particular attribute. For example, you might want to provide specific data types or define a minimum and maximum value for a numeric attribute.

You can extend the properties dialogs for ReThink blocks, instruments, resources, and work objects by customizing:

The order in which user-defines attributes appear on a tab.
The tab on which the attributes appear and the tab name.
The user interface control that the attribute uses.

You can only add and modify attributes on user-defined tabs; you cannot add attributes to the default tabs or modify the controls used for the default attributes.

To customize properties dialogs, you use these two methods:

Method
Description
bpr-get-dialog-info

Allows you to add one or more custom tabs to the default properties dialogs and to control the organization of the attributes. This method returns a list of additional tabs to add to the dialog and lists, in order, the attribute names to display on each tab.
bpr-get-attribute-properties

Provides additional specifications for the user-interface control to use for your custom attributes. Note that ReThink uses the custom attribute specification when importing values from Excel, a CSV file, or a database.

For details, see Application Programmer's Interface.
