Customizing the Remove Block

Customizing What the Remove Block Considers "Empty"

You can customize what the Remove blocks considers "empty" by editing the following attribute on the Block tab of the properties dialog:

Attribute
Description
empty-breakpoint

Determines when the container object is considered empty and, thus, when the Remove block passes the container to the empty output path. The block looks at the contents of the item-list of the container when the container first arrives at the block. By default, if the item-list contains a single object upon arrival, it passes the input object to the empty container output path. The default is 1.
