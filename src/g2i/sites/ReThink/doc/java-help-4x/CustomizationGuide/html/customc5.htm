Topic - Customizing Resources and Work Objects

Common Customization Attributes of Resources and Work Objects

A resource is a subclass of a work object, which means they define some of the same attributes. The common G2 and ReThink attributes of resources and work objects available in Developer mode are:

Attribute
Description
gfr-uuid

The internal identification number of the object.
notes

The current status of the resource or work object.
names

The internal name of the resource or work object. ReThink uses the label attribute rather than the names attribute to identify objects and avoid naming conflicts.
reset-procedure-name

The procedure name the object uses when the simulation is reset. The default value is bpr-reset-object. .
delete-procedure-name

The procedure name the object uses when it is deleted. The default value is bpr-delete-object.

update-procedure-name

The procedure name the object uses when the Update button on the object is clicked. The default value is bpr-update-object.
start-procedure-name

Available for backward compatibility only. See note below.
stop-procedure-name

Available for backward compatibility only. See note below.
animation-subtable

Subobject that specifies the default colors the resource or work object uses when it is in an active, inactive, or error state. The default value is an instance of a bpr-object-animation-subtable.

duration-subtable

Subobject that specifies the timing and utilization parameters of the resource or work object, and computes summary timing and utilization statistics. The default value is an instance of a bpr-object-
duration-subtable.

cost-subtable

Subobject that specifies how the resource or work object computes cost statistics. The default value is an instance of a bpr-object-cost-subtable.

blocks-waiting

An item-list that contains the blocks that are waiting for the resource. A Resource Manager updates this list when it allocates resources from the same pool for multiple tasks. You can describe this attribute to see the blocks that are waiting. This attribute is not relevant for work objects. This attribute is available in Modeler mode on the General tab.

For additional information, see:

G2 Reference Manual.
Editing Subobjects.
Caution: The attributes start-procedure-name and stop-procedure-name for resources and work objects are available for backward compatibility only. When you upgrade your model to ReThink Version 3.1, you should migrate from procedures to methods because support for these named procedures will be removed in a future release of ReThink.
