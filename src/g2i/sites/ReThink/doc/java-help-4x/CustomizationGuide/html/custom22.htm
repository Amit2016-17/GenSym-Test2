Topic - Customizing Blocks

Common Customization Attributes of Blocks

The common attributes of blocks available in Developer mode are:

Attribute
Description
gfr-uuid

The internal identification number of the object.
notes

The current status of the block.
names

A symbol that represents the name of the block. ReThink uses the Label attribute rather than the names attribute to identify blocks, to avoid naming conflicts.
reset-procedure-name

The procedure name the block uses when the simulation is reset. The default value for most blocks is bpr-reset-block. See also Customizing Specific Blocks.
delete-procedure-name

The procedure name the block uses when it is deleted. The default value is bpr-delete-block.

update-procedure-name

The procedure name the block uses when the Update button is clicked. The default value is bpr-update-block.
start-procedure-name

Available for backward compatibility only. See note below.
stop-procedure-name

Available for backward compatibility only. See note below.
animation-subtable

Subobject that specifies the default colors the block uses when it is in an active, inactive, or error state. The default value is an instance of a bpr-block-animation-subtable.

duration-subtable

Subobject that specifies the timing parameters of the block, and computes summary timing statistics. The default value is an instance of a bpr-default-block-duration-subtable.

cost-subtable

Subobject that specifies how the block computes cost statistics. The default value is an instance of a bpr-block-cost-subtable.

For additional information, see:

G2 Reference Manual
Customizing the Default Behavior of Blocks or Instruments.
Editing Subobjects.
Caution: The attributes start-procedure-name and stop-procedure-name for blocks and instruments are available for backward compatibility only. When you upgrade your model to ReThink Version 3.1, you should migrate from procedures to methods because support for these named procedures will be removed in a future release of ReThink.
