Topic - Block Processing

The Stop Activity Phase

Each activity has a stop-event, which is a subobject of bpr-event, which defines the following attributes:

Attribute
Description
time

The start time of the event.
priority

The priority of the event.

To implement the behavior of any event, you implement the following method:

bpr-event-method(event: class bpr- event, scenario: class bpr-scenario,    ui-client-item: class ui-client-item)

For details, see bpr-schedule-an-event.

Note: You no longer implement the procedure-name, as in ReThink 3.1

The stop activity phase performs these tasks:

Execute the Block's Stop Method
Update Block Statistics
Stop Allocated Resources
Stop the Work Objects
Update Total Cost of Work Objects
Evaluate Instruments When Phase is Stop
Send the Work Object Downstream
Clean Up
