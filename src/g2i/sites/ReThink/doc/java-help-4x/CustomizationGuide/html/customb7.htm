Topic - Customizing Resources and Work Objects

Attributes of Duration Subtable

The attributes of the duration-subtable of a resource or work object are:

Attribute
Description
maximum-utilization

For resources, the amount of the resource that is available to be allocated. The default value is 1. This attribute is not relevant for work objects.
efficiency-factor

For resources, a number that the resource multiplies by the duration to determine the actual duration.
total-work-time

The total amount of time that the resource has been allocated or the work object has been active.
total-elapsed-time

The total amount of simulation time that a resource or work object has existed in the current simulation.
total-idle-time

The total amount of simulation time that the resource has not been allocated or the work object has been idle, which is the total-work-time minus the total-elapsed-time.
not-available-time

The amount of time that a resource is not available during the course of the simulation, due to temporal constraints on the resource. This attribute is not relevant for work objects.
creation-time

The simulation time at which the resource or work object was created.
current-utilization

For resources, the amount of the resource that is currently allocated, based on the utilization of the Resource Manager and the maximum-utilization of the resource. This attribute is not relevant for work objects.
average-utilization

For resources, the amount of time the resource is allocated compared to the amount of time it is available for allocation, which is the total-work-time divided by the total-elapsed-time. This attribute is not relevant for work objects.
reset-procedure-name

The name of the procedure the resource or work object uses to reset the duration subtable when the simulation resets. The default value is bpr-reset-object-duration-subtable.
procedure-name

The name of the procedure the resource or work object uses to compute duration and utilization. The default value is bpr-object-duration.

For general information on how to customize subobjects, see Editing Subobjects.
