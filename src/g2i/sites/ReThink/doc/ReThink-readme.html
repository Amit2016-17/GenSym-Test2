ReThink Version 4.0 Rev. 4 Readme

(March 2003)

ReThink provides a graphical modeling environment for the dynamic analysis of supply chains, based on the Supply Chain Council's SCOR reference model. You can use ReThink to:

  • Represent your supply chain in terms of the four SCOR processes-Plan, Source, Make, and Deliver-and to compute numerous SCOR metrics, based on your supply-chain model, such as inventory levels, standard asset calculations, order fulfillment response times, and operating costs.

  • Make timely, informed business decisions about the structure of your supply chain, such as the number and location of distribution sites.

  • Determine the correct service levels to offer to customers and the costs of providing that service when, for example, outsourcing finished goods.

  • Benchmark your current supply chain and compare the results of alternative supply-chain scenarios, for example, by contrasting normal and increased demand, or by juxtaposing alternative order-fulfillment strategies, using the same demand.

  • Analyze the impact on your supply chain of implementing e-business strategies.

For information about SCOR, visit http://www.supply-chain.org.

System Requirements

This section describes the minimum and recommended system requirements for hardware, operating systems, and networking capability. The requirements are less for clients running ReThink Offline.

Minimum Requirements

The minimum requirements for clients running ReThink Offline, and ReThink Online are:

  • Pentium II 400MHz with 256 Mbytes of memory

  • Windows 2000 Professional

  • Excel 2000 SR-1

  • 400 Mbytes of free disk space and CD drive

  • 1024 x 768, 256 color screen resolution

  • 14.1" TFT display for laptops or 17" monitor for desktops

  • TCP/IP protocol installed and configured

The minimum requirements for client installations only of ReThink Offline are:

  • Pentium II 400MHz with 128 Mbytes of memory

  • Windows 2000 Professional

  • Excel 2000 SR-1

  • 100 Mbytes of free disk space and CD drive

  • 1024 x 768, 256 color screen resolution

  • 14.1" TFT display for laptops or 17" monitor for desktops

  • TCP/IP protocol installed and configured

Additionally, a 10 Mbps local network connection to the server; performance may vary based on network availability.

Also supported:

  • Windows XP Professional and Windows NT 4.0 with Service Pack 6

  • Excel 97 SR-2

Recommended Requirements

  • Pentium III 850MHz with 512 Mbytes of memory

  • Windows 2000 Professional

  • Excel 2000 SR-1

  • 10 GByte hard drive and CD drive

  • 1024 x 768, 256 color screen resolution

  • 14.1" TFT display for laptops or 17" monitor for desktops

  • TCP/IP protocol installed and configured

Software Requirements

ReThink integrates with Excel to report results and chart trends; therefore, you need to install Excel 2000 SR-1 or Microsoft Excel 97 SR-2, depending on whether you are running Windows XP/2000 Professional or Windows NT 4.0 with Service Pack 6, respectively.

ReThink requires that you install Acrobat Reader 4.0 or later, which you need to read online documentation. Acrobat Reader is freely available on the Internet. Gensym does not provide any support or warranty for this product. Please contact Adobe for any issues about Acrobat Reader.

As part of the installation process, ReThink automatically installs the Java Runtime Environment Version 1.3 (JRE 1.3), which is required to run the ReThink client. ReThink uses its own installed version of JRE, so it does not conflict with any existing versions you might have installed.

Windows XP and 2000 Users

To install ReThink on Windows XP or 2000, you must have administrator privileges.

After installation, a user without administrator privileges cannot run ReThink until the file permissions on the installed files have been changed. The recommended way to do this is to have the administrator follows these steps after installation:

  1. Choose Properties on the folder that contains the ReThink software, escor_rethink, by default.

  2. Click the Security tab, then click the Advanced button.

  3. Enable the "Reset permissions on all child objects and enable propagation of inheritable permissions" option.

Enabling this option allows the user to run ReThink without giving administrator privileges. The user can now run ReThink.

New Features

The new features for ReThink Version 4.0 Rev. 4 are:

  • Uses Version 6.2 Rev. 3 of the G2 Bundle. ReThink Version 4.0 Rev. 3 included Version 6.2 Rev. 2 of the bundle.

  • Provides various new features related to the generation of repeatable random number sequences.

Repeatable Random Number Generation

You can now reproduce sequences of randomly generated numbers when using the Scenario Manager. You do this by specifying a seed value, which is a user-defined or randomly generated integer upon which a random number sequence is based. By using the same seed value for different simulations, you can ensure that every aspect of the simulation is identical, including randomly generated numbers. Previously, there was no way to run the identical simulation, because randomly generated numbers always varied with each simulation.

ReThink uses random numbers to compute mathematical distributions such as Random-Normal, Random-Exponential, and Random-Triangular. You use these random distributions to generate values for attributes such as Order Size, Order Period, Receive and Putaway Time, Total Build Time, and Order Receipt to Entry Time.

To implement this new functionality, ReThink has the following new features:

  • Additional Attributes of a Scenario

  • Additional Keywords for Scenario Manager

  • Seed Value Displayed in Reports

Additional Attributes of a Scenario

The Scenario object has two new attributes on the General tab of the dialog:

  • Seed Value - An integer that holds the seed value, which is either randomly generated or user-defined. ReThink uses this seed value as the basis for randomly generated numbers.

  • Generate New Seed - A truth-value that determines whether to use a randomly generated seed value or a user-defined seed value. The default value is true, which uses a randomly generated seed value.

ReThink sets the seed value, based on these attributes, when you reset the Scenario.

Additional Keywords for Scenario Manager

The Scenario Manager provides two new keywords to allow you to set the seed value in the Scenario Manager:

SET-SEED(<new-value>)

    Sets the Seed Value attribute of the Scenario to an integer value.

CREATE-NEW-SEED(<new-value>)

    Sets the Generate New Seed attribute of the Scenario to true or false.

These keywords are in the Scenario and Model Parameter Keywords category.

Seed Value Displayed in Reports

ReThink reports display the Seed Value used for random number generation at the top of the report so that you can keep track of which report data belongs to which seed value. This header appears in report views within ReThink, as well as in Excel reports.

Known Problems

ReThink Version 4.0 Rev. 4 has the following known problems:

  • If you are using NT 4.0 Service Pack 5 or earlier you will need to upgrade to NT 4.0 Service Pack 6, which is available from Microsoft

  • If you pause the server from the client, the user interface may hang since it cannot execute the required procedures on the server. The feature to control the server is only intended to start the server if you open an older model, based on G2 Version 5.x. If you open such a model into the new version, which is based on G2 6.0, the server will update the KB, however, you will need to start the server manually.

  • We know of at least one product, namely Visio, which conflicts with creating reports in Excel, using ReThink. These products create hidden files with an Excel file extension (.xls) in the XLStart directory in Excel. The macros of these hidden files conflict with ReThink Excel macros. In the case of the Visio macros, ReThink opens the spreadsheets, but then opens a second spreadsheet in which it displays the results. To fix this problem, remove all files from the XLStart directory and ensure that only the ReThink workbook is loaded into Excel.

  • If you are using Microsoft Office 2000, you have to change the security level within Excel from High to Medium or Low. Otherwise, Excel will never prompt you to Enable Macros. To do this, choose Tools Macro Security. The default ReThink report defines a macro that format the report in Excel; thus, you must enable macros when you first open the default report or any report that is based on this default.

  • On certain installations, when you exit Excel with an ReThink spreadsheet loaded, Excel might not shut down properly, due to the Visual Basic code and ActiveX control that is loaded. The error happens when you are running Windows NT 4.0 with Office 97-SR-2; however, Excel shuts down properly if you are running Windows NT 2000 with Excel 2000 SR-1 or Windows NT 4.0 with Excel 2000 SR-1.

  • The Model Browser tree does not refresh correctly when connecting to a server after the client has been launched. When connecting to a server at startup, the Model Browser works fine. The Model Browser does not automatically refresh when renaming objects, adding objects to the model, or deleting objects from the model. To refresh the Model Browser display, collapse and expand the tree.

  • When using ReThink for database access with MS Access or SQL 2000, the prefix for bind variables is ? and not : as in Oracle. In addition, you must not add the variable name after the bind variable, whereas in Oracle you do. If you specify that the bind variable prefix as ? on a network connection pool:

    • You must write your SQL statement prefixing bind variables with ? and add the variable name, which is a G2 attribute. For example:

        SELECT lastname, firstname from employees where employeeid = ?employeeid

    • The internal code parses the SQL statement and reformats it by removing the variable name after the ? to what's expected by SQL2000 before sending it to the database. In the example above, it is reformatted to:

        SELECT lastname, firstname from employees where employeeid = ?

    • It uses the variable name after the ? (employeeid in the example) to fetch the value from an attribute of the work object named by this variable.

  • If you configure the Expression To Display of a readout table and apply the change before configuring the Label To Display, the label never appears. Instead, the readout table shows the Expression To Display in what should be the label section of the table. When you then try to enter a label, an error occurs.

  • If a dialog is currently showing and you display a menu from the menu bar such that it overlaps the dialog, the menu appears behind the dialog, instead of in front of it. You must move or close the dialog so that it no longer obstructs the menu.

Starting ReThink

ReThink is a client/server application. In general, to start ReThink, you can simply start the client, then respond to the prompt to automatically start and connect to the server running on the local host computer at port 1111. The installation program creates shortcuts in the ReThink program group on the Start menu for starting both the server and the client.

By default, when you start ReThink, ReThink opens the default model into the server, which is escor.kb. The next time you start ReThink, ReThink opens the last saved model.

You can open a different model at startup by creating and modifying a shortcut to the ReThink Server. To start ReThink with a different model opened, first, you start the server, then you start the client.

The server runs as a hidden process, which you can view on the Processes tab of the Task Manager by pressing Ctrl + Alt + Del and clicking the Task Manager button. The server process appears in the list as g2.exe.

To open the default ReThink model:

    -> Choose Start > Programs > Gensym e-SCOR & ReThink > ReThink Client, then click Yes to start and connect to the ReThink server on the local machine at port 1111.

The splash screen provides status updates as the server is launching, which can take more than a minute, depending on the speed of your computer.

To start ReThink with a different model opened in the server:

  1. Create a shortcut to the Server.bat file located in the bin directory of your e-SCOR product directory.

  2. Display the properties dialog for the shortcut, click the Shortcut tab, and add the default KB file to open, following the path name in the Target field, for example:

      "C:\Program Files\Gensym\escor_rethink\bin\Server.bat"
      escor-demos.kb

  3. Double-click the shortcut to start the server from the new batch file.

  4. Choose ReThink Client from the Start menu to start the client, which automatically connects to the server.

Exiting ReThink

When you exit the client, ReThink prompts you to exit the server as well. In general, you should exit the server as well as the client. If you do not exit the server when you exit the client, logging out of your Windows session exits the server automatically.

Caution: Be sure to save your application before you exit ReThink.

To exit ReThink:

  1. In the client, choose File > Exit.

  2. In the confirmation dialog that appears, click Yes to exit the server as well.

If you do not exit the server at the same time, you must exit the server manually.

To exit the ReThink server:

  1. Choose File > Exit.

  2. In the confirmation dialog that appears, click the Shutdown button.
