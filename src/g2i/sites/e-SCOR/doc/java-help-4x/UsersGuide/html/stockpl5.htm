Understanding How the Role Computes Orders Size for Replenishment Planning

e-SCOR computes the size of a replenishment or build order by comparing these values:

The minimum number of products the role can reorder at one time (Minimum Reorder Quantity).
The desired inventory level (Replenishment Threshold) and current inventory (Inventory Level), given current orders (On Order and Awaiting Order Total).

The role orders the maximum of these two values, unless the current inventory level can cover current orders, in which case, it orders nothing.

If the size of the order exceeds the Maximum Threshold, the role adds the excess amount to the next order.
