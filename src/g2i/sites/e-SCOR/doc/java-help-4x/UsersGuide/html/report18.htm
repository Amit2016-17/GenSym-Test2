Adding Attributes to Reports through the Dialogs

You can add attributes of any object to a report, directly through the object's properties dialog. This approach can be easier than configuring the list of visible attributes when you know exactly which attributes you want to add. Each parameter and metric on each dialog has an associated button to the right of the attribute. Clicking the button shows a list of reports that define the attribute and a list of available reports to which you can add the attribute. You can add attributes to one or more reports.

For example, you might want to create a Role Parameters report and a Role Metrics report with a subset of parameters to configure and metrics to view, respectively. To do this, first you would delete all the existing attributes in the report, then you would add parameters and metrics to each report, as needed, by selecting the attributes to add through the dialogs.

To add attributes to reports through the dialogs:

1 Create a report object and delete all the existing attributes.
For details, see Configuring the List of Visible Attributes.
Note: If you are creating reports in Excel, do not delete the UUID attribute.
2 Display the properties dialog for an object in the model whose attribute you want to add to a report.
3 Click the button next to the attribute you want to add.
e-SCOR displays a dialog that shows the list of reports that currently contain the attribute and a list of applicable reports to which you can add the attribute. The lists include only those reports associated with the model and only those reports that are relevant for the particular attribute.
Note: You can add parameters to both input and output reports; however, you can add metrics to output reports only.
4 Select one or more reports from the list of available reports in the right column, then click the left arrow or double-click the report to move the reports to the list of reports containing the attribute in the left column.

The attribute appears at the end of the list of visible attributes for the report. When you create the report template, the report contains the attribute you added.
