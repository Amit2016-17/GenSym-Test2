Connecting to the Server

By default, you start e-SCOR by starting the client, then responding to the prompt to start and connect to the server. If you have licensed e-SCOR Online, both the server and the client must be running on the local host at port 1111. If you have licensed e-SCOR Online Team, you can install the server on a remote host and access it via a TCP/IP LAN.

With an e-SCOR Online license, multiple users can connect to the server simultaneously; otherwise, only one user can connect to the server at a time.

When using e-SCOR Online Team, e-SCOR keeps an audit trail of clients connecting to the server and stores this information in escor_rethink*.log files. To comply with your license agreement and for audit purposes, you must keep these log files. The active log file is always called escor_rethink*.log. You can remove the other log files to save disk space, as long as you keep a backup of these files for audit purposes.

You can:

View the connection status.
Disconnect from and connect to the server from the client.
Connect to a specific server at startup.
