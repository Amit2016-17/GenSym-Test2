1 Determine order type.
If the Order Type parameter of the Mb: Make Base Manufacturer category is stock or made-to-order, the Mb category of a Base Manufacturer role receives build orders from the Make Planning process, which triggers the next step.
For detail on make planning, see Make Planning Process.
2 Determine batches.
2.1 The Mb category determines the number of batches it needs to fill the build order by looking at the Maximum Batch Size and Minimum Batch Size parameters of the role's delivery product. It does this by breaking up the build order into separate batches, each of which is no larger than the Maximum Batch Size and no smaller than the Minimum Batch Size, which creates no more than one partial batch.
If additional build orders arrive during this step, the Mb category fills any existing partial batches from the new build order first, before it determines new batches. For example, an additional build order might arrive, which was created to compensate for build yield. This new build order might cause the Mb category to fill an existing partial batch.
2.2 Because a Base Manufacturer role does not order source products to manufacture its delivery products, it determines its cost of raw materials from the Purchase Cost parameter, which you configure on the Source tab of the Base Manufacturer's delivery product. For a Base Manufacturer role only, the Mb category computes the cost of manufacturing the build order by multiplying the value of the Order Size of the build order by the Purchase Cost parameter to determine the cost of the build order.
2.3 The Mb category generates a chit, which triggers the build-order selection process in the next step.
3 Select build order.
Build-order selection triggers when the Mb category receives a chit and when a manufacturing resource becomes available. If no resource is available, the Mb category waits until one becomes available before selecting build orders. This means the model can trigger build-order selection asynchronously.
If the Mb category has multiple build orders competing for resources, it sorts build orders for delivery products on a random basis. For each delivery product, it chooses complete batches before partial batches. It only selects partial batches when it has no complete batches from which to choose.
4 Manufacture products.
4.1 The Mb category increments the Build Orders Started metric by one, then it manufactures its selected build order by creating the number of finished products in the build order that it determined in Step 2, Determine batches..
4.2 The Mb category waits the amount of time indicated by the Order Release to Build Time parameter.
4.3 For each batch, the Mb category manufactures the build order, which takes the amount of time indicated by the Total Build Time parameter. The Mb category takes the same amount of time to manufacture a partial batch as it takes to manufacture a complete batch.
If you configured the Total Build Time multiplier parameter of the delivery product to model differences in build times for different products, the Mb category adjusts the value of the Total Build Time parameter by the multiplier.
4.4 If you configured the Yield parameter of the delivery product to model defective production in the manufacturing process, the Mb category adjusts the number of products it makes by the yield; the rest is considered defective and discarded. The Mb category then creates new build orders for the amount of defective product and sends the build order back to Step 2, Determine batches.. The Mb category treats this build order the same way it treats any build order.
The Mb category then computes the Total Products Rejected metric, based on the Yield parameter.
4.5 The Mb category increments the Build Orders Completed metric by one. Finally, it computes the Make Cycle Time (Days) metric to determine the performance of the manufacturing process.
5 Stage products.
When the Order Type is stock or made-to-order, the Mb category effectively places the finished products in inventory. It does this by incrementing the Inventory Level and Potential Inventory Level metrics of the delivery product by the number of products manufactured and updating the Inventory Status metric to reflect the current inventory status.
6 Generate and send payment.
The Mb category generates a payment for its raw materials, which is based on the Purchase Cost parameter of its delivery product and the number of batches it built. You configure the Purchase Cost on the Source tab for the Base Manufacturer role only.
The Mb category waits the amount of time indicated by the Financial Payment Terms parameter of the buyer role, then computes the Financial Payments Total metric for the role.
7 Release products.
The Mb category releases the product, which takes the amount of time indicated by the Move to Delivery parameter.
The Mb category computes the Total Products Accepted metrics of the delivery product, based on the Yield parameter of the delivery product.
The Mb category generates a chit, which it sends to the downstream Deliver category for delivery. The chit initiates the order-processing and fulfillment process, described in Order/Product Process.
Accumulate payments.
As an independent process, each Financial Period parameter, the Base Manufacturer role increments the Financial Payments Period metric for the role.
