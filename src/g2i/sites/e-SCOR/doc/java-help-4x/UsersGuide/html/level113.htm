Determining the Number of Roles to Use

Once you have determined the level of detail in the product hierarchy and configured the product hierarchy, you must determine the number of roles to use.

The most straightforward way to model your supply chain is to create separate roles for each product specification in the product hierarchy. This approach allows you to use a different financial model and to track cost, asset, and financial metrics for each product composite separately. It also allows you to use a different planning mode-pull or push-for each source and delivery product. However, this approach also means your model contains more roles, which means you must configure more Level 2 category parameters.

Often, your supply chain contains products that behave similarly at the role level. For example, a role that delivers multiple products might use the same financial model for all its products, keep track of cost, asset, and financial metrics for all its products together, and use the same planning mode for sourcing and delivering products. In this case, your supply chain can have fewer roles, where the same role sources, makes, and/or delivers multiple products.

To determine whether to create separate roles for each product specification in the product hierarchy, you must ask these three questions:

Q Does the role use a different financial model to source and deliver its products?

Q Do you need to measure cost, asset, and financial metrics individually for each product that a role sources and delivers?

Q Does the role use a different planning mode-pull or push-for its source and delivery products?

If you answer yes to any of these questions, then you must create separate roles for each product whose financial model, cost metrics, asset metrics, financial metrics, and/or planning mode are distinct. If you answer no to all of these questions, then you can use a single role to source, make, and/or deliver multiple products.

If you use a single role to source, make, and/or deliver multiple products, you can add categories to the role detail, as needed. Creating separate categories for each product lets you configure Level 2 SCOR parameters and track Level 2 SCOR metrics separately for each product, while still using the same role.
