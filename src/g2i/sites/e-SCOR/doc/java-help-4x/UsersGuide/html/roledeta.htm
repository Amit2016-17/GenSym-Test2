Configuring Role Details for Multiple Products

By default, each role detail contains a single Source, Make, and/or Deliver category, depending on the role, which sources, makes, and/or delivers stock products. This means that if the role sources, makes, and/or delivers multiple products:

Each source product uses the same Source category.
Each delivery product uses the same Make and/or Deliver category, depending on the role.

Therefore, if a role sources, makes, and/or delivers multiple products, each product takes the same amount of time to source, make, and deliver, respectively, has the same associated costs, and uses the same modeling parameters.

If the role sources, makes, and/or delivers multiple products, you can use different Source, Make, and/or Deliver categories for each source and/or delivery product, depending on the role. To do this, you add multiple categories of the same type to the role detail.

You add categories to the role detail when products behave similarly at the role level but differently at Level 2 SCOR. For example, different products might require different values for the Receive and Putaway Time parameter of a Source category, the Production Material Handling cost parameter of a Make category, or the Order Selection parameters of a Deliver category.

By creating separate categories for each source and/or delivery product, you can track performance, financial, and cost incurred metrics separately for each product associated with each category.

When adding categories to the role detail, you must create one category for each product composite; you cannot use the same category for some product composites and separate categories for others.

You connect multiple categories of the same type through the category routers on the role detail.

To configure the role detail for multiple products, you can:

Add multiple Make and Deliver categories to the role detail of a supplier role.
Add multiple Source categories to the role detail of a buyer role.
Model seasonal demand for consumer products.

You also add categories to the role detail to use alternative planning strategies-make-to-order or engineer-to-order, as described in Using Alternative Planning Strategies.
