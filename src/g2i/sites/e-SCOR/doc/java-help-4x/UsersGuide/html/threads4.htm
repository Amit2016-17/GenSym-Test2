Topic - What Happens When the Simulation Runs

How Processes Interacts

When you run the simulation for a model, e-SCOR performs these high-level processes, which interact with each other as this figure shows:

This diagram assumes the model does not use contracts.

As the diagram illustrates, each process initiates, based on some triggering event or some other process, and certain processes trigger other process. Here is a high-level description of each process in terms of the triggering event, the high-level task it performs, and the process it triggers, where relevant:

Role Verification process
- Initiates when the simulation starts and takes 1 second.
- Determines available suppliers for source products.
- Coordinates pull and push planning.
Order/Product process
- Initiates based on customer orders from Consumer role or replenishment orders from Source Planning process.
- Sends orders to suppliers for source products.
- Initiates order selection, based on customer or replenishment orders, or based on the Build process.
- Processes and fulfills those orders by sending product shipments to buyers.
- Triggers the Financial process when products get shipped.
Build process
- Initiates based on a cyclical Make Planning process or by receiving source products, based on Order/Product process.
- Processes build orders and manufactures finished products.
- Triggers Order/Product process when finished product is released.
Financial process
- Initiates based on customer orders from Consumer role or replenishment orders from Source Planning process.
- Performs accounting of financial collections and payments, both ongoing and per Financial Period.
- Updates asset metrics, based on costs and inventory once every Financial Period.
Make Planning process
- Initiates on a regular planning cycle.
- Plan when Base Manufacturer and Manufacture roles manufacture delivery products.
- Triggers the Build process.
Source Planning process
- Initiates on a regular planning cycle.
- Plans when Distributor and Manufacturer roles create replenishment orders for source products.
- Triggers the Order/Product process.
Contracts process
- Initiates based on Source Planning process.
- Chooses suppliers and determines order size, based on contracts.
- Can split the order among multiple suppliers.
