Importing Input Report Data from Excel CSV Files

You can import data into the model from a .csv file rather than from an Excel report.

To import data into the model from an Excel CSV file:

1 Display the properties dialog for the input report object whose data you want to import from a CSV file and click the Excel tab.
2 Enable the Excel CSV File Reporting Enabled option.
3 Configure the Excel CSV File Name to be the name of the .csv file from which to import the data.
4 Configure the attributes to appear in the report.
For details, see Configuring the Attributes to Appear in a Report.
5 Configure the time unit, as needed.
For details, see Configuring the Time Unit.
6 Create the input report template, which also creates a template for the CSV file.
For details, see Creating a Report Template in Excel.
The template contains a header row that identifies each input parameter to configure and rows for each object in the report. Each row contains default values for each parameter in the report.
The next time you create or update the report template, e-SCOR leaves the CSV file in tact, if it already exists.
Tip: To create a new CSV file, you must first delete the existing CSV file by choosing Delete CSV Report File on the report object.
7 Open the CSV file in Excel, edit the input report data, and save the data in CSV file format.
Note: Do not save the file in Excel format; otherwise, e-SCOR cannot read the data. Also, do not edit the first row of the report or the object labels that identify each row of the report.
Alternatively, you can open the CSV file in an editor, such as Notepad, and save the data as a text file.
8 Click the Import Data from File button in the dialog or choose Import Data from File on the report object.

Importing data from the CSV file automatically applies the data to the model and updates the associated Excel report.
