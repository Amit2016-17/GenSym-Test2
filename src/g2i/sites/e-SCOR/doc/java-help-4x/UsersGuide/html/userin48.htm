Configuring Objects Through Dialogs

e-SCOR provides two types of dialogs for configuring objects: built-in dialogs and automatically generated dialogs. Most objects use built-in dialogs, including roles, categories, product specifications, resources, and reports. Certain objects use automatically generated dialogs, for example, class definitions, rules, objects on the New Item palette, and database interface objects, which you use to customize e-SCOR and to connect with external databases.

Configuring Objects Through Automatically Generated Dialogs

When configuring objects through automatically generated dialogs, you configure certain attributes through text boxes and check boxes, and you configure other attributes through the Text Editor. The Text Editor provides prompts that guide you through the required syntax.

This figure shows an example of an automatically generated dialog for a class definition:

To configure an object through automatically generated dialogs:

1 Display the properties dialog for the object.
See Displaying the Properties Dialog for Objects.
Automatically generated dialogs have two types of attributes: those with text boxes and those that use the Text Editor.
2 To configure attributes that do not have a particular syntax, type the value in the text box, then press return to apply the value to the model.
For example, the Class Name of a class definition does not require any syntax, so you enter the value directly in the text box.
3 To configure attributes that require syntax, click the button next to the text box:
For example, the Direct Superior Classes of a class definition requires special syntax, so you must use the Text Editor.
4 Enter values in the text area by typing the text, by clicking the prompts at the bottom of the screen, or by entering Ctrl + Space to complete the text.
