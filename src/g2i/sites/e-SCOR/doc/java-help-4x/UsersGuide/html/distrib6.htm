Topic - Modeling a Distribution Process

Configuring Order Selection Parameters

You can control how a Distributor role sorts and selects orders for delivery, based on a set of order-selection criteria. You do this to describe how your current distribution process works or to experiment with alternative techniques to enhance the performance of the delivery phase of your distribution process.

For example, if two product shipments are waiting to be delivered, you can configure the Deliver category to deliver the highest cost orders first, the oldest orders first (FIFO), orders that are within a certain size of the maximum or minimum order first, or orders "just in time" to meet the buyer's desired order fulfillment lead time.

Note: The Order Selection parameters are relevant for the D1: Deliver Stocked Product category only, which appears in the detail of a Base Manufacturer, Distributor, and Manufacturer role.

e-SCOR uses an exclusion strategy to identify the orders to fill. The Deliver category places all orders in the Orders pool into a list, then applies the first set of sort criteria to the orders in the list. If an order does not meet the sort criteria, the Deliver category removes the order from the list. If two orders have the same value for the set of sort criteria, the Deliver category applies the second set of criteria, and so on. You can configure the Deliver category to sort orders, based on up to four sets of sort criteria.

Each set of sort criteria specifies the direction of the sort (smallest or biggest) and cutoff parameters, which are relative to the selected property of an order. You can sort orders, based on numeric and time-based property values of an order. For example, if the property to sort by is the time at which the order was placed and the sort direction is smallest, the Deliver category sorts orders, based on a first-in, first-out (FIFO) basis.

You can also configure the Deliver category to use a cutoff value, which lets you further restrict the orders that are candidates for delivery. For example, if you choose to deliver the biggest orders first, based on orders size, small orders will accumulate in the Orders pool. To avoid this situation, you can configure a maximum delta acceptable cutoff to ensure that small orders also get delivered, but only after big orders.

To sort orders for delivery, you configure the following order selection parameters for the Deliver category:

Parameter
Description

Sort Direction

Determines whether to sort orders, based on the Smallest or Biggest value. You can also sort orders, based on a Random value, which chooses an order at random, based on the specified criteria. The default value is Smallest.

For examples, see Sorting Orders with No Cutoff.

Sort By

Determines the value the Deliver category uses to sort orders. The options include numeric and time-based properties of an order. The default value is Internal Order Number.

The numeric options are: Cost, Internal Order Number, Product Preference, Customer Preference, Order Lead Time, Order Placed Time, and Order Size.

The time-based options are: Order Lead Time, and Order Placed Time.

For examples, see Sorting Orders with No Cutoff.

Cutoff

Determines whether the Deliver category includes all orders in its sort (None) or whether it includes only those orders that meet an acceptable cutoff.

If Sort By is a numeric value, the options for Cutoff are: Minimum Acceptable, Maximum Acceptable, and Maximum Delta Acceptable.

If Sort By is a time-based value, the options are: Time Delta Acceptable and Current Time Delta Acceptable.

If Cutoff is Maximum Delta Acceptable or Time Delta Acceptable, Sort Direction must be Smallest or Biggest.

If Cutoff is Current Time Delta Acceptable, Sort Direction must be Random and Sort By must be Order Lead Time.

For examples, see Restricting the Orders to Sort Based on a Numeric Cutoff and Restricting the Orders to Sort Based on a Time Cutoff.

Cutoff Value

When Cutoff is a value other than None and when Sort By is a numeric value, specifies the value that determines whether an order is excluded from the sort. If the specified value of an order does not meet the cutoff criteria, the Deliver category does not fill the order.

For examples, see Restricting the Orders to Sort Based on a Numeric Cutoff.

Cutoff Duration

When Cutoff is a value other than None and Sort By is a time-based value, specifies the value that determines whether an order is excluded from the sort. If the specified time of an order does not meet the cutoff criteria, the Deliver category does not fill the order.

For examples, see Restricting the Orders to Sort Based on a Time Cutoff.

To configure order selection criteria:

1 Display the properties dialog for the D1: Deliver Stocked Product category and click the Order tab.
The parameters to configure appear within the Order Selection group. The Order Selection group has four tab pages, one for each set of order selection criteria.
2 Configure the parameters in the table above on the First tab to configure the basic parameters that the Deliver category uses to sort orders.
3 Click the Second tab to configure a second set of parameters, which the Deliver category applies after the first set of criteria.
4 Repeat Step 3 for the Third and Fourth tab, as needed, to configure up to four sets of criteria, which the Deliver category applies after the previous sets of criteria.
