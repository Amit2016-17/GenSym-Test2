Configuring Level 2 SCOR for the Basic Model

Each role in the model contains detail, which is where you configure the behavior of the model at Level 2 SCOR. To configure Level 2 SCOR, you configure:

Category parameters on the role detail.
Product composite parameters.

For the Basic Model, you can configure parameters for these objects for each role:

Consumer role:
- The Order Size and Order Period of the Sc: Source Consumer category, which determine initial order demand for the overall supply chain.
- The Desired Turnaround of the source product, which determines the desired amount of time from when the Consumer role places an order to when it receives the shipment.
Base Manufacturer role:
- The Total Build Time of the Mb: Make Base Manufacturer category, which determines the time it takes to manufacture a batch of products.
- The Planning Period of the P3: Plan Make category, which determines the frequency with which the role computes build orders for planning.
- The Order Shipment to Receipt Time of the D1: Deliver Stocked Product category, which determines the amount of time it takes to deliver a shipment.
- The Maximum and Minimum Batch Size of the delivery product, which determines the number of products in each manufacturing batch.
- The Net Selling Price parameter of the delivery product, which determines the price the Consumer role pays for products and allows the model to compute financial and asset metrics for roles.
- The Replenishment Threshold and Minimum Reorder Quantity parameters of the delivery product, which determine the inventory level the role attempts to maintain and the reorder size for planning.

To configure Level 2 SCOR parameters for the basic model:

1 Show the detail of the Consumer role.
The Consumer role detail contains the ES: Enable Source category and the Sc: Source Consumer category. It also contains a number of repositories, which the model uses to store objects it creates when the simulation runs, such as orders. The Products pool contains the product composites the model creates when you assign a product specification to a role.
This figure shows the detail for the Consumer role:

2 Display the properties dialog for the Sc: Source Consumer category, click the Order tab, and configure the Order Size and Order Period parameters.
By default, you configure the Min, Mode, and Max of a triangular distribution to generate random values. You can also configure the Mode Type to use a different mathematical distribution for generating random values.
3 Choose Go To Products Detail on the Consumer role to show the Products pool.
The Products pool contains the source product associated with the Consumer role, as follows:

You can also show the detail of the Products pool on the Consumer role detail.
4 Show the properties dialog for the source product, click the Source tab, and configure the Desired Turnaround parameter to be 5 days.
You configure parameters on different tabs, depending on whether the product composite is a source product or a delivery product.
5 Show the detail of the Base Manufacturer role.
The Base Manufacturer role detail contains the Mb: Make Base Manufacturer category, the P3: Plan Make category, the D1: Deliver Stocked Product category, and the ED: Enable Delivery category. It also contains a number of repositories, similar to the Consumer role.

6 Display the properties dialog for the Mb category, click the Build tab, and configure the Total Build Time parameters to be 7, 8, and 9 hours, respectively.
7 Configure the Planning Period parameter of the P3 category to be 2 weeks.
8 Configure the Order Shipment to Receipt Time parameter on the Transportation tab of the D1 category to be 3, 4, and 5 days, respectively.
9 Display the Products pool detail for the Base Manufacturer role, then:
a Click the Manufacturer tab and configure the Maximum and Minimum Batch Size parameter to be 50 and 25, respectively.
b Click the Delivery tab and configure the Net Selling Price parameter to be 100.
c Click the Planning tab and configure the Replenishment Threshold parameter to be 100 and the Minimum Reorder Quantity parameter to be 50.
10 Run the simulation again and view the various metrics in the Metrics toolbar.

Notice that now the model computes Financial and Asset metrics for each role, based on the Net Selling Price parameter of the Base Manufacturer role's delivery product.
