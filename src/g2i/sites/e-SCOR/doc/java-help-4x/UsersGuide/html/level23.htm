Topic - Configuring Level 2 SCOR

Standard Objects on Role Details

Each role detail contains process categories appropriate to the role. In addition, each role detail has one or more category routers, which appear on the detail and send objects to appropriate categories. Each category router label indicates whether it is for a Source (S), Make (M), or Deliver (D) category, and whether it is for an upstream (U) or downstream (D) path. For example, the category router on the Base Manufacturer role detail labeled MU Router indicates it is for the upstream path of a Make category.

When adding multiple categories to the role detail, you must connect the paths through these category routers.

Each role detail also contains a number of pools in which e-SCOR stores a variety of objects. The role detail contains two types of pools:

Pools that store permanent objects, which e-SCOR creates when you assign product specifications and resources to a role. For example, the Products pool contains source and delivery products associated with each role, and the Manufacturers pool contains manufacturing resources required to make finished products.
Pools that store transient objects, which e-SCOR creates and deletes when the simulation runs. For example, the Orders pool stores orders that buyers create when sourcing products from suppliers.

Finally, each role detail contains objects labeled "LM," which are "lock managers" and which the model requires to run. You do not need to be concerned with these objects.
