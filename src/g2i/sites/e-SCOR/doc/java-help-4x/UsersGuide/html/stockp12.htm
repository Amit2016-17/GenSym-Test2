Configuring Buyer Roles to Use Forecast Planning

When the Plan Type of a role's Plan category is Forecast Customer, the role uses forecast planning to compute the size of replenishment orders for source products or the size of build orders for delivery products, depending on the role.

To use forecast planning, you configure contract parameters for the downstream buyer role. First, you configure the following contract parameters for the buyer role's Source category:

Parameter
Description

Use Contracts

Set to yes when you want to use contracts to determine order size, based on forecasts.

Initial Contract Delay

When contracts are in effect, the time from the start of the simulation until the buyer sends its first purchase request to upstream suppliers for source products.

Contract Duration

The length of time during which the contract is valid.

Contract Response Time

When contracts are in effect, the number of days a supplier has to respond to a purchase request with a purchase response.

Then, you configure the following contract parameter for the buyer role's source product:

Parameter
Description

Estimated Contract
Amount

The estimated number of products the role plans to order over the life of the contract.

To guarantee the buyer always has a valid contract for its source products, use this formula when configuring these parameters:

Initial Contract Delay + Contract Response Time > Initial Order Delay Time

Similar to a stock/replenishment planning strategy, a stock/forecast planning strategy determines its maximum inventory, based on the source or delivery product's Maximum Threshold. This means the size of the order never causes the Inventory Level to exceed the Maximum Threshold. If the role requires additional inventory such that it exceeds this maximum, it creates a back order for the excess and adds it to the order for the next planning cycle.

You must configure the Maximum Threshold parameter to be less than the Inventory Maximum parameter.

To configure the model to use a stock/forecast planning strategy:

1 Show the detail of each Distributor, Manufacturer, or Consumer role that is downstream of the supplier role that you configured to use a stock/forecast planning strategy.
For details, see Configuring the Stock Planning Strategy.
2 Configure the contract parameters for the buyer role's Source category, as described in the first table above.
3 Show the detail of the Products pool for each buyer role to show its source products.
4 Configure the Estimated Contract Amount parameter, described in the second table above, for the source product of the buyer role.
