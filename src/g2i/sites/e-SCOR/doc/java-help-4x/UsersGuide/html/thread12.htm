Topic - What Happens When the Simulation Runs

Financial Process

Buyer and supplier roles compute financial, cost, and asset metrics while the simulation runs, as follows:

Financial metrics include buyer payments and obligations, and supplier bookings, outstanding, and collections.
Cost metrics include the total cost of doing business for a role.
Asset metrics include asset turns, cash-to-cash cycle time, and inventory days of supply for the role.

The model computes financial metrics, based on the cost of buying source products and revenue from selling delivery products. The financial metrics do not include costs incurred metrics, which represent the cost of doing business.

The model computes Costs Incurred metrics, based on Cost per Transaction parameters, which you configure for various categories on the role detail. For example, for a Deliver category, you configure the Order Entry, Order Fulfillment, and Distribution costs, among others, and for the ES: Enable Source category, you configure the Create Customer Order and Supply Chain Finance costs.

The model computes asset metrics, based on inventory levels, inventory flow, and inventory value, over time.

The Financial process is responsible for computing outgoing and incoming financial metrics for categories and roles, total cost metrics for roles, and asset metrics for roles. The Financial process is not responsible for computing cost incurred metrics for categories, and it is not responsible for computing financial metrics for the Base Manufacturer role. These costs are described in the process of which they are a part.

The three configurations of the Financial process are:

Basic Financial process - Suppliers send invoices to buyers for goods ordered, and buyers send payments to suppliers within the Financial Payment Terms parameter of the buyer. This process represents payment after delivery.
Financial process for immediate payments - Buyers place orders for goods and send payments simultaneously with placing the order; the Financial Payment Terms parameter of the buyer is 0. This process represents payment upon ordering.
Financial Periods process - On a periodic basis, supplier roles accumulate financial collections, total cost, and asset metrics, and buyer roles accumulate financial payments, total cost, and asset metrics.
