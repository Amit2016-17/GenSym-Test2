Opening Models

By default, when you start e-SCOR, it automatically opens the last model that was open when you shut down the server. To open a different model, specify the file name associated with the top-level module in the module hierarchy.

By default, the top-level .kb file to open is escor.kb. Typically, you save your model to a unique file name; therefore, once you have saved your model, you open your specific file.

Note: Opening a new model replaces the existing model in memory. Therefore, before you open a new model, be sure to save the existing model; e-SCOR does not prompt you before replacing the existing model.

By default, when opening a model, e-SCOR automatically resolves conflicts and does not bring formats up to date. For information about modifying the options, see Merging Modules.

To open a model:

1 Choose File > Open, enter Ctrl + O, or click the standard toolbar button to display the Open Project dialog.
2 Enter or choose the .kb file to open and click Open.

e-SCOR displays a progress dialog as it loads the .kb file and all required modules in the server. When all modules have been successfully loaded, the display in the client updates. You must wait until the KB has finished loading in the server before you can display a workspace.
