Configuring the Planning Mode for Roles

In general, roles have a source phase, in which a downstream buyer sources products from an upstream supplier, and a delivery phase, in which an upstream supplier delivers products to a downstream buyer. The exception is a Base Manufacturer role, which has just a delivery phase, and a Consumer role, which has just a source phase.

Each role can use a different planning mode for sourcing and delivering its products, as follows:

Pull planning mode - A buyer sources products from suppliers by placing orders, and a supplier delivers products to buyers, based on orders.
Push planning mode - A supplier pushes a fixed number of products onto buyers, and a buyer simply receives those products from suppliers when the supplier pushes them.

You specify the planning mode of a role by configuring these parameters for the role:

Parameter
Description

Source Type

Determines whether the role places orders for its source products with suppliers (Pull) or whether it simply receives sources products when a supplier pushes them downstream (Push).

Delivery Type

Determines whether the role delivers products to buyers, based on orders (Pull) or whether it pushes a fixed number of delivery products onto downstream buyers (Push).

The Source Type of a Base Manufacturer role is grayed out, because a Base Manufacturer role does not source products. Similarly, the Delivery Type of a Consumer role is grayed out, because a Consumer role does not deliver products.

When configuring the planning mode for each role in the model, you must coordinate pull and push planning appropriately across roles so that the Source Type and Delivery Type are compatible.

By default, all roles use pull mode for sourcing and delivering products; therefore, by default, a downstream buyer role controls the amount of product that an upstream supplier role delivers, by placing orders. Individual roles can also specify a different Source Type and Delivery Type. For example, a Manufacturer role might obtain components from upstream supplier roles by using pull mode and deliver finished products to downstream buyer roles by using push mode. Typically, you model this by configuring the Delivery Type of the upstream role to correspond to the Source Type of the downstream role.

Note: The Source Type of a Base Manufacturer role is always pull. If a role has multiple upstream suppliers, the Delivery Type of each supplier must be the same.

When you first run a simulation, e-SCOR verifies that the model uses valid combinations of pull and push across roles.

Caution: If you specify an incompatible combination of Source Type and Delivery Type, the model generates a warning when you run the model.

To configure the source and delivery type of a role:

Display the properties dialog for a role, click the Roles tab, and configure the Source Type and Delivery Type to be either pull or push, according to the table above.
