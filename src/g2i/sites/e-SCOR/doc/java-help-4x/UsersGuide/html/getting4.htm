Configuring the Basic Model

To configure the Basic Model, you:

Connect the roles together, using the stubs provided.
Assign a product specification to each role.
Assign a resource to the Base Manufacturer role.

To configure the basic model:

1 Drag the magenta stub from the Base Manufacture role into the magenta stub on the Consumer role, then click to connect the stubs to form a path.
2 Now, connect the blue paths together to form a path.
3 To ensure the roles are connected, move one of the roles slightly on the workspace.
If the roles are connected, the path should adjust to form a straight line and each path should have an arrow indicating the direction in which objects flow in the model. If the paths are not connected properly, the path does not adjust.
The roles should look like this when the paths are connected:

4 Right-click the Consumer role to display its popup menu and choose Properties to display the properties dialog for the role.
Shortcut: To display the properties dialog for any item on a workspace, you can double-click the item, select the item and press the F4 key, or click the right mouse button on the item to display its popup menu, then choose Properties. You can also select the item, then choose Properties from the Edit menu or click the Properties toolbar button: 
The properties dialog for a role contains parameters that you configure and metrics that the model computes, including financial, asset, and cost metrics. The properties dialog is also where you configure the products and resources associated with the role.
5 Click the Products tab to display a tree representation of the product structure associated with the model.
The Basic Model contains a single product, named Product.
6 Select the Product in the tree view and accept the dialog.
The dialog should look like this:

The Consumer role is now configured to place orders for Products. By default, it places orders for 10 units once every three days.
7 Assign the Product to Base Manufacturer role, using the same technique.
The Base Manufacturer role is now configured to manufacture and deliver Products. By default, it manufactures products in batches of 100 units; when inventory falls below 500 units, it creates build orders for at least 100 units; and it determines how much it needs to manufacture once a week on a regular planning cycle.
8 Click the Resources tab on the Base Manufacturer role's properties dialog.
You configure the resource the role uses in manufacturing on this tab.
9 Configure the Resource parameter by choosing Base Mfg. from the list and accept the dialog.

The dialog should look like this:

The Basic Model is now fully configured. You do not need to configure any Level 2 SCOR parameters to run the simulation. The model uses default values for all Level 2 SCOR parameters for process categories on the role details and for the source and delivery products associated with each role.
