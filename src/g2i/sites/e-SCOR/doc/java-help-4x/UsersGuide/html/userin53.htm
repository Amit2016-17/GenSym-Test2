Using Readout Tables

You can use a readout table to display the current value of an attribute of any e-SCOR object. You can specify how frequently the readout table updates its current value. You can also choose to format time values, using hours, minutes, and seconds.

You typically use readout tables to display path attributes, since you cannot probe the attributes of a path.

To use a readout table, you must name the object whose attributes you want to display.

Caution: Be careful when you copy models that contain readout tables because the readout in the copied model will be displaying data about the original model. Be sure to update the Expression to Display of the readout tables in the copied model to refer to objects in the copied model.

To create a readout table:

1 Select a Readout Table from the Displays tab of the ReThink toolbar and place it on a workspace near the objects whose value you want to display.
2 Display the properties of the readout table and edit the Label for Display to be the label.
3 To name an object:
a Choose Tools > Developer Mode.
b Display the properties dialog for the object.
c Click the Customize tab.
d Configure the Names.
e Choose Tools > Modeler Mode.
For example, to display the value of the Current Waiting metric of a path whose name is output-path-1, you would use this expression:
the current-waiting of output-path-1
4 Configure the Expression to Display to specify a G2 expression that refers to the name of the attribute whose value you want to display.
Most objects use labels, rather than names; therefore, you typically need to name the object before you can create a readout table.
5 Configure the Default Update Interval to be the frequency with which the readout table updates its value.
6 Configure the Display Format to be interval to display time values using days, hours, minutes, and seconds, as opposed to seconds.

The readout table begins updating immediately according to the update interval you specify.
