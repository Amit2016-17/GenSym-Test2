Deleting Modules

You might have merged a module into your model or created a new module that you do not actually want to save with your model. Before you save your model, you must delete these modules.

Caution: You should only delete merged modules or user-defined modules. Do no delete e- SCOR required modules of the top-level module; otherwise, your model will not work correctly.

When you delete a module, e-SCOR automatically removes all references to the module in the module hierarchy. By default, e-SCOR deletes all associated workspaces. When deleting modules, we recommend that you use this default.

You can also delete a module without deleting associated workspaces. However, if you use this option, before you save your model, you must ensure that the workspaces have an associated module. To do this, you must manually edit the Module Assignments of each workspace, which is only available in Developer mode.

To delete a module:

1 Choose File > Modules > Delete or choose Delete on a module in the module hierarchy.
2 Click Module List or Module Tree, depending on how you want to view the modules.
For example, suppose you opened eSCOR.kb, then merged escor-demos.kb. Before saving the model, you might want to delete escor-demos.kb.

3 Select the module to delete.
By default, deleting the selected module deletes all workspaces associated with the module.
4 To delete the module without deleting its associated workspaces, click the Options tab and click the Delete Associated Workspaces option off.
5 Click OK to delete the module.

The module hierarchy no longer includes the deleted module. If the module hierarchy is currently showing, you must refresh the view to see the updated module hierarchy.
