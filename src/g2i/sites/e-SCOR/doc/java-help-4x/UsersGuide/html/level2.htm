Configuring Level 2 SCOR

Once you have configured the supply-chain model by configuring Level 1 roles, you configure Level 2 SCOR by configuring parameters for each category and product composite associated with each role in the model.

Depending on the planning mode you have configured for each Level 1 role, you only need to configure category parameters that are relevant to that planning mode. For example, if you are using a push planning mode, you do not need to configure parameters relating to orders. Similarly, you configure different parameters depending on whether a role uses a stock, make-to-order, or engineer-to-order planning strategy.

You can run an e-SCOR model without configuring any Level 2 SCOR parameters, using the default parameters. However, by default, the model does not compute any cost, financial, or asset metrics. To compute Cost Incurred metrics, you must configure the Cost per Transaction parameters for the Source, Make, and Deliver categories. To compute financial and asset metrics, you must configure the Net Selling Price of the delivery products.

This topic describes how to configure Level 2 SCOR for the simplest e-SCOR model, which consists of a Base Manufacturer role and a Consumer role. The topics on configuring parameters provide sample dialogs for the categories and product composites that appear in the simplest model. If the dialogs for other categories are significantly different, sample dialogs for these categories appear, as well. The descriptions of each category parameter apply to all categories, unless otherwise indicated in the description. For example, the Order Size parameter is only relevant for the Sc: Source Consumer category.

To configure Level 2 SCOR, you:

View the default role details for each role.
Configure process category parameters on the detail of each role, which includes the Plan, Source, Make, and Deliver categories.
Configure product composite parameters for each source and delivery product on the detail of the Products pool for each role.
