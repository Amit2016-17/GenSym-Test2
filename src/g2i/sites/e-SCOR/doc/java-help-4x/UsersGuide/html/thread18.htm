1 Determine role type.
1.1 After waiting the Initial Plan Delay parameter of the P2: Plan Source category, each Planning Period parameter, the P2 category increments the Number of Planning Periods metric by one.
1.2 The P2 category sorts all the source products by the Product Preference parameter, which you configure for each source product.

2 Get delivery products.
If the role is a Distributor, the P2: Plan Source category determines the delivery products for the role.
3 Determine order type.
If the role is a Distributor, the P2 category determines the Order Type parameter of each delivery product to determine the planning strategy to use.
3.1 Create MTO plan.
If the Order Type is Made-to-order (MTO), the P2 category determines the size of the replenishment order for source products, based on the size of the customer or replenishment orders for delivery products. The Distributor role receives the order for delivery products from the Order/Product process. The P2 category then sends the order for source products to the upstream Source category to source MTO products, which triggers the order-processing and fulfillment process, described in Order/Product Process.
The P2 category increments the On Order metric of the delivery product by the size of the replenishment order.
For details on how to configure the model to source MTO products, see Using Alternative Planning Strategies.
After completing this step, the process for a Distributor role continues to Step 6, Get source components..
3.2 Create ETO plan.
If the Order Type is Engineered-to-order (ETO), the P2 category determines the size of the replenishment order, based on the size of the customer or replenishment orders for delivery products. The Distributor role receives the order for delivery products from the Order/Product process. The P2 category also tags the order with the specific buyer, so the order-processing and fulfillment process can make and deliver ETO products for the specific customer that ordered it. The P2 category then sends the order to the upstream Source category to source ETO products, which triggers the order-processing and fulfillment process, described in Order/Product Process.
For details on how to configure the model to source ETO products, see Using Alternative Planning Strategies.
After completing this step, the process for a Distributor role continues to step 6, Get source components..
4 Determine planning mode.
If the role is a Distributor and if the Order Type of the delivery product is Stock, then the P2 category determines the planning mode, which is either Pull or Push. The role's Delivery Type parameter determines the planning mode.
4.1 Create push plan.
If the planning mode for stock products is Push, the P2 category computes the size of the order, based on the Push Amount parameter of the upstream supplier role's delivery product and the Push Amount parameter of the Distributor role's source product. The P2 category then sends the order to the upstream Source category to source stock products, which triggers the order-processing and fulfillment process in push mode.
For details on how the P2 category computes order size, using a push planning strategy, and how each role behaves in push mode, see Using Pull and Push Planning Modes.
After completing this step, the process for a Distributor role continues to step 6, Get source components..
5 Determine plan type.
If the role is a Distributor, if the Order Type is Stock, and if the planning mode is Pull, the P2 category determines its Plan Type parameter. The options are Replenishment and Forecast Customer. The Plan Type determines how the P2 category computes replenishment order size, based on planning parameters that you configure for the source product.
5.1 Create stock /replenishment plan.
If the planning mode for stock products is Pull and the Plan Type is Replenishment, the P2 category computes replenishment order size, based on the following parameters, which you configure for the delivery product of the Distributor role: Replenishment Threshold, Minimum Reorder Quantity, and Maximum Threshold. The P2 category then sends the order to the upstream Source category to source stock products, which triggers the order-processing and fulfillment process, described in Order/Product Process.
The P2 category increments the On Order metric of the delivery product by the size of the replenishment order.
For details on how the P2 category computes order size, using a stock/replenishment planning strategy, see Using a Stock/Replenishment Planning Strategy.
After completing this step, the process for a Distributor role continues to step 6, Get source components..
5.2 Create stock/forecast plan.
If the planning mode for stock products is Pull and the Plan Type is Forecast Customer, the P2 category computes order size, based on the following parameters, which you configure for the source product of the downstream buyer role: Estimated Contract Amount and Contract Duration. The P2 category then sends the order to the upstream Source category to source stock products, which triggers the order-processing and fulfillment process, described in Order/Product Process.
The P2 category increments the On Order metric of the delivery product by the size of the replenishment order.
For details on how the P2 category computes order size, using a stock/forecast planning strategy, see Using a Stock/Forecast Planning Strategy.
After completing this step, the process for a Distributor role continues to step 6, Get source components..
6 Get source components.
If the role is a Distributor, the P2 category determines the source products for the role after it performs steps 2, 3, 4, and 5.
If the role is a Manufacturer, the P2: Plan Source category determines the source products for the role. It performs this step immediately after step 1, Determine role type..
7 Determine order type.
If the role is a Distributor, the P2 category determines the Order Type parameter of each source product to determine the planning strategy to use. It performs this step after it performs steps 2, 3, 4, 5, and 6.
If the role is a Manufacturer, the P2 category determines the Order Type parameter of each source product to determine the planning strategy to use. It performs this step immediately after step 6, Get source components.
A Manufacturer role allows stock, MTO, and ETO planning strategies for source products, whereas a Distributor role allows only stock and MTO. If the planning strategy is stock, the roles allow only a stock/replenishment planning strategy; they do not allow a stock/forecast planning strategy. This means the P2 category ignores the Plan Type parameter.
7.1 Create stock /replenishment plan.
If the Order Type is Stock, the P2 category determines the size of the replenishment order, based on the following planning parameters for the source product of the role: Replenishment Threshold, Minimum Reorder Quantity, and Maximum Threshold. The P2 category then sends the order to the upstream Source category to order source products, which triggers the order-processing and fulfillment process, described in Order/Product Process.
The P2 category increments the On Order metric of the delivery product by the size of the replenishment order.
For details on how the P2 category computes order size, using a stock/replenishment planning strategy, see Using a Stock/Replenishment Planning Strategy.
7.2 Create MTO plan.
If the Order Type is Made-to-order (MTO), the P2 category determines the size of the replenishment order, based on the size of the customer or replenishment orders for delivery products, which the Order/Product process determines. The P2 category then sends the order to the upstream Source category to order MTO products, which triggers the order-processing and fulfillment process, described in Order/Product Process.
The P2 category increments the On Order metric of the delivery product by the size of the replenishment order.
For details on how to configure the model to source MTO products, see Using Alternative Planning Strategies.
7.3 Create ETO plan.
For Manufacturer roles only, if the Order Type is Engineered-to-order (ETO), the P2 category determines the size of the replenishment order, based on the size of the customer or replenishment orders for delivery products. The Manufacturer role receives the order for delivery products from the Order/Product process. The P2 category also tags the order with the specific buyer, so the order-processing and fulfillment process can make and deliver ETO products for the specific customer that ordered it. The P2 category then sends the order to the upstream Source category to source ETO products, which triggers the order-processing and fulfillment process, described in Order/Product Process.
For details on how to configure the model to source ETO products, see Using Alternative Planning Strategies.
Get source products (one time only, stock products).
The Source Planning process performs this step as an independent process, one time only, for stock products only. The P2 category determines all the stock source products of the role and creates replenishment orders for each source product.
