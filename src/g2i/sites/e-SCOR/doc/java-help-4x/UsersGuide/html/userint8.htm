Saving Models

When saving models, you typically save the file associated with the top-level module that you opened when you started the server. When you first create a model, e-SCOR opens the KB module file named escor.kb, which you then save to a unique file name. We recommend that you also change the name of the top-level module to match the file name. For details, see Renaming Modules.

You can also save the model to a single file, rather than as separate modules; however, in general, we do not recommend it.

Note: If your model is not properly modularized, e-SCOR requires that you save your model to an "all" file. We do not recommend saving models to an "all" file under these circumstances, because you will be unable to load or save individual modules. For information on how to avoid this condition, see Checking for Consistent Modularization.

By default, e-SCOR shows a progress dialog as it saves the model and saves the current workspace layout. Also, by default, it saves the top-level module only; it does not save the required modules. Unless you are customizing e-SCOR, you do not generally need to save the required modules.

To save a model:

To save a model to the .kb file that e-SCOR loaded when you started the server, choose File > Save, enter Ctrl + S, or click the standard toolbar button.
or
To save the model to a different .kb file, choose File > Save As and enter or choose the file name to save.

If you do not specify a file extension, e-SCOR adds .kb to the file name you specify.

To edit the save options:

Click the Options tab and configure the options to:
Save the model without showing the file progress dialog.
Save the model without saving the current workspace layout.
Save the required modules, as well as the top-level module.

To save the model and all dependent modules to a single file:

Choose File > Save as All File and configure the options, as needed.

e-SCOR creates a single .kb file whose name appends ALL to the default file name, for example, e-SCORALL.kb.
