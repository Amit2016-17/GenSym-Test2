Topic - Modeling a Manufacturing Process

Coordinating Source and Make Planning

A Manufacturer role uses source planning to determine when to place orders for source products and the size of those orders. The role uses make planning to determine when to create build orders for delivery products and the size of those orders.

You should coordinate source and make planning within the Manufacturer role to optimize performance as follows, depending on the planning mode:

If the
planning mode is...

Configure the Initial Plan Delay to be...

Pull

Smaller for the P3: Plan Make category than for the P2: Plan Source category so that make planning occurs before source planning.

Push

Smaller for the P2: Plan Source category than for the P3: Plan Make category so that source planning occurs before make planning.

In pull mode, you want the Manufacturer role to determine build order size before it creates customer orders for its source products; whereas in push mode, you want the role to determine customer order size before it creates build orders for delivery products that it will push downstream.

Also, the Planning Frequency should either be the same for P2 and P3, or a multiple. For example, if the Planning Frequency for P2 is 3 weeks, the Planning Frequency for P3 should be 3 weeks, 6 weeks, 12 weeks, and so on.

For information on configuring the planning mode, see Configuring the Planning Mode for Roles.
