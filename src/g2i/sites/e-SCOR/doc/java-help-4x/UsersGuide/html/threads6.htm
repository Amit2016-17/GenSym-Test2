1 Generate order
1.1 The ES: Enable Source category of the buyer receives an order from one of the following roles:
Consumer role: Customer orders for consumer products originate from a Consumer role. The size, timing, and frequency of customer orders are based on the following parameters, which you configure in the Sc: Source Consumer category:
- Start Time determines when the Consumer role generates its first order.
- Stop Time determines when the Consumer role stops placing orders.
- Order Period determines how often the Consumer role generates orders.
- Order Size determines the size of each order.
Distributor or Manufacturer role: Replenishment orders originate from the Source Planning process of a role, described in Source Planning Process. The P2: Plan Source category determines the frequency of replenishment orders and when the buyer generates its first order. The order size depends on the planning strategy, as follows:
- For stock products, the order size depends on the Plan Type parameter of the P2: Plan Source category, which computes order size, based on a stock/replenishment or stock/forecast planning strategy.
For details, see Using Stock Planning Strategies.
- For make-to-order and engineer-to-order products, the order size depends on the size of the order for delivery products.
For details, see Using Alternative Planning Strategies.
Base Manufacturer role: A Base Manufacturer role does not generate orders.
1.2 The ES category sends the order to the qualified supplier(s) and increments the Orders Sent metric of the ES category by one.
The ES category can send a single order to a single selected supplier, or it can split the order among multiple selected suppliers, based on the Award Criteria parameters of the Source category. It can also select suppliers based on contracts.
For information on choosing among multiple suppliers for the same source product, see Modeling a Process with Multiple Suppliers.
The Source category applies the Create Customer Order cost parameter to the Create Customer Order costs incurred metric and to the aggregate cost metrics for the role.
The Source category determines the Desired Turnaround and Order Fulfillment Preference parameters of the source product for the role.
Finally, the Source category computes the Quantity Ordered metric for the source product.
2 Enter order.
2.1 The ED: Enable Delivery category of the supplier receives the order and increments its Orders Received metric by one.
2.2 The Deliver category of the supplier enters the order, which takes the amount of time indicated by the Order Receipt to Entry Time parameter.
The Deliver category applies its Order Entry cost parameter to its Order Entry costs incurred metric and to the aggregate cost metrics associated with the role.
Once the Deliver category has entered the order, the Deliver category stores the order in the Orders pool on the role detail.
The Delivery category increments its Orders Received metric by one.
The Deliver category generates a chit, which triggers the next step.
3 Select order.
This step occurs asynchronously, based on one of three triggering events:
The Deliver category of the supplier receives a chit from the ED: Enable Deliver category indicating that an order is waiting to be filled.
The inventory level of the supplier's delivery product goes up, based on one of the following processes, depending on the role:
If the supplier is a Base Manufacturer or Manufacturer role, the Build process generates a chit, indicating that delivery products are available. For details, see Build Process.
If the supplier is a Distributor role, the Source Planning process generates a replenishment chit for source products, indicating that delivery products are available.
The Order Selection parameters of the supplier's Deliver category are configured to indicate that delivery products should be delivered, based on a time cutoff, such as "just-in-time" delivery. For details, see Restricting the Orders to Sort Based on a Time Cutoff.
Once triggered, the supplier's Deliver category determines which orders to fill first, based on the Order Fulfillment Preference parameter and based on one of the following, depending on the planning strategy:
The D1 category chooses an order, based on the Order Selection parameters.
The D2 category chooses an order on a first-in, first-out basis.
The D3 category chooses an order, based on the Build Selection parameters of the M3 category.
If the inventory level of the supplier's delivery product is sufficient to fulfill the selected order, based on the Order Fulfillment Preference of buyer's source product, then the Deliver category passes the order to the next step to fulfill the order. If the inventory level is not sufficient, the Deliver category waits for a new triggering event to select a new order; it never selects the order if the inventory level is not sufficient.
For details on how the D1 category chooses which order to fulfill first, see Configuring Order Selection Parameters.
4 Fulfill order.
4.1 The Deliver category of the supplier determines whether the order must be filled in its entirety or whether it can be filled as a partial order, based on the Order Fulfillment Preference parameter of the buyer's source product, as follows:
If the supplier has enough inventory to fill the entire order, the Deliver category sends the order to be filled, regardless of the Order Fulfillment Preference parameter.
If the supplier's Order Fulfillment Preference parameter is Whole and the supplier does not have enough inventory to fill the entire order, then the order would not have been selected in step 3. Select order.
If the supplier's Order Fulfillment Preference is Partial and the supplier has enough inventory to fill part of the order, the Deliver category sends the order to be filled for the part it can and creates a back order for the balance of the order. The back order goes back to step 2.3 under Step 2. Enter order., which generates a new chit.
Note that if the Deliver category creates a partial shipment, the Product Shipments Sent metric of the Deliver category does not equal the Orders Received metric of the Deliver and ED: Enable Delivery categories, and the Orders Sent metrics of the Source and ES: Enable Source categories.
4.2 The Deliver category applies its Order Fulfillment cost parameter to its Order Fulfillment costs incurred metric and to the aggregate cost metrics associated with the role.
5 Pick material.
The Deliver category of the supplier chooses delivery products from inventory to fulfill the order, which takes the amount of time indicated by the Pick Material Time parameter.
When the Deliver category removes products from inventory, it decrements the Inventory Level and Potential Inventory Level metrics and updates the Inventory Status metric of the delivery product.
Note that the D1: Deliver Stocked Product and D2: Deliver Make-to-Order Product categories remove products from inventory; however the D3: Deliver Engineer-to-Order Products does not because ETO products are not placed in inventory.
6 Pack container.
The Deliver category of the supplier packs the delivery products into a container for shipment, which takes the amount of time indicated by the Packing Time parameter.
7 Send shipment.
7.1 The Deliver category of the supplier sends the product shipment to the buyer, which takes the amount of time indicated by the Order Shipment to Receipt Time parameter.
The Deliver category applies the Distribution and Transportation cost parameter to its Distribution and Transportation costs incurred metric and to the aggregate cost metrics associated with the role.
The Deliver categories compute the Order Entry to Ship Time (Days) metric.
7.2 If the Financial Payment Terms parameter of the buyer role is greater than zero, then sending a shipment generates an invoice to the buyer, which triggers the Financial process, described in Financial Process.
The Deliver category does not send the invoice until the buyer receives the shipment. Therefore, the role waits the amount of time indicated by the Order Shipment to Receipt Time parameter before sending the invoice.
If the Financial Payment Terms is zero, then sending a shipment does not generate an invoice.
7.3 The Deliver category increments its Product Shipments Sent metric by one.
7.4 The Deliver category computes the Mfg Ready to Ship Time (Days) and the Fill Rates (%) metrics.
The ED: Enable Delivery category increments its Product Shipments Sent metric by one and computes the Delivery Performance (%) and Perfect Order Fulfillment (%) metrics.
8 Receive shipment.
8.1 The ES: Enable Source category of the buyer receives the product shipment, which increments its Product Shipments Received metric by one.
The ES: Enable Source category computes the Supplier On-Time Performance (%) metric. The Source category computes its Product Shipments Lead Time Delta (Days) metric and the Received Average Fulfillment Time (Days) of the source product.
Receiving a shipment takes the amount of time indicated by the Receive and Putaway Time parameter.
8.2 The Source category applies the Product Acquisition cost parameter to its Product Acquisition costs incurred metric and to the aggregate cost metrics associated with the role.
8.3 The Source category receives the product shipment, which increments its Product Shipments Received metric by one. The Source category increments the Quantity Received metric and the Incoming Amount metric of the source product by the number of products it receives.
If the buyer is a Consumer role, the role simply consumes the products.
If the buyer is a Distributor or Manufacturer role, the Source category effectively stores the products in inventory, which increments the Inventory Level and Potential Inventory Level metrics and updates the Inventory Status metric of the source product.
Note that the S1: Source Stocked Product, S2: Source Make-to-Order Product, and the S3: Source Engineer-to-Order Product all place source products in inventory; however, the Sc: Source Consumer category does not.
The Source category decrements the On Order metric of the buyer's source product and decrements the Awaiting Order Total metric of the supplier's delivery product by the number of products it receives.
