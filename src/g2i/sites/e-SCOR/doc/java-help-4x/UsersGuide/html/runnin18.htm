Showing Transient Objects

The model creates and deletes a number of objects as the simulation runs. The model sends these transient objects along the paths between categories and roles and stores some of these objects in pools on the detail of the role.

You can view the properties of these objects while the simulation is running. To view these objects, you must run the simulation in step mode.

Note: Objects flow on paths on the detail of each category, as well as between categories and roles. Therefore, depending on what you are viewing, you might or might not see any objects flowing through the model when you step through the simulation. Typically, you view objects as they flow through the model when customizing e-SCOR at Level 3, 4, and 5 SCOR.

To view the properties dialog for transient objects:

1 Pause the simulation.
For details, see Starting and Stopping the Simulation.
2 Show the detail of a role, category, or pool.
You can also show the detail of a Level 2 category to view the Level 3 SCOR implementation of the category, then continue drilling down into the detail until you see objects flowing through the model.
3 In Step mode, step through the simulation until you see the transient object whose properties you want to view.
You will see objects flowing through the model between Level 1 roles, Level 2 categories on the role detail, and Levels 3, 4, and 5, on category details. You will also see objects in pools, which are located on the role detail.
For details, see Running the Simulation in Step Mode.
4 Display the properties dialog for the transient object.

Each object has a different set of properties, which determines the behavior of the model. The properties derive from parameters that you configure for roles, categories, and product composites in the model.
