Topic - Using Pull and Push Planning Modes

Manufacturer Role in Push Mode

In push mode, a Manufacturer role determines how many components to source, when to make and deliver finished products, and how many finished products to make and deliver, as follows:

Process Category
When
How Much

Source

Source planning is ignored. Instead, the role receives components whenever the upstream supplier pushes them downstream.

Receives the Push Amount of the upstream supplier's delivery product, assuming the Push Amount of the role's source product is greater than zero.

The number of components the role receives indirectly determines how many finished products the role makes and delivers.

Make

Creates internal build orders for finished products, based on available components and the make planning cycle (P3: Plan Make).

Makes as many finished products as it has build orders, which depends indirectly on inventory levels for components.

Deliver

As it creates internal build orders, creates internal replenishment orders for finished products, based on build-order size, which takes place on the same make planning cycle (P3: Plan Make) as the build orders.

Delivers finished products whenever it can, based on inventory levels for finished products.

Delivers as much as it can, based on build orders for finished products, which depends indirectly on inventory levels for components.

The Push Amount of the delivery product is ignored.
