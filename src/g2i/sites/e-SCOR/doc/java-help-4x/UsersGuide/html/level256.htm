Description

Product

Maximum Batch Size
Minimum Batch Size

The maximum and minimum number of delivery products to manufacture in each batch. The role can only manufacture finished products if the order size is greater than or equal to the Minimum Batch Size and less than or equal to the Maximum Batch Size. The value of the Total Build Time parameter can be proportional to the batch size, based on the Built Time Dependent on Order Size option on the product composite. The default values are 100 and 0, respectively.

Product

Yield

A percentage of the Batch Size, which determines the number of delivery products the role can actually use to fulfill build orders. The rest is considered defective product. The default value is 1.0.

If you configure this number to be less than 1.0, the Make category creates additional build orders for delivery products to make up for the loss.

Product

Build Time Dependent on Order Size

Whether the Total Build Time parameter of the Make category depends on the actual size of a batch. If Build Time Dependent on Order Size is true and the size of the build order is for less than a complete batch, the actual time it takes to manufacture the batch is adjusted proportionally. If Build Time Dependent on Order Size is false, the time it takes to manufacture a batch of products is the entire Total Build Time, regardless of whether the batch is complete. The default value is false.

Multipliers

Total Build Time

A number that the role multiplies by the Total Build Time parameter of the Make category to determine the actual time it takes to make one batch of delivery products. The default value is 1.0, which means it takes exactly the Total Build Time.

For details, see Configuring Multipliers for Timing Parameters.

Multipliers

Order Release to
Build Time

A number that the role multiplies by the Order Release to Build Time parameter of the Make category to determine the actual time it takes from when the Make category receives a build order for delivery products to when it receives components. The default value is 1.0, which means it takes exactly the Order Release to Build Time.

For details, see Configuring Multipliers for Timing Parameters.

Multipliers

Production Material
Cycle Time

A number that the role multiplies by the Production Material Cycle Time parameter of the Make category to determine the actual time it takes from when the role receives components to when it starts to manufacture delivery products. The default value is 1.0, which means it takes exactly the Production Material Cycle Time.

For details, see Configuring Multipliers for Timing Parameters.

Multipliers

Move to Delivery

A number that the role multiplies by the Move to Delivery parameter of the Make category to determine the actual time it takes from when the Make category finishes manufacturing a batch of delivery products to when the finished build order is transferred to the delivery location. The default value is 1.0, which means it takes exactly the Move to Delivery.

For details, see Configuring Multipliers for Timing Parameters.

Multipliers

Engineering Time

A number that the role multiplies by the Engineering Time of the Make category to determine the actual time it takes to engineer a single batch of engineer-to-order delivery products. The default value is 1.0, which means it takes exactly the Engineering Time.

For details, see Configuring Multipliers for Timing Parameters.
