Determining Optimal Replenishment Levels

Suppose you want to determine the optimal value for the Replenishment Threshold parameter for the delivery product of a Distributor role. At the same time, you want to ensure that the Supplier On-Time Performance (%) metric of the downstream Consumer role never falls below 100%.

You could create a Simulation Manager that runs a number of simulations, using different values for Replenishment Threshold, as this script shows:

select-scenario(scenario-1)
set-replenishment-threshold-parameter(site a,product,1800,0)
set-replenishment-threshold-parameter(site b,product,4000,0)
set-replenishment-threshold-parameter(site c,product,1200,0)
set-replenishment-threshold-parameter(central distributor,
   product,7000,0)
start-simulation(45 weeks)
wait-simulation
set-file-reporting(level 2 output,true,
   Level 2 Output Report 1.csv)
save-report(level 2 output)
set-replenishment-threshold-parameter(site a,product,2000,0)
set-replenishment-threshold-parameter(site b,product,5000,0)
set-replenishment-threshold-parameter(site c,product,1400,0)
set-replenishment-threshold-parameter(central distributor,
   product,8300,0)
start-simulation(45 weeks)
wait-simulation
set-file-reporting(level 2 output,true,
   Level 2 Output Report 2.csv)
save-report(level 2 output)
set-replenishment-threshold-parameter(site a,product,2250,0)
set-replenishment-threshold-parameter(site b,product,6000,0)
set-replenishment-threshold-parameter(site c,product,1500,0)
set-replenishment-threshold-parameter(central distributor,
   product,9750,0)
start-simulation(45 weeks)
wait-simulation
set-file-reporting(level 2 output,true,
   Level 2 Output Report 3.csv)
save-report(level 2 output)
