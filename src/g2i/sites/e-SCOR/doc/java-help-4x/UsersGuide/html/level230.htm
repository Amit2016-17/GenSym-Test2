Order Type

(Mb) The planning strategy for the category, which can be Stock, Made-to-order, or Engineered-to-order. The default is Stock.

The Order Type determines how a Base Manufacturer role computes build order size for delivery products. You specify Order Type when configuring the role to use alternative planning strategies, as described in Using Alternative Planning Strategies.

Build Order

Batch Preference

Build Orders

(M1 and M2) Whether to manufacture complete batches (whole) or to allow partial batches (partial). The default value is partial, which allows the Make category to manufacture as many finished products as it can, given available components. That way, the Manufacturer role can manufacture batches, even if does not receive components fast enough to keep up with build orders. This situation can occur under these conditions:

Large batch sizes.
Low component inventory.
Components arrive slower than P3 frequency.
Customer orders for finished products arrive faster than P3 frequency.

When Batch Preference is whole, the Make category waits to manufacture finished products from build orders until it has enough components to make a complete batch.

Note: A Batch Preference of whole can result in situations where batches are never manufacture, when the Batch Size is too large and the order quantity is too small.

Build Orders

(M1) The criteria that determines how to sort build orders for stock products.

The Make category sorts build orders waiting to be manufactured, based on up to four criteria of an order, which are numeric values.

The Make category sorts build orders, based on the criteria on the First tab. If two build orders have the same value for the specified criteria, the Make category sorts build orders, based on the criteria on the Second tab, and so on.

For a description of the Build Selection parameters and examples of how to use them, see Configuring Build Selection Parameters.
