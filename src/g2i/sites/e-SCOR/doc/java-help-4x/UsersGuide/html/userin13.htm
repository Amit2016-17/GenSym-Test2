Showing the Module Hierarchy

To begin working with modules, you should become familiar with the existing module hierarchy. When showing the module hierarchy, you can see which modules require which other modules in a tree view. You can also see the contents of each module, including all the top-level workspaces assigned to the module and all the system tables associated with the module.

You can go to and display properties of workspaces and system tables associated with a module. For information on editing workspace properties, see Editing Workspace Colors and Margins. For information on editing system table properties, see the G2 Reference Manual.

To show the module hierarchy:

1 Choose Tools > Module Hierarchy.
2 To expand the module hierarchy, click the plus sign next to a module, double-click a collapsed module, or click the Expand All button: .
3 To collapse the module hierarchy, click the minus sign next to a module, double-click an expanded module, or click the Collapse All buttons: .
4 To display the module hierarchy with a different module as the top-level module, enter the name of a module in the Root Module Name field, press Enter, and expand the module hierarchy.
5 To refresh the module hierarchy after making modifications, click the Refresh button: .
