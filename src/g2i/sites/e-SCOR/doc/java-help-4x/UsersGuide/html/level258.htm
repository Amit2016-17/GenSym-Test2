Planning Tab

Group/
Parameter

Description

Pull

Replenishment Threshold

When using a stock/replenishment planning strategy in pull mode, the minimum number of delivery products the role needs to maintain in inventory before creating a new build order. The default value is 500.

You configure this parameter when the Plan Type of the P2: Plan Source of a Distributor role or the P3: Plan Make category of a Manufacturer role is Replenishment.

For details, see Using a Stock/Replenishment Planning Strategy.

Pull

Minimum Reorder
Quantity

When using a stock/replenishment planning strategy in pull mode, the minimum number of delivery products for which the role creates build orders, when the inventory dips below the Replenishment Threshold. The default value is 100.

You configure this parameter when the Plan Type of the P2: Plan Source of a Distributor role or the P3: Plan Make category of a Manufacturer role is Replenishment.

For details, see Using a Stock/Replenishment Planning Strategy.

Pull

Maximum Threshold

When using a stock/replenishment planning strategy in pull mode, the maximum number of delivery products the role tries to maintain in inventory. The default value is 100,000.

When planning how many finished products to make, the role plans to have no more than this maximum in inventory.

Note: Unlike the Inventory Maximum on the Inventory tab, the Maximum Threshold does not represent a fixed maximum; the role does not discard inventory if the Inventory Level exceeds the Maximum Threshold. It merely tries never to exceed this threshold when planning how much to make.

You configure this parameter when the Plan Type of the P2: Plan Source of a Distributor role or the P3: Plan Make category of a Manufacturer role is Replenishment.

For details, see Using a Stock/Replenishment Planning Strategy.

Push

Push Amount

When using a push planning strategy, the number of delivery products per make planning cycle a supplier role delivers to all its downstream buyers. The default value is 0.

You configure this parameter when the role's Delivery Type is Push.

For details, see Using Push Mode Planning.
