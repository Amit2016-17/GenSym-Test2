1 Finalize engineering.
The M3: Engineer-to-Order category finalizes engineering, which takes the amount of time indicated by the Engineering Time parameter.
The M3 category applies the ECO cost parameter to its ECO costs incurred metric and to the aggregate costs for the role.
2 Select build orders.
The M3 category computes the Order Entry to Mfg Time (Days) metric.
If the M3 category has multiple build orders competing for resources, it sorts build orders for delivery products on a random basis. The M3 category does not create back orders, and it only chooses build orders for which all the components are available.
Selecting a build order takes the amount of time indicated by the Order Release to Build Time parameter.
3 Get components.
The M3 category gets the source products for the build order, which takes the amount of time indicated by its Production Material Cycle Time parameter.
The M3 category applies the Production Material Handling cost parameter to its Production Material Handling costs incurred metric and to the aggregate cost metrics associated with the role.
The M3 category retrieves the number of components from inventory that are required to manufacture the build order. The M3 category increments the Work in Progress metric of the source product to indicate that the role is currently manufacturing finished products, which are not yet reflected in the inventory metrics for the delivery product. It also decrements the Inventory Level and Potential Inventory Level metrics and updates the Inventory Status metric of the source product for the role.
The M3 category increments the Build Orders Started metric by one.
4 Determine yield.
If you configured the Yield parameter of the delivery product to model defective production in the manufacturing process, the M3 category adjusts the number of products it makes by the yield; the rest is considered defective and discarded.
5 Determine batches.
The M3 category determines the number of batches it needs to fill the build order by looking at the Maximum Batch Size and Minimum Batch Size parameter of the role's delivery product. It does this by breaking up the build order into separate batches, each of which is no larger than the Maximum Batch Size and no smaller than the Minimum Batch Size, which creates no more than one partial batch.
6 Manufacture products.
The M3 category manufactures its selected build order by creating the number of batches it determined in Step 2, Determine batches., which takes the amount of time indicated by the Total Build Time parameter, for each batch. The M3 category takes the same amount of time to manufacture a partial batch as it takes to manufacture a complete batch.
If you configured the Total Build Time multiplier parameter of the delivery product to model differences in build times for different products, the M3 category adjusts the Total Build Time by the multiplier.
The M3 category computes the Total Products Rejected metric of the delivery product, based on the Yield parameter of the delivery product. The M3 increments the Build Orders Completed metric by one.
7 Stage products.
The M3 category places the manufactured products directly in the Staging Area pool, which takes the amount of time indicated by the Move to Delivery parameter. It does not place products in inventory; therefore, it does not increment the inventory metrics for the delivery product.
Once the delivery products are in the staging area, the M3 category decrements the Work in Progress metric of the source product for the role.
The M3 category computes its Make Cycle Time (Days) and the Total Products Accepted metric of the delivery product.
The M3 category generates a chit, which it sends to the downstream Deliver category for delivery. The chit initiates the order-processing and fulfillment process, described in Order/Product Process.
