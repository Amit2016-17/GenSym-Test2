Saving a Model in a Snapshot File

You can save the contents of a running model in a snapshot file at any time during the simulation. When you save a model in a snapshot file, e-SCOR pauses all running models, saves the entire model in its current running state, then starts all the models running again.

The snapshot file is a single, unmodularized KB file with a .snp extension.

The advantage to saving a snapshot is that when you load the snapshot, the simulation can continue running at exactly the point at which you saved it.

For information on loading snapshot files, see Warmboot from a Snapshot.

By default, e-SCOR shows a progress dialog as it saves the snapshot and saves the current workspace layout.

To save a running model in a snapshot file:

1 While a model is running, choose File > Save Snapshot.
2 Click Choose to display the Save File dialog for choosing the location for the snapshot file:

3 Choose the host and server URL or use the default, localhost:1111, then double-click the serverFileSystem directory to display the list of available drives.
4 Navigate to the desired drive and directory, enter a file name, and click Save File.
Note: The filename of the snapshot file must end in the .snp file extension.
5 Click OK in the Save Snapshot dialog.

By default, e-SCOR displays a progress dialog as it saves the snapshot and it does not save the workspace layout. For information on how to configure save options, see Saving Models.
