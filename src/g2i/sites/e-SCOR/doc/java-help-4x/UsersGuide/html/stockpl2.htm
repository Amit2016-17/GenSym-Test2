Configuring the Stock Planning Strategy

To determine the planning strategy for sourcing, making, and delivering stock products, you configure the Plan Type of a role's Plan category.

You configure different Plan categories for each role, which has different effects on planning, as this table describes:


For this role...

the Plan Type for the...

Which determines...

Base
Manufacturer

P3: Plan Make category

The size of build orders for delivery products, which determines how many delivery products the role makes and delivers. The role determines the size of build orders, based on the size of customer or replenishment orders for delivery products from downstream buyers.

Distributor

P2: Plan Source category

The size of replenishment orders for source products, which determines how many delivery products the role delivers. The role determines the size of replenishment orders for source products, based on the size of customer or replenishment orders for delivery products from downstream buyers.

Manufacturer

The P3: Plan Make category; the role ignores the Plan Type for the P2: Plan Source category

The size of build orders for delivery products, which determines how many delivery products the role makes and delivers. The role determines the size of build orders, based on the size of customer or replenishment orders for delivery products from downstream buyers. The role determines the size of replenishment orders for source products, based on the size of its build orders.

By default, when using a stock planning strategy, the role creates orders each source or make planning cycle, depending on the role. It waits to create replenishment orders for source products or build orders for delivery products until the next planning cycle, even if orders from downstream buyers come in during the planning cycle.

You can configure a role to create orders on demand, which means it creates replenishment orders for source products and build orders for delivery products whenever the role receives an order from a downstream buyer for delivery products. You can configure the role to create orders on demand in addition to creating orders each planning cycle or instead of each planning cycle.

To configure the stock planning strategy for a role:

1 Show the detail for each Base Manufacturer, Distributor, and Manufacturer role in the model for which you want to configure the stock planning strategy.
2 Display the properties dialog for the P2: Plan Source and P3: Plan Make categories, as appropriate for the role, and configure the Plan Type parameter to be either Replenishment or Forecast Customer.
For details on which planning strategy to use, see:
Using a Stock/Replenishment Planning Strategy.
Using a Stock/Forecast Planning Strategy.
3 Configure the Planning Period and Initial Plan Delay parameters to determine the frequency with which the role creates orders and when the first plan goes into effect.
4 To configure the role to create orders on demand, enable the Replenishment Planning option for the same Plan category whose Plan Type you configured.
5 To configure the role to compensate for expected loss due to yield, enable the Compensate for Yield option for the same Plan category whose Plan Type you configured.
Tip: To configure the role to create replenishment orders on demand only, configure the Planning Period to be a very large number, such as 10 years, so that cyclical planning never takes place.
