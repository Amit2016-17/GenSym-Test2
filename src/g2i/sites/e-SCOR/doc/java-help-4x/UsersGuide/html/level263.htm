Parameter

Description

Pull

Replenishment Threshold

When using a stock/replenishment planning strategy in pull mode, the minimum number of source products the role needs to maintain in inventory before creating a new replenishment order. The default value is 500.

You configure this parameter when the Plan Type of the P2: Plan Source of a Distributor role or the P3: Plan Make category of a Manufacturer role is Replenishment.

For details, see Using Stock Planning Strategies.

Note: A Consumer role ignores this parameter because it generates orders, based on parameters you configure in the Sc: Source Consumer category, rather than based on source planning.

Pull

Minimum Reorder
Quantity

When using a stock/replenishment planning strategy in pull mode, the minimum number of source products for which the role creates replenishment orders, when the inventory dips below the Replenishment Threshold. The default value is 100.

You configure this parameter when the Plan Type of the P2: Plan Source of a Distributor role or the P3: Plan Make category of a Manufacturer role is Replenishment.

For details, see Using Stock Planning Strategies.

Note: A Consumer role ignores this parameter because it generates orders, based on parameters you configure in the Sc: Source Consumer category, rather than based on source planning.

Pull

Maximum Threshold

When using a stock/replenishment planning strategy in pull mode, the maximum number of source products the role tries to maintain in inventory. The default value is 100,000.

When planning how many source products to order, the role plans to have no more than this maximum in inventory.

Note: Unlike the Inventory Maximum on the Inventory tab, the Maximum Threshold does not represents a fixed maximum; the role does not discard inventory if the Inventory Level exceeds the Maximum Threshold. It merely tries never to exceed this threshold when planning how much to source.

You configure this parameter when the Plan Type of the P2: Plan Source of a Distributor role or the P3: Plan Make category of a Manufacturer role is Replenishment.

For details, see Using Stock Planning Strategies.

Note: A Consumer role ignores this parameter because it generates orders, based on parameters you configure in the Sc: Source Consumer category, rather than based on source planning.

Push

Push Amount

When using a push planning strategy, the relative number of source products per make planning cycle the buyer role receives from its upstream supplier, compared with other buyers. The default value is 0.

You configure this parameter when the upstream supplier role's Delivery Type is Push.

For details, see Using Pull and Push Planning Modes.
