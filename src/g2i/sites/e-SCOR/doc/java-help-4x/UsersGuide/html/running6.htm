Topic - Running the Simulation

Problems that Can Occur When Running a Simulation

The most common mistakes you can make when configuring a model are:

You do not label each role with a unique name, when multiple roles of the same type exist.
You fail to connect roles together correctly, using input and output paths.
You forget to assign product specifications to roles.
You forget to assign resources to Base Manufacturer and Manufacturer roles.

These configuration errors cause the simulation to generate an error, then stop.

This table describes errors that e-SCOR can generate, when they occur, and how to correct the problem:

Errors
When It Can Occur
Solution

Invalid path

When you first start the simulation and after resetting

Disconnect and reconnect the yellow path.

No upstream supplier for <product>

When the simulation begins initializing

Ensure that you have assigned product specifications to roles correctly, and ensure that you have connected all paths between roles and through routers correctly.

Incompatible combination of pull and push for <role>

When the simulation begins initializing

Ensure that the Source Type and Delivery Type of each role is compatible.

No award exists for <product>

When the simulation begins processing orders

Ensure that you have connected all paths correctly.

Invalid order for <product>

When the simulation begins delivering product shipments

Ensure that all orders have an order size greater than zero.

Other things to note when a simulation is running are:

The simulation does not compute all metrics immediately. For example, it does not compute financiafSl metrics for roles until one or two financial cycles have completed, due to the time lag in payments.
If the simulation does not compute financial or asset metrics for a role after several financial periods have completed, then you must configure the Net Selling Price parameter of the delivery products for the role. For a Base Manufacturer role, you must configure the Purchase Cost parameter for delivery products, located on the Source tab to compute outgoing financial metrics for the role.
Occasionally, when you pause the simulation, certain metrics that should add up do not add up. This situation only occurs between clock ticks; it does not occur while the model is running.
Because the Role Verification process takes 2 seconds of simulation time to complete, you should configure all timing initialization parameters to begin after 2 seconds from the start time of the simulation. These parameters are Initial Plan Delay and Initial Contract Delay. Neither of these parameters should have a value of less than 2 seconds.
