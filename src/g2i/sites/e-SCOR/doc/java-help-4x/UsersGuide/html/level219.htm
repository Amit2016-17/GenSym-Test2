Configuring Initial Order Demand through a Report

By default, initial order demand for the overall supply chain is based on parameters that you configure for each Sc: Source Consumer category in the model.

You might have actual order-demand data for your supply chain, which you would like to use in your model. To do this, you configure initial order demand through a report by:

Creating a Demand Input Report with the initial order-demand data.
Enabling the importing of data through the report and configure the report to use.

When you run the simulation, the model creates orders for as many products as you configured in the report, then stops. You configure a duration for each order, which is offset from the start of the simulation. The duration can be a number, which the model interprets using the time unit of the report, or it can be a duration, which uses a format such as 4 weeks, 2 days, 1 hour, and 30 minutes.

You can configure through a report values for the following attributes of an order:

Product Name/SKU
Order Period
Order Size
Desired Turnaround
Order Fulfillment Preference
Customer Preference

These attributes correspond to the Level 2 category and product composite parameters of the same name.

Note: The Demand Input Report uses Duration as the column header for the Order Period parameter. The rows can be in any order; you do not need to sort them based on the Duration.

To configure initial order demand through a report:

1 Create product specifications for each product whose initial order demand you want to configure through a report, then assign each of these product specifications to a Consumer role in the model.
For example, if you are configuring initial order demand through a report for computers and monitors, create one product composite named Computer and another named Monitor and assign each to a Consumer role. You must also assign them to the appropriate supplier roles upstream in the model.
2 Display the e-SCOR toolbar, click the Reports tab, and create a Demand Input Report on the model detail.
3 Create a report template for the Demand Input Report.
For details, see one of the following, depending on whether you want to create the report locally or in Excel:
Creating Reports.
Creating Reports in Excel.
4 In the report, configure the Product Name associated with each order, the duration of each order, and the parameters of each order.
To use the default value for the parameter, leave cells empty.
5 Apply the data to the model.
For details, see one of the following, depending on whether you created the report locally or in Excel:
Applying Input Report Data to the Model.
Applying Input Report Data to the Model from Excel.
6 On the Source tab of the Sc: Source Consumer category, click the Enabled option of the Order Demand Import group "on".
7 Configure the Report Title of the Sc category to be the name of the Demand Input Report that contains the order demand data to use.

e-SCOR displays all Demand Input Reports that exist on the model detail in the Report Title dropdown list.

Here is a sample Demand Input Report in Excel, which creates orders for computers and monitors. The report specifies the duration of each order, using the time unit of the report, which is hours. It generates orders at each time unit in sequence; it generates orders for two monitors 48 hours into the simulation, for one computer 120 hours into the simulation, for two computers 144 hours into the simulation, and so on. The order size for computers varies between 21 and 33 units, and the order size for monitors varies between 2 and 3 units. Computer orders have a Desired Turnaround of 3 days (72 hours), and monitors have a Desired Turnaround of 4 days (96 hours). Computers have an Order Fulfillment Preference of complete, and monitors have an Order Fulfillment Preference of partial.

When the simulation runs, the model generates 13 orders for the products listed in the report, then stops.
