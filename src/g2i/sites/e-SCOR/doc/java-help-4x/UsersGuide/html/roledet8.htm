Modeling Seasonal Demand for Consumer Products

You can configure a Consumer role to model seasonal demand for a single consumer product. To do this, you create multiple Sc: Source Consumer categories on the role detail and configure each category to have different order timing, period, and size parameters. In addition, you configure the Product Selection parameters and the Source ID parameter of the Sc category to uniquely identify each source product.

This table shows how you might configure two Sc categories associated with the same consumer product for normal and holiday season demand:

Default Consumer Product
Holiday Season Consumer Product

Source ID = Normal Demand
Start Time = 1 week
Stop Time = 46 weeks
Order Period = 4, 5, 6
Order Size = 8, 10, 12

Source ID = Holiday Demand
Start Time = 47 weeks
Stop Time = 52 weeks
Order Period = 7, 8, 9
Order Size = 23, 25, 28

Note: If the Consumer role's Source Type is Push, the role cannot source multiple products of the same type, which means it cannot have multiple Sc: Source Consumer categories on its detail. In this case, the Consumer role receives products regardless of consumer demand.
Caution: Whenever you add categories to the role detail, you should click the Upgrade option for the role "off;" otherwise, when you install a new version of e-SCOR, the new version of e-SCOR will overwrite your changes to the role detail to use the default role template.

To model seasonal demand for consumer products:

1 Follow the steps under Adding Multiple Source Categories to Buyer Role Details.
2 Display the properties dialog for each Sc: Source Consumer category on the role detail.
When modeling seasonal demand, you must uniquely identify each source product by configuring the Source ID parameter.
3 Configure the Source ID parameter on the Source tab of the Sc category to be a unique name.
For example, you might use Holiday Demand for one Sc category's Source ID and Normal Demand for the other.
4 Configure the Start Time, Stop Time, Order Period, and Order Size parameters on the Order tab of each Sc: Source Consumer category to reflect the timing, frequency, and size of orders for each unique product specification.

This figure shows the Products pool detail and the detail of a Consumer role with two Sc: Source Consumer categories, where one sources computers under conditions of normal demand and the other sources computers during the holiday season. The role uses separate Sc: Source Consumer categories for each finished product, because each has a different value for the Start Time, Stop Time, Order Period, and Order Size parameters. The Source ID parameters uniquely identify each source product.
