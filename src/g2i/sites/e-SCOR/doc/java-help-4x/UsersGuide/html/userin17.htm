Renaming Modules

You typically rename the top-level module to reflect the name of your application. Each new workspace that you create is automatically assigned to the top-level module. Any existing workspaces are also automatically reassigned to the renamed module.

For information on configuring the Module Assignment of a workspace, see Assigning Top-Level Workspaces to Different Modules.

Caution: You should only rename the top-level module or user-defined modules. Do not rename any of the built-in modules; otherwise, you model will no longer run.

To rename a module:

1 Choose File > Modules > Rename to display the Rename Module dialog.
By default, the Rename Module dialog displays an alphabetical listing of the modules, as follows:

2 To view the module hierarchy as a tree view, click the Module Tree option, then double-click each node in the tree to expand it, as follows:

3 Select the module to rename, for example, the top-level module named escor.
4 Enter the new name for the module in the New Name field and click OK.

The module name must be a symbol.

The module hierarchy shows the new name for the module. If the module hierarchy is currently showing, you must refresh the view to see the updated module hierarchy.
