Switching User Modes

You build and run e-SCOR models in one of three user modes. The user mode determines what you can and cannot do when you create a model and run the simulation. For example, the user mode determines the parameters that you can configure when creating a model.

e-SCOR supports three user modes for these classes of users:

Managers and end users

Browser

View pre-built models without damaging it in any way. Browsers cannot open, save, run, or configure models.

Business analysts who create models

Modeler

Create, connect, and configure Level 1 roles and Level 2 categories to create models and run simulations. This is the default user mode.

e-SCOR customizers experts and G2 programmers

Developer

Customize the behavior of the model at Level 3 and Level 4 SCOR. Developers have the full capabilities of ReThink, the underlying modeling toolkit upon which e-SCOR is based.

End users of fully developed e-SCOR applications generally work in Browser mode. Browser mode is restricted so that users may run a model but may not create, configure, or delete objects.

As a model developer, you will almost always be working in Modeler mode. This manual assumes you are working in Modeler mode, unless otherwise stated. Occasionally, as a model developer, you will also need to go into Developer mode to perform certain tasks.

If you are an expert who is customizing e-SCOR, you will be working exclusively in Developer mode. The user mode that is available to you depends on your login privileges.

To switch to a different user mode:

Choose Tools > Browser Mode, Modeler Mode, or Developer Mode.
