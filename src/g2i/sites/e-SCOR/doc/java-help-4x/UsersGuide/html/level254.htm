Parameter

Description

Product

Starting Inventory

The number of delivery products in inventory at the start of the simulation. Configure this parameter to avoid skewed order fulfillment lead times at startup while the model accumulates inventory. The default value is 0.

This metric is not applicable for a Consumer role, because a Consumer role does not keep an inventory.

Product

Inventory Maximum

The maximum number of delivery products the role can accommodate in its inventory. If the Inventory Level exceeds this maximum, the role discards the excess. The default value is 100,000,000.

The only time the inventory could potentially exceed this maximum is if a supplier pushes products onto a buyer, causing the buyer to exceeds its Inventory Maximum, or if the Maximum Threshold is greater than the Inventory Maximum. You should always configure the Maximum Threshold parameter to be less than the Inventory Maximum.

For details, see Using Push Mode Planning.

This metric is not applicable for a Consumer role, because a Consumer does not keep an inventory.
