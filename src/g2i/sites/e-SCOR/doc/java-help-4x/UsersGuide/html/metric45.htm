Viewing Metrics for Routers

e-SCOR models contain two types of routers:

Routers that appear between roles, which you create.
Category routers that appear between categories on the detail of a role.

Routers that appear between roles are only necessary when sending objects from a single role to multiple roles.

Both types of routers compute two metrics that help you determine whether you have configured your model correctly, both at the role level and on the role detail.

The following figure shows a sample properties dialog for a category router on the detail of a Base Manufacturer role:

Metric
Description

Total Input Objects

The total number of objects that the router has received since the start of the simulation, which should be positive.

Total Objects Not Output

The total number of objects that the router has received but not output since the start of the simulation, which should be zero.

Note: If the Total Input Objects is zero or if the Total Objects Not Output is non-zero, for either type of router, e-SCOR displays a message indicating that you have not configured the model properly. Ensure that the paths are connected to the routers correctly. If not, delete the connection and create it again.

To view metrics for routers:

Display the properties dialog for the router.
