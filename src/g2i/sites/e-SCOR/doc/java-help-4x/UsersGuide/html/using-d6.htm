Topic - Accessing External Databases

Creating and Configuring the Database Interface Object

The Database Interface object specifies:

A name, which the reports use to connect to the database.
The ODBC source as a connect string.
The connection configuration, which includes the connection type (TCP/IP), host, and port.

If the bridge process is running on the local machine, the host is localhost. The port number is 22041, or 22042, or 22043, etc., depending on the number of clients that are currently connected on that port.

You create a Database Interface object for each database you want your model to access. Typically, you write data to one database and read data from another database, which means you must create two Database Interface objects.

To create and configure a Database Interface object:

1 Display the e-SCOR toolbar and click the Database tab:


2 Create a Database Interface object and place it on the model detail or organizer detail.
The Database Interface object has a number of attribute displays that indicate the name, the connection status, and the connect string.
3 Display the properties dialog for the Database Interface object.
4 Configure the Names attribute to be any symbol, for example, reports-database-interface.
Tip: This is the Database Interface Name you specify when you configure the report for database access.
5 Configure the Gsi Connection Configuration attribute to specify the connection type, the host name, and the port number, for example:
tcp-ip host "localhost" port-number 22042
6 Configure the Database Connect String attribute to be the name of the ODBC data source, for example: "role_report"
