Disconnecting from and Connecting to the Server from the Client

You can disconnect from the server, then reconnect to the server again from the client. For example, you might want to do this to save computational resources when running multiple simulations, using the Scenario Manager.

The server to which you connect must be running before you can connect to it from the client.

To disconnect from and connect to the server from the client:

1 Choose Go > Disconnect from Server.
The status bar shows no connection in red.
2 Choose Go > Connect to Server, then enter the Host and Port of the server to which to connect.

This figure shows the Login dialog for connecting to the server:

The default Host and Port are localhost and 1111, respectively. You cannot configure the Url.

You configure the parameters on the Security tab if you have customized e-SCOR to require a secure login. By default, you cannot configure the User Mode.

To switch connections to a specific server from the client:

Choose Go > Switch Connection, then choose the server to which to connect.

e-SCOR automatically disconnects from the current server and connects to the new server. The status bar indicates the host and port of the new server to which the client is connected.
