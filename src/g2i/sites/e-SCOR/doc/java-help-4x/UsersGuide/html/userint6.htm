Topic - Working with Models

Saving and Opening Models

An e-SCOR model consists of a set of related modules that form a "knowledge base." Each file contains a stand-alone module, which together make up a module hierarchy. Each module is associated with its own .kb file, whose name typically corresponds to the module name. The module hierarchy consists of a top-level module and a number of lower-level modules. The top-level module requires the lower-level modules to run.

When working with models, you:

Open a model, which loads the top-level module and all directly required modules.
Save a model, which saves the top-level module and all the directly required modules, or the entire model in a single file.
Open a specific model at startup by creating a shortcut to the batch file that starts the e-SCOR server and configuring the model to load.
Save a model in a snapshot file and warmboot from a snapshot.
