Base Manufacturer role

1 Determine order type.
If the Order Type parameter of the Mb category is engineered-to-order, the Mb: Make Base Manufacturer of a Base Manufacturer role receives build orders from the Make Planning process, which triggers the next step.
For detail on make planning, see Make Planning Process.
2 Finalize engineering.
The Mb category finalizes the engineering, which takes the amount of time indicated by the Engineering Time parameter.
The Mb category applies the ECO cost parameter to its ECO costs incurred metric and to the aggregate costs for the role.
3 Select build orders.
Build-order selection triggers when it receives a build order from the previous step and when a manufacturing resource is available. If no resource is available, the Mb category waits until one becomes available to select build orders; therefore, the model can trigger build-order selection asynchronously.
If the Mb category has multiple build orders competing for resources, it sorts build orders for delivery products on a random basis. The Mb category does not create back orders.
The Mb category selects a build order, which takes the amount of time indicated by the Order Release to Build Time parameter.
4 Determine yield.
4.1 The Mb category increments its Build Orders Started metric by one.
4.2 If you configured the Yield parameter of the delivery product to model defective production in the manufacturing process, the Mb category adjusts the number of products it makes to account for the defective products.
5 Determine batches.
The Mb category determines the number of batches it needs to make to fill the build order by looking at the Maximum Batch Size and Minimum Batch Size parameters of the role's delivery product.
6 Manufacture products.
6.1 The Mb category manufactures its selected build order by creating the number of batches it determines in Step 2, Determine batches..
For each batch, the Mb category manufactures the finished products, which takes the amount of time indicated by the Total Build Time parameter. The Mb category takes the same amount of time to manufacture a partial batch as it takes to manufacture a complete batch.
If you configured the Total Build Time multiplier parameter of the delivery product to model differences in build times for different products, the Mb category adjusts the Total Build Time by the multiplier.
6.2 The Mb increments the Build Orders Completed metric by one and computes the Make Cycle Time (Days) metric to determine the performance of the manufacturing process. The Mb category computes the Total Products Rejected metric of the delivery product, based on the Yield parameter of the delivery product.
6.3 The Mb category computes the Total Products Accepted metric of the delivery product, based on the Yield parameter of the delivery product.
The Mb category releases the product to the staging area, which takes the amount of time indicated by the Move to Delivery parameter.
7 Generate and send payment.
The Mb category generates a payment for its raw materials, which is based on the Purchase Cost parameter of its delivery product and the number of batches it built. You configure the Purchase Cost on the Source tab for delivery product of the Base Manufacturer role.
The Mb category waits the amount of time indicated by the Financial Payment Terms parameter of the buyer role, then updates the Financial Payments Total metric for the role.
8 Stage products.
When the Order Type is engineered-to-order, the Mb category places the manufactured products directly in the Staging Area pool. It does not place products in inventory; therefore, it does not increment the inventory metrics for the delivery product.
The Mb category generates a chit, which it sends to the downstream Deliver category for delivery. The chit initiates the order-processing and fulfillment process, described in Order/Product Process.
Accumulate payments.
As an independent process, each Financial Period parameter, the Base Manufacturer role increments the Financial Payments Period metric for the role. This is represented in the diagram by the Accumulate Payments cyclical process.
