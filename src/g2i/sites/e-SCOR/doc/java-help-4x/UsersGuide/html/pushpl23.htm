Topic - Using Pull and Push Planning Modes

Distributor Role that Uses Pull-Push

The following table describes the behavior of a model in which a Distributor role's Source Type is pull and its Delivery Type is push. The numbers represent the order in which the planning events occur. In this case, push mode of the delivery phase takes precedence over pull mode of the source phase in terms of when planning takes place.

Note: The role must use a stock/replenishment planning strategy for its Deliver category; stock/forecast, make-to-order, and engineer-to-order are not valid.

When
How Much

(push)

1) Creates internal replenishment orders for finished products, based on the source planning cycle (P2: Plan Source).

Delivers finished products whenever it can, based on inventory levels for components.

Delivers the Push Amount of the role's delivery product, given default assumptions.

Source
(pull)

2) Creates internal replenishment orders for components, based on orders for finished products, which takes place based on the same source planning cycle (P2: Plan Source) as the internal replenishment orders for finished products.

Sources components, based on source planning cycle (P2: Plan Source).

Sources components, based on customer or replenishment orders for finished products, combined with the Replenishment Threshold and current inventory level of the source product.
