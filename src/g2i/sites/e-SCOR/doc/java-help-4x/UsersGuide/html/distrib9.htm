Restricting the Orders to Sort Based on a Time Cutoff

You can configure time-based cutoff values to restrict the orders that the Deliver category considers for delivery.

You can configure the cutoff to be a maximum acceptable time delta to use for determining which orders the Deliver category can fill. If the current order meets the sort criteria but cannot be filled from the current inventory, the Deliver category considers all other orders that meet the sort criteria that fall within the delta time. You can use this configuration to model a FIFO delivery strategy in which orders are considered for delivery if they fall within a given delta time relative to the order-placed time. The Time Delta Acceptable cutoff for sorting orders, based on time-based properties of an order behaves in an identical manner to the Maximum Delta Acceptable cutoff for numeric properties of an order.

You can also configure the cutoff to implement a just-in-time (JIT) deliver strategy. To do this, you configure the cutoff to be Current Time Delta Acceptable, and you configure the Cutoff Duration to be the amount of time it takes to pick, pack, and ship an order. The Deliver category ships only those orders whose Order Lead Time is less than the current time plus the Cutoff Duration. The Order Lead Time contains a time in the future when the buyer expects to receive the order.
