Creating Top-Level Workspaces for Modules

When you first create a module, the module appears in red in the module hierarchy, indicating that it does not have any top-level workspaces assigned to it yet. Whenever you create a new module, you should create a top-level workspace associated with the module. You create your model on this top-level workspace.

Tip: To verify the Module Assignment of a workspace, go into Developer mode, display the properties of the workspace, and click the Customization tab.

To create a top-level workspace assigned to a new module:

Display the module hierarchy and choose New > Workspace on a module.
or
Choose Workspace > New Workspace.

e-SCOR creates a new unnamed workspace that is assigned to that module. The module no longer appears in red in the module hierarchy tree view. If the module hierarchy is currently showing, you must refresh the view to see the updated module hierarchy.
