Controlling the Server State

For most modeling operations, the e-SCOR server must be running. Occasionally, you must pause the server to perform specific tasks, such as moving buttons on a workspace; otherwise, clicking the button executes the action. After pausing, you must resume running the server. You should never have to restart or reset the server, except when the software is not behaving as it should.

Note: Do not confuse starting, restarting, resetting, and resuming the server with starting, resetting, pausing, and continuing the simulation. Controlling the server state is not the same as controlling the simulation.
Caution: The server should normally be in the running state at all times; pausing or resuming the server may cause the client user interface to stop functioning.

You control the server state from the Go menu, as follows:

Menu Choice
Icon
State
Description

Start

running

Starts the server when it is paused.

Restart

running

Resets and starts the server when it is running or paused.

Pause

paused

Pauses the server when it is running.

Resume

running

Resumes running the server when it is paused.

Reset

reset

Resets the server without restarting when it is running or paused.

To control the server state:

Choose Go > Server State > Start, Restart, Pause, Resume, or Reset.
