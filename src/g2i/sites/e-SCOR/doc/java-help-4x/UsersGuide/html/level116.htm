Creating and Connecting an Individual Role to Multiple Roles

When a buyer role sources products from multiple upstream supplier roles and when a supplier role delivers products to multiple downstream buyer roles, you connect individual roles to multiple roles.

To create and connect an individual role to multiple roles, either upstream or downstream, the model requires:

A router, which routes appropriate objects to appropriate roles.
A junction, which merges objects together into a single path.

To create and connect a single role to multiple roles:

1 Display the e-SCOR toolbar and click the Roles tab.
This toolbar contains the Base Manufacturer, Distributor, Manufacturer, and Consumer roles.
2 Place on the model detail as many roles as are required to describe your supply chain, given your product hierarchy.
Place Base Manufacturer roles on the left side of the workspace and Consumer roles on the right side of the workspace.
3 Configure the Role Label parameter to be unique within the model.
If the model contains only one occurrence of a particular type of role, you do not need to configure the label.
4 Select a Router from the Roles tab of the e-SCOR toolbar and place it on the model detail between two adjacent roles, then connect the stubs leading from the single role to the multiple roles through the router.
A router is required on a path whenever a single role must send objects to multiple roles. This situation can occur when a role sends objects upstream or downstream in the supply chain. For example, this figure shows a single downstream Consumer role that sends objects to two upstream Base Manufacturer roles through a router:

Tip: If necessary, you can add input or output stubs to the router by choosing Create Input and Create Output on the router.
5 To connect multiple roles to a single role, connect the stubs together through a junction.
To do this, connect two roles together, then drag the additional stub into the connected path to create the junction.
A junction is required on a path whenever multiple roles must send objects to a single role. Whenever you have a router on one path connecting multiple roles, you must have a junction on the corresponding path connecting the same roles.
This figure shows multiple upstream Base Manufacturer roles that send objects to a single downstream Consumer role through a junction:

When you connect paths through a junction, the color of the path downstream of the junction changes to gray. You can edit the path color of the part of the path that is downstream of the junction to correspond to the path color of the upstream part of the path.
6 Display the properties dialog for the path, click the Animation tab, and configure the Empty Color to be blue or magenta, depending on whether it is an output or input path, respectively.
