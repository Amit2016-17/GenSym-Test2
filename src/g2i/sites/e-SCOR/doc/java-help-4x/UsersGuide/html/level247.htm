Source ID

Source

(Sc) A unique identifier for the source product of a Consumer role. Initially, the default value is the Sequence Number of the role. When you assign a product specification to the Sc category, the default becomes the product name of the assigned product specification.

You configure this parameter when sourcing multiple identical consumer products, using separate Sc categories.

For details, Modeling Seasonal Demand for Consumer Products.

Product Selection

All Products
Specific Product Name

Source

Whether the Source category parameters apply to all source products associated with the role or to a specific source product. By default, All Products is enabled.

You configure these parameters when the Source category sources multiple products.

When All Products is disabled, configure the Specific Product Name parameter to determine the specific source product to which the Source category parameters apply.

For details, see Adding Multiple Source Categories to Buyer Role Details.

Receive and Putaway
Time

Mode Type
Min
Mode
Max

Source

(S1, S2, S3) The time it takes from when the Source category receives source products to when they are put away in inventory. By default, you specify the Min, Mode, and Max of a triangular distribution. The default values are 0.

For information on how to use other distributions, see Configuring the Mathematical Distribution.

For information on configuring this parameter through a Logistical Lookup Report, see Configuring Logistical Parameters through a Report.

Note: This parameter is not relevant for the Sc category, because Consumer roles do not keep an inventory of source products.

Tip: Assets transfer from supplier to buyer at the end of the Order Shipment to Receipt Time and before the Receive and Putaway Time of the buyer role's Source category. When assets transfer from supplier to buyer, the In Transit metric of the buyer's source product increments. To cause assets to transfer at shipment time, configure the Receive and Putaway Time to be zero and configure the Order Shipment to Receipt Time to include the putaway time. The asset metrics for the supplier and buyer roles reflect this difference, as do the Supplier On-Time Performance and Delivery Performance (%) metrics of the ES: Enable Source and ED: Enable Delivery categories, respectively.

Contract Specifications

Use Contracts

Contract

Whether the buyer role uses contracts to choose its suppliers. By default, the Use Contracts option is "off," which means the buyer chooses its suppliers at random from among all qualified suppliers. Enable the Use Contracts option when you want to use contracts when sourcing identical products from multiple upstream suppliers.

For details, see Modeling a Process with Multiple Suppliers.

Contract Specifications

Initial Contract Delay

Contract

When contracts are in effect or when using a stock/forecast planning strategy, the time from the start of the simulation until the role sends its first purchase request to upstream suppliers for source products. The default value is 2 seconds.

For details, see Modeling a Process with Multiple Suppliers and Using a Stock/Forecast Planning Strategy.

Contract Specifications

Contract Duration

Contract

When contracts are in effect or when using a stock/forecast planning strategy, the length of time during which the contract is valid. The default value is 90 days.

For details, see Modeling a Process with Multiple Suppliers and Using a Stock/Forecast Planning Strategy.

For details, see Modeling a Process with Multiple Suppliers and Using a Stock/Forecast Planning Strategy.

Award Criteria

Sort 1 Attribute
Sort 2 Attribute
Sort 3 Attribute
Sort 4 Attribute

Contract

Determines how the buyer chooses qualified suppliers. The buyer chooses suppliers, based on the smallest value of the Sort 1 Attribute. If two suppliers have the same value for the sort attribute, the buyer looks at Sort 2 Attribute to determine which supplier should receive an award, and so on.

The options are Product Cost, Order Turnaround, and Supplier Preference, where:

Product Cost is based on the Net Selling Price of the supplier's delivery product.
Order Turnaround is based on the Published Delivery Lead Time of the supplier's delivery product.
Supplier Preference is based on the Supplier Preference of the supplier's delivery product.

The default value is None.

You only need to configure this parameter when sourcing identical products from multiple upstream suppliers.

For details, see Configuring the Award Criteria for Choosing Suppliers.

Supplier Information

Number of Suppliers

Contract

Determines the number of qualified suppliers to which the buyer issues awards. A buyer can issue awards to from one to four suppliers. The default value is 1.

You only need to configure this parameter when sourcing identical products from multiple upstream suppliers.

For details, see Configuring the Award Criteria for Choosing Suppliers.

Supplier Information

Split Order

Contract

Determines whether to send the order to a single supplier ("off") or whether to split the order among the specified number of suppliers ("on"). The default value is "off".

You only need to configure this parameter when sourcing identical products from multiple upstream suppliers.

For details, see Configuring the Award Criteria for Choosing Suppliers.

Supplier Information

Proportion 1
Proportion 2
Proportion 3
Proportion 4

Contract

When Split Order is "off," determines the percentage of the time, on average, that the order will go to each qualified supplier.

When Split Order is "on," determines the proportion of the order that goes to each qualified supplier.

The proportions correspond to each chosen supplier, based on the Award Criteria.

You only need to configure this parameter when sourcing identical products from multiple upstream suppliers.

For details, see Configuring the Award Criteria for Choosing Suppliers.
