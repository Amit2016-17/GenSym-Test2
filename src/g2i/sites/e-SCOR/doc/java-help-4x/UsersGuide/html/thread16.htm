e-SCOR User's Guide
Topic - What Happens When the Simulation Runs

Planning Processes

e-SCOR has two distinct planning processes, which occur in specific roles:

Make Planning process - A Base Manufacturer role and a Manufacturer role perform make planning to determine the size of each build order for delivery products and when to create those orders.
Source Planning process - The Distributor role and the Manufacturer role perform source planning to determine the size of each replenishment order for source products and when to create those orders.

A Base Manufacturer and Manufacturer role determine the size of build orders for delivery products, based on the planning strategy, which is either stock/replenishment or stock/forecast. Similarly, a Distributor role determines the size of replenishment orders for source products, based on the planning strategy. However, a Manufacturer role determines the size of replenishment orders for source products by looking at the size of build orders for delivery products.

Because a Manufacturer role performs both source and make planning, you should coordinate source and make planning to optimize the performance of your simulation, depending on the planning mode, which is either pull or push. For details, see Coordinating Source and Make Planning.

Source and make planning occur at regular time intervals, based on parameters that you configure in the P2: Plan Source and P3: Plan Make categories, respectively.

A Consumer role does not perform any planning. Instead, it places orders for a given size and frequency, based on parameters that you configure in the Sc: Source Consumer category.
