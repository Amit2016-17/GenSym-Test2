Topic - What Happens When the Simulation Runs

Process Diagrams

Each process provides a diagram that uses these symbols to represent various parts of the process:

Example
Description

The diagrams use large, curved arrows to represent the triggering event that initiates the process. Typically, the triggering event is the creation of an object by some other process, in which case the name of the object and the category that creates it appear next to the arrow. Click the name of the process to go to the diagram for that process.

Certain diagrams use a thin, cyclical arrow next to the triggering event to represent a periodic timing frequency, which initiates the triggering event.

The diagrams represent each step as a numbered box, whose name and number correspond to the numbered steps in the descriptions following the diagrams. Click the name of the step to go to the description of that step.

The category or categories in which the step occurs appears in the lower-right corner of the numbered box.

The diagrams use Sx, Mx, and Dx to refer to any of the three Source, Make, and Deliver categories-stock, make-to-order, or engineer-to-order-or the Sc: Source Consumer or Mb: Make Base Manufacturer categories, as appropriate. If a diagram uses a specific category, the description of the step refers to the specific category.

If the step causes the simulation clock to advance, a small "t" appears next to the box indicating that the step has associated timing.

If the step contributes to the aggregate cost metrics for the role, a small "c" appears next to the box indicating that the step has an associated cost.

The diagrams use thin, straight arrows below each step to indicate the sequential flow of events in the process. If a step creates an object, the name of the object appears next to the arrow.

In some processes, an asynchronous event occurs, which is either the result of another process or triggers another process or step within a process. The diagrams use a broken arrow for these events.

Certain diagrams use dashed lines between steps and for the step to indicate that the step is part of another process.

Following each process diagram is a table that describes the parameters each step uses to perform its tasks and the metrics each step computes, where relevant. The table contains these columns:

Name - The parameter or metric name.
P/M - "P" or "M" depending on whether it is parameter or metric.
Categorization - The categorization for each parameter and metric, which includes the SCOR functional categorizations.
e-SCOR Location - The role, category, or product composite in which the parameter or metric appears, plus the appropriate tab on the properties dialog.

The numbered steps that appear after the table describe each step in more detail. In general, the descriptions refer generically to buyers and suppliers. If a step applies to a specific role or category, the description refers to the specific object by name.

For a description of the categorizations, see Parameters and Metrics by Function.
