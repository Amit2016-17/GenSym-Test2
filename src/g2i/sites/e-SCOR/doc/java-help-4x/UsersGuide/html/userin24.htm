Transferring Objects Between Modules

Once you have started building a model, you might find you need to transfer objects between modules. For example, you might decide you want to separate your model into multiple modules, or you might decide you want to transfer definitions to the escor-customiz module.

One technique for transferring items between modules is to use a transfer module.

To transfer items between modules:

1 Create a new module named transfer, which directly requires the escor-customiz module.
For details, see Creating Modules.
2 Create a new top-level workspace associated with the transfer module and name it.
For details, see Creating Top-Level Workspaces for Modules.
3 Cut and paste the items you want to transfer to the top-level workspace associated with the transfer module.
4 Save the transfer module.
For details, see Saving Models.
5 Open the existing model to which you want to transferring the items.
For details, see Opening Models.
6 Merge the transfer module into the existing model.
For details, see Merging Modules.
7 Transfer the items from the top-level workspace associated with the transfer module to a workspace associated with the existing module.
8 Save the merged module with the new items.
9 Save the transfer module so it is empty again, then delete the transfer module.

For details, see Deleting Modules.
