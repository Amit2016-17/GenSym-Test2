Creating a Product Hierarchy with Components

You create a product hierarchy with components when modeling a Manufacturing process and when modeling a Distribution process that assembles components into kits.

To create a product hierarchy with components, create a single product specification, then add product specifications to the detail, as needed. You may add as many levels of detail as you need to describe the product hierarchy.

To create a product hierarchy with components:

1 Create a Product Structure and place a Product Specification on its detail.
For details, see Creating a Product Hierarchy with a Single Product Specification.
2 Choose Allow Components on the product specification to create detail.
3 Create and label additional product specifications on the detail, as needed, to describe the next level in the product hierarchy.
4 Repeat steps 2 and 3 as many times as needed to describe the product hierarchy.

For example, the following figure shows a product hierarchy for a computer, which is a consumer product. The computer consists of a CPU component, which is itself a finished product that consists of a computer case and electronics. The case and electronics components are considered base components, because they do not contain any components.

To show the components of a product specification:

Choose Show Components on a product specification in the hierarchy.

e-SCOR displays the detail of the product specification that contains components.

To delete all components associated with a product specification:

Choose Delete All Components on a product specification in the hierarchy.

e-SCOR deletes the detail for the product specification that contains the components.
