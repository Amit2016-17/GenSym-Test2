1 Determine production requirements.
1.1 After waiting the Initial Plan Delay parameter of the P3: Plan Make category, each Planning Period parameter, the P3 category determines all the delivery products of the Base Manufacturer or Manufacturer role.
The P3 category increments the Number of Planning Periods metric by one.
1.2 The P3 category sorts all the delivery products by the Product Preference parameter, which you configure for each delivery product.
2 Determine order type.
The P3 category looks at the Order Type parameter of the role's delivery product to determine the planning strategy to use. Based on the Order Type, the P3 category performs one of the following three steps for each delivery product:
3   Determine planning mode.
4   Create MTO plan.
5   Create ETO plan.
3 Determine planning mode.
If the Order Type is Stock, the P3 category determines the planning mode, which is either Pull or Push. The role's Delivery Type parameter determines the planning mode.
3.1 Determine plan type.
If the Order Type is Stock and the planning mode is Pull, then the P3 category determines the Plan Type parameter, which you configure for the P3 category. The options are Replenishment and Forecast Customer. The Plan Type determines how the P3 category computes the size of a build order, based on planning parameters that you configure for the delivery product.
3.2 Create stock /replenishment plan.
If the planning mode for stock products is Pull and the Plan Type is Replenishment, the P3 category determines the size of the build order, based on the following parameters, which you configure for the delivery product of the manufacturer role: Replenishment Threshold, Minimum Reorder Quantity, and Maximum Threshold. The P3 category then sends the build order to the upstream Make category to make stock products, which triggers the build-order and manufacturing process, described in Build Process.
The P3 category increments the On Order metric of the delivery product by the size of the build order.
For details on how the P3 category computes build order size, using a stock/replenishment planning strategy, see Using a Stock/Replenishment Planning Strategy.
3.3 Create stock/forecast plan.
If the planning mode for stock products is Pull and the Plan Type is Forecast Customer, the P3 category computes build order size, based on the following parameters, which you configure for the source product of the downstream buyer role: Estimated Contract Amount and Contract Duration. The P3 category then sends the build order to the upstream Make category to make stock products, which triggers the build-order and manufacturing process, described in Build Process.
The P3 category increments the On Order metric of the delivery product by the size of the build order.
For details on how the P3 category computes build order size, using a stock/forecast planning strategy, see Using a Stock/Forecast Planning Strategy.
3.4 Create push plan.
If the planning mode for stock products is Push, the P3 category determines build order size, based on the Push Amount parameter, which you configure for the delivery product. The P3 category then sends a build order to the upstream Make category to make stock products, which triggers the build-order and manufacturing process, described in Build Process.
For details on how the P3 category computes build order size, using a push planning strategy, see Using Pull and Push Planning Modes.
4 Create MTO plan.
If the Order Type is Made-to-order (MTO), the P3 category determines the size of the build order, based on the size of the customer or replenishment order the role receives from the order-processing and fulfillment process, described in Order/Product Process. The P3 category then sends the build order to the upstream Make category to make MTO products, which triggers the build-order and manufacturing process, described in Build Process.
The P3 category increments the On Order metric of the delivery product by the size of the build order.
For details on how to configure the model to manufacture and deliver MTO products, see Using Alternative Planning Strategies.
5 Create ETO plan.
If the Order Type is Engineered-to-order (ETO), the P3 category determines build order size, based on the size of the customer or replenishment order the manufacturer role receives from the order-processing and fulfillment process, described in Order/Product Process. The P3 category also tags the build order with the specific buyer, so the order-processing and fulfillment process can deliver the ETO product to the customer that ordered it. The P3 category then sends the build order to the upstream Make category to make ETO products, which triggers the build-order and manufacturing process, described in Build Process.
For details on how to configure the model to manufacture and deliver ETO products, see Using Alternative Planning Strategies.
Get delivery products (one time only, stock and MTO products).
The Make Planning process performs this step as an independent process, one time only, for stock and MTO products only. The P3 category determines all the stock delivery products of the role and creates build orders for each delivery product.
