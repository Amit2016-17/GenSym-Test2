Configuring Planning Parameters for Product Composites

Once the model creates its initial customer orders in pull mode, it uses source and make planning to determine the size of each replenishment or build order.

If a role uses a make-to-order or engineer-to-order planning strategy for sourcing, making, and/or delivering products, the role computes replenishment and/or build order size, based on the order size of the downstream buyer role. The only planning parameter you need to configure for product composites that use MTO is Maximum Threshold, which determines the maximum amount of product the role can maintain in inventory.

If the role uses a stock planning strategy, the default, the Plan Type of the P2: Plan Source or P3: Plan Make category determines how the role computes order size. You configure different planning parameters for product composites, depending on the Plan Type, which is either Replenishment or Customer Forecast.

If a role uses a stock/replenishment planning strategy, configure these planning parameters for source and delivery products:

Replenishment Threshold
Minimum Reorder Quantity
Maximum Threshold

If a role uses a stock/forecast planning strategy:

Configure the Estimated Contract Amount parameter for the source products of the downstream buyer role.
Configure these parameters for the buyer role's Source category:
- Initial Contract Delay
- Contract Duration
- Contract Response Time

To configure planning parameters for product composites:

1 Show the Products pool detail for each Base Manufacturer, Distributor, and Manufacturer role in the model.
2 Display the properties dialog for each source and delivery product, then configure the parameters listed above, depending on the planning strategy.
