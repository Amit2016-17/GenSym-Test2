Topic - What Happens When the Simulation Runs

Build Process

The Make category on the detail of a Base Manufacturer or Manufacturer role receives build orders from the Make Planning process and manufactures finished products.

The Build process has four configurations, depending on the role and depending on the planning strategy the role uses:

Base Manufacturer role: Stock and make-to-order
- Does not use source products; therefore, does not need components.
- Generates and accumulates payments as part of the Build process.
- Applies engineering time and costs.
- Requests extra finished products by creating additional build orders to compensate for build yield, as necessary.
Base Manufacturer role: engineer-to-order
- Does not use source products; therefore, does not need components.
- Generates and accumulates payments as part of the Build process.
- Builds extra finished products for each build order to compensate for build yield for each build order.
Manufacturer role: Stock and make-to-order products
- Uses source products; therefore, needs components.
- Does not generate and accumulate payments; the Financial process performs this task, as described in Financial Process.
- Requests extra finished products by creating additional build orders to compensate for build yield, as necessary.
Manufacturer role: Engineer-to-order products
- Uses source products; therefore, needs components.
- Does not generate and accumulate payments; the Financial process performs this task, as described in Financial Process.
- Applies engineering time and costs.
- Builds extra finished products for each build order to compensate for build yield.
