Triggering Regular Updates for Multiple Reports

You might want to trigger multiple reports to update at the same time. The easiest way to do this is to associate each report with an Update Trigger tool, which triggers updates for all associated reports.

Another advantage of using an Update Trigger tool is that you can configure a time delay before the first update.

To trigger regular updates for multiple reports:

1 Display the e-SCOR toolbar and click the Tools tab:

2 Select an Update Trigger tool and place it on the workspace that contains the report object.
3 Display the properties dialog for the Update Trigger and on the General tab, configure the Label.
For example, you might label the Update Trigger "Hourly Update."
By default, the Update Trigger tool triggers updates continuously.
4 Click the Block tab and configure the Maximum Starts to be the maximum number of times the trigger should update, as needed.
You might want to begin triggering updates after a time delay or stop triggering updates after a certain simulation time.
5 Configure the Start Time and End Time to be the time at which the trigger tool should start and finish triggering updates, as needed.
For a role report, you typically use the Financial Period parameter as the value of the Mean parameter so the report computes financial metrics once per financial period.
Tip: To ensure that the report computes metrics for the current Financial Period, click the Block tab and configure the Start Time of the Update Trigger to be 2 seconds, which causes the report to trigger two seconds after the end of the Financial Period.
6 Click the Duration tab and configure the Period to be the frequency with which to trigger updates.
For example, to update the reports once an hour of simulation time, enter 1 hour.
7 Choose the Choose Update Trigger menu choice on a report object, then choose Select on the Update Trigger.
A magenta arrow appears indicating that you have selected the trigger.
8 Repeat Step 7. for each report object whose data you want to update, based on the Update Trigger.

This figure shows how you use a single Update Trigger tool to update two reports. The Role Output Report and Level 2 Output Report are both associated with the Financial Period Update Trigger, which is configured to update its associated reports once per financial period, which is 30 days, by default. By configuring the Start Time of the trigger to be 2 seconds, the reports update two seconds after each update period, which means the report shows the most current values.
