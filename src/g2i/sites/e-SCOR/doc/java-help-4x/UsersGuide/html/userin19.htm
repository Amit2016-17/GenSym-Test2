Editing Module Properties

You can edit various information about the module, including the name of the file that e-SCOR uses when it saves the module, the name of the module in the hierarchy, and the directly required modules.

For example, when creating a new module, you typically edit the Directly Required Modules to require the customiz module so it has access to existing definitions. Similarly, when merging a module, you edit the Directly Required Modules of the top-level module to include the merged module.
