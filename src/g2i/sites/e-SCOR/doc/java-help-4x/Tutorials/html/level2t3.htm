Configuring Source Category Parameters: Consumer Role

The Source category of a role is responsible for placing orders with upstream suppliers for source products, receiving shipments from suppliers, and making payments to suppliers. You configure timing parameters related to receiving source products and placing them in inventory, and cost parameters related to creating orders, receiving shipments, and making payments.

Normally, when a role receives source products, it places them in inventory. However, a Consumer role does not keep an inventory; therefore, you do not configure the Receive and Putaway Time parameter for a Consumer role, which defines the time it takes to receive source products and place them in inventory.

Distributor and Manufacturer roles use source planning to determine the size of each replenishment order for source products and when to place an order, based on orders for its delivery products. However, because a Consumer role has no downstream buyers, it places orders for its source products, based on parameters that you configure in the Sc: Source Consumer category. You configure the Start Time and Stop Time, and the Order Period and Order Size for generating orders.

You configure the Order Period and Order Size parameters by using any one of a number of mathematical distributions. By default, you configure the Min, Mode, and Max of a random triangular distribution. If you have initial order demand data in tabular format, you can also configure these and other parameters through a Demand Input Report.

If the buyer role sources the same product from multiple upstream suppliers, you can also configure the Source category of the buyer role to choose suppliers, based on contracts. You configure the award criteria for choosing suppliers, the number of suppliers to choose, and timing parameters related to the length of the contract. In the simplest e-SCOR model, the Consumer role has a single upstream supplier, so contracts are not relevant.

Configure the Sc category so that it:

Generates orders once a week, using a random exponential distribution.
Generates orders for 10 product families, plus or minus 5 units, using the default start and stop times.
Costs $10 to receive the product family, create an order, and make a payment.
