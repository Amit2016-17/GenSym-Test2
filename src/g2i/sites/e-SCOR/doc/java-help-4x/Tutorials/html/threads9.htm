Summary: What Happens When the Simulation Runs

When the simulation runs, e-SCOR performs numerous processes, which interact to create and store orders, product shipments, invoices, payments, and other objects, and to determine parameters and compute metrics. These processes are:

Role Verification process
Order/Product process
Build process
Financial process
Make Planning process
Source Planning processes
Contracts process

You should now have a basic understanding of the basic processes e-SCOR runs when you run a simulation.
