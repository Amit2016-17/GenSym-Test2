Configuring the Model Environment

To configure the model environment, you:

Create a top-level workspace, which is a blank area on which you create one or more e-SCOR models.
Create a model, which is a container in which you build an e-SCOR model.
Create a scenario, which is the discrete event simulation engine for controlling the simulation.
Save the model to a new .kb file name.
Start e-SCOR with your model open by configuring the server to load your model.

Many e-SCOR objects have detail, which is a workspace associated with an object on which you place and view other objects. You create e-SCOR models on the detail of a Model object, also known as the model detail.

You place Model objects on workspaces, and you place a single Scenario object on the model detail.

Shortcut: If you do not have time to work through the exercises in this part of the tutorial, in e-SCOR, open the file named Tutorial-Simplest-Model.kb, which shows the finished model. Follow along in the existing model as you read through the exercises.
