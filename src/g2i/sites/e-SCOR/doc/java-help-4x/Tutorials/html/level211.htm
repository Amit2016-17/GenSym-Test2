Configuring Source Product Parameters: Consumer

Normally, you configure inventory and planning parameters for the source products of a Distributor or Manufacturer role, just as you did for the delivery product of the Base Manufacturer role. You also configure parameters related to desired turnaround time and order fulfillment preference.

However, because a Consumer role has no downstream buyer roles, it does not keep an inventory of its source products. Furthermore, a Consumer role creates orders for its source products on a cyclical basis, based on parameters that you configure in the Sc: Source Consumer category, rather than based on source planning. Therefore, you do not need to configure source product parameters for a Consumer role that relate to inventory or planning. Specifically, a Consumer role ignores these parameters:

Order Type
Starting Inventory
Inventory Maximum
Replenishment Threshold
Minimum Reorder Quantity
Maximum Threshold

By default, if a buyer role has multiple upstream supplier roles for the same source product, it chooses the supplier at random. You can configure the buyer role to use contracts when placing orders for source products by configuring parameters on the Contract tab of the Source category. In addition, you configure the Estimated Contract Amount parameter on the Source tab of the source product.

In the simplest e-SCOR model, the Consumer role sources products from a single upstream supplier; therefore, you do not need to configure contract parameters for source products.

Configure the source product parameters on the Source tab for the Consumer role so that it:

Requests that suppliers deliver orders for source products within 1 week of receiving them.
Fulfills only complete orders; does not allow partial shipments.
