Creating a Supply-Chain Model: Simplest Model

To create a supply-chain model, you create roles from the toolbar, then connect the roles. Each role has pairs of input and output stubs, which you connect between roles to form paths.

A Base Manufacturer role has input-output stub pairs on the downstream side of the role; a Consumer role has stubs on the upstream side of the role; and a Manufacturer and Distributor role have stubs on both the upstream and the downstream sides of the roles. A Base Manufacturer must be the first role in the supply chain, and a Consumer must be the last role in the supply chain.

When you connect the roles and run the simulation for the simplest model, the following objects flow around the model:

Orders flow upstream from the Consumer role to the Base Manufacturer role.
Product shipments and invoices flow downstream from the Base Manufacturer role to the Consumer role.
Payments flow upstream from the Consumer role to the Base Manufacturer role.

The model is automatically configured to process these and other required objects, such as purchase requests, purchase responses, and purchase awards.

To create the simplest supply-chain model:

1 Select a Base Manufacturer role from the Roles tab of the e-SCOR toolbar and place it on the left side of the model detail.
A Base Manufacturer role has a blue output stub and a magenta input stub on the downstream side of the role.
2 Select a Consumer role from the toolbar and place it to the right of the Base Manufacturer role.
A Consumer role has a blue input stub and a magenta output stub on the upstream side of the role.
Now, you connect the input and output stubs of the same color to create input and output paths between the roles.
3 Click the blue output stub of the Base Manufacturer role and drag it until it overlaps the blue input stub of the Consumer role, then click to connect the stubs to form a path.
An arrow appears on the path indicating that objects flow downstream.
4 Connect the magenta output stub of the Base Manufacturer role to the magenta input stub of the Consumer role.
The arrow on the path indicates that objects flow upstream.
5 Move one of the roles slightly on the workspace to adjust the alignment of the input and output paths between the roles.
Tip: If the stubs are actually connected, then moving the role adjusts the alignment of the paths to create straight lines. If the alignment does not adjust, then the stubs are not properly connected. Be sure the stubs are connected before continuing.
6 Shrink wrap the workspace and adjust the window size to fit the model.
