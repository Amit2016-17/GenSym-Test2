Running the Simulation

When you run a simulation, the status of the Scenario changes depending on whether the model is running, paused, resetting, or stopped.

By default, e-SCOR runs the simulation in jump mode, which processes each discrete event continuously and automatically advances the simulation clock with each new event.

To run the simulation:

1 Choose Start All from the Simulation menu, press the F6 key, or click the equivalent toolbar button:
The status changes to resetting, then running.
Note: Clicking the Start All button activates, resets, and starts the simulation running. Step 5 describes what happens when you reset the simulation.
e-SCOR verifies that you have configured the model correctly. If so, you will begin to see objects flowing through the model. These objects represent purchase requests, purchase responses, purchase awards, orders, product shipments, invoices, and payments, which the model creates and deletes when it runs. The model advances the simulation clock with each new event and computes metrics for all objects as it runs.
If you do not see these objects flowing through the model, you have not configured the model correctly. See Troubleshooting Problems Running Simulations.
2 Choose Pause from the Simulation menu, press the F8 key, or click the equivalent toolbar button to pause the simulation:
The status changes to paused.
Pausing the simulation stops the simulation clock, causes all objects to stop flowing through the model, and stops computing metrics.
3 Choose Continue from the Simulation menu, press the F9 key, or click the equivalent toolbar button to resume running the simulation after pausing: 
The status changes to running.
The simulation clock begins advancing, objects begin flowing through the model, and the model continues to computes metrics.
4 Choose Reset from the Simulation menu, press the F7 key, or click the equivalent toolbar button to reset the simulation:

The status changes from resetting to reset.

Resetting the simulation resets the simulation clock to the start time, deletes all objects the model created, and resets all metrics to their initial values.
