Configuring Plan Category Parameters: Base Manufacturer Role

The Plan category is responsible for computing the size of an order, the frequency with which to create orders, when to place the first order, and to whom to ship each order. Base Manufacturer and Manufacturer roles use make planning to determine the size of each build order, and Distributor and Manufacturer roles use source planning to determine the size of each replenishment order.

You configure timing parameters in the Plan category for each role, as appropriate. For stock products, you can configure the planning strategy the role uses, which can be either Replenishment or Customer Forecast. You can also configure the role to use replenishment planning, also known as demand-based planning, in addition to cyclical planning. When replenishment planning is enabled, the role creates a build order whenever the downstream buyer places a customer order for the role's delivery products. By default, the Plan category performs cyclical planning only.

Note: A role must wait a minimum of 2 seconds before initiating its first plan.

You configure planning parameters related to replenishment thresholds and order quantities in the source and delivery products for the role, which you will do later.

Configure the P3: Plan Make category so that it:

Uses replenishment planning to determine how the role computes build-order size when manufacturing stock delivery products, the default.
Initiates a plan once every 2 weeks.
Waits 2 days before initiating its first plan, the default.
Perform cyclical planning only; does not perform demand-based planning, the default.
Compensates for lower yield rates when determining build order size.
