Configuring Basic Role Parameters: Simplest Model

Each role defines a set of basic parameters, which you can configure. You typically configure the Role Label to indicate the type of product a role delivers.

You can also configure the Source Type and Delivery Type parameters for a role, which determine whether the role uses a pull or push planning mode.

By default, each role uses a pull planning mode, which means that downstream buyer roles place orders with suppliers for their source products, based on source planning. Source planning determines the number of source products a buyer role orders and the frequency of those orders.

You can also configure a role to use a push planning mode, in which case the upstream supplier role determines how many products it delivers, based on the amount you specify in the supplier role.

The combination of the Source Type and Delivery Type parameters for a role determine the planning mode the role uses. You can combine pull and push planning modes between roles and within the same role to use different types of planning at each stage in the supply chain. For example, a Manufacturer role might source components, using a pull planning strategy, but it might deliver finished products, using a push planning strategy.

A Base Manufacturer role does not have a Source Type, because it does not source any input products, and a Consumer role does not have a Delivery Type, because it does not deliver any output products.

The simplest e-SCOR model uses a pull planning mode; therefore, you do not need to configure the Source Type or Delivery Type parameters.
