Troubleshooting Problems Running Simulations

The most common mistakes you can make when configuring a simple e-SCOR model are:

You forget to assign product specifications to roles.
You fail to connect roles together correctly, using input and output stubs.
You forget to assign resources to manufacturing roles.
You do not label each role with a unique name, when multiple roles of the same type exist.

If you do not configure the model correctly, e-SCOR immediately pauses the simulation and displays the detail of the category or role that is configured incorrectly. You will see a detail with a magenta arrow pointing to the block with the error. You can drag the arrow away from the block to see the associated error message, which usually gives some indication of the error. Typically, these errors occur at Level 4, Level 5, or higher. You can also choose Message Board or Logbook from the View menu to view these errors.

If you forget to assign resources to manufacturing roles, the model places orders, but it never manufactures any finished products.

To correct configuration errors when running a simulation:

Reset the model, then configure the model correctly and run the simulation again.
