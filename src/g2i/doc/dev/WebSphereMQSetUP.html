Introduction

This document explains the steps required to set up IBM Websphere MQ 5.3, so that JMS bridge for Websphere can be connected to it and messages from G2 can be sent to the configured queue/topic successfully, using the bridge. These setup instructions are specific to IBM Websphere MQ 5.3, and users may need to modify their setup with other versions of the software.

 

Installation

  1. 1.Install IBM WebSphere MQ version 5.3, do a custom install and select the JMS service installation option while doing the install. Be sure to set up a default configuration, the installation will prompt you to do so at the end. 

 

  1. 2.Install SupportPac MA0C: MQSeries Publish/Subscribe ,to support publish/subscribe mechanism. It can be found at the URL : 

         http://www-1.ibm.com/support/docview.wss?rs=203&uid=swg24000643&loc=en_US&cs=utf-8&lang=en

 

3.   Install IDK1.3.1 or later if its not already installed on the system

 

Set up System Environment

  1. 1.Set JAVA_HOME = c:\jdk1.3.1 

  2. 2.Set MQ_JAVA_DATA_PATH= C:\Program Files\IBM\WebSphere MQ 

  3. 3.Set MQ_JAVA_INSTALL_PATH= C:\Program Files\IBM\WebSphere MQ\Java 

  4. 4.Set IBM_DIR=%MQ_JAVA_INSTALL_PATH% 

  5. 5.set IBM_BIN_DIR=C:\Program Files\IBM\WebSphere MQ\bin 

  6. 6.IBM_LIB=%IBM_DIR%\lib\com.ibm.mq.jar;%IBM_DIR%\lib\com.ibm.mqjms.jar;%IBM_DIR%\lib\com.ibm.mqbind.jar;%IBM_DIR%\lib\jms.jar;%IBM_DIR%\lib\jndi.jar;%IBM_DIR%\lib\fscontext.jar;%IBM_DIR%\lib\connector.jar;%IBM_DIR%\lib\jta.jar;%IBM_DIR%\lib\ldap.jar;%IBM_DIR%\lib\providerutil.jar;%IBM_DIR%\lib\mqjbnd05.dll;%IBM_DIR%\lib\mqjbdf02.dll;%IBM_DIR%\lib\MQXAi02.dll 

  7. 7.set PATH=%IBM_BIN_DIR%;%IBM_DIR%\lib;%IBM_DIR%\bin;%JAVA_HOME%\bin 

  8. 8.set CLASSPATH=%IBM_LIB%

Setup JMSAdmin

First, set JMSAdmin to use WebSphere as the name service. Edit JMSAdmin.config in

C:\Program Files\IBM\WebSphere MQ\Java\bin so that it includes these lines:

 

INITIAL_CONTEXT_FACTORY=com.sun.jndi.fscontext.RefFSContextFactory

PROVIDER_URL= file:/C:/tmp

Set up required for Publish/Subscribe Model

To be able to use the publish/subscribe model of Websphere MQ, following steps need to be done:

  1. 1.Go to directory C:\Program Files\IBM\WebSphere MQ\Java\bin and on the command prompt type : 

      > runmqsc QM<mqjms_psq.mqsc                                                                                        

      Here QM is your default QueueManager. This creates WebSphere MQ objects  

      needed for pub/sub processing.

  1. 2.Type strmqbrk -m QM and press Enter. This starts the WebSphere MQ pub/sub broker.

Addendum 1. Create TCF Objects for Use in a Network Environment

 

From the IBM manual:

"The TopicConnectionFactory in this case is an administered object that has been created and administered using the JMSAdmin tool. Use this method of obtaining TopicConnectionFactory objects because it ensures code portability. The TopicConnectionFactory in the example is testTCF in JMSAdmin. Create testTCF in JMSAdmin before running the application.

"To create a TopicConnectionFactory object, invoke the JMSAdmin tool and execute one of the following commands, depending on the type of connection you want to make to the broker:

Bindings connection: (NOTE. THIS DID NOT WORK IN ONE CUSTOMER’S CASE)
InitCtx> def tcf(testTCF) transport(bind)
or, because this is the default transport type for TopicConnectionFactory objects:
InitCtx> def tcf(testTCF)
This creates a TopicConnectionFactory with default settings for bindings transport, connecting to the default queue manager.

Client connection: (NOTE. THIS WORKED)
InitCtx> def tcf(testTCF) transport(client)
This creates a TopicConnectionFactory with default settings for the client transport type, connecting to localhost, on port 1414, using channel SYSTEM.DEF.SVRCONN.

Direct TCP/IP connection to WebSphere MQ Event Broker: (NOTE.  DID NOT TRY THIS OPTION)
InitCtx> def tcf(testTCF) transport(direct)
This creates a TopicConnectionFactory to make direct connections to a WebSphere MQ Event Broker, connecting to localhost on port 1506."
