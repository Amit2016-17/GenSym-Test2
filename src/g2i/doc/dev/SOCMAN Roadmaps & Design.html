Overview

Introduction

This adaptor enables G2 applications to connect to many systems such as legacy systems via a TCP/IP connection. .

Purpose

The purpose of this document is to list the requirements for the plug-in module including a phased in delivery approach and describe key design issues and implementation issues.  The intended audience for this document includes the following:

  • .potential users of the product, such as consultants, solution engineers, and end users; 

  • .managers involved in the marketing or management of the product;

General Requirements

Software Environment

The SOCMAN adaptor needs to run on Windows and Unix platforms. The SOCMAN adaptor is written in Java and relies on Javalink to communicate with G2.

Scalability and Performance

None

Security

None

Compatibility / Migration

None

International

The SOCMAN module needs to support internationalization for all user, error messages and text strings send or received from remote systems.

Licensing

No specific requirement. It needs to be licensed similar to all G2i modules.

Bundle Inclusion

This module will be part of the G2 bundle and therefore be available included in all applications as well.

High Level Design & Implementation

Introduction

Since this module will initially be branded from functionality available in Integrity, its design and APIs are derived from this original concept.
