Properties

 G2ItemHolder has the following properties, which you can set in a container application. Properties configure the behavior of G2ItemHolder and indicate its status. Most container applications save the state of the properties in your document and restore them when you load the document. The properties can be modified either by using the Property window or programmatically.

Design Time Properties

You can set the CallTimeout, ObjectAddress, ObjectAddressClass, and SubscribedAttributes properties at design time in the properties window or programmatically.

Property Description
AutoDownload As Boolean
If true, you automatically receive an event when the item in G2ItemHolder is updated in G2. If false, you do not automatically receive events when the item is updated in G2. If you do not receive events, your application takes up less CPU time.
AutoUpload As Boolean
If true, updates you make in a Visual Basic application are automatically uploaded to G2. If false, your updates to the item in G2ItemHolder are not automatically uploaded to G2.
CallTimeout As Long
Sets the maximum amount of time for the Telewindows2 Toolkit to wait for G2 to respond. The default is 30 seconds.
ObjectAddress As String
A reference to a particular item in named connection to G2 in the format: G2Item@GatewayId The named connection to G2 is the GatewayId property of the G2UiGateway object. If ObjectAddress is blank, there is no downloading of an object at startup.
ObjectClass As String
The class of object referenced by ObjectAddress. You can use this property to restrict the class type for the item specified by the ObjectAddress property. The class may be a superclass, such as ITEM or OBJECT. Default is ITEM.
SubscribedAttributes As String
For the item specified by ObjectAddress, SubscribedAttributes is a comma-delineated list of attributes to subscribe to at startup.

You can use the ObjectAddress, ObjectClass, and SubscribedAttributes properties to reference an item in G2 at startup. Doing so enables you to open your Visual Basic application already pointing to the specified item in G2. You immediately have access to its attributes.

If you want to assign or change a G2 item reference once your Visual Basic application is running, use the Item property, described in Runtime Properties.

Runtime Properties

You can use the Attribute, Element, and Item properties at runtime, not design time. Once a connection to G2 is established, you can programmatically access the values of these properties of the connected object

Property Description
Attribute(AttributeName As String)
Access the value of the named attribute
AttributeNames As Variant
A read-only list of attribute names in the G2 item.
Element(Index As Long)
Access an element in a list or array.
ElementCount As Long
Specifies the length of an array. It is read only.
Elements As Variant
Specifies all the elements of an array.
ElementType As Long
Specifies the enumerated G2 type in a list or array. It is read only. The type is one of the following:
  • Const g2Integer = 1
  • Const g2Symbol = 3
  • Const g2Text = 4
  • Const g2TruthValue = 5
  • Const g2Float = 6
  • Const g2Item = 7
  • Const g2Value = 8
  • Const g2ItemOrValue = 9
  • Const g2Quantity = 10

Item As G2Item
Assigns a reference to an object in G2. This property enables G2ItemHolder to persistently point to an object on the G2 server.
