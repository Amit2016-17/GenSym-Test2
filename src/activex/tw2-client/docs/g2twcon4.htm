Events

 G2UiGateway can raise the events described in this section. In your Visual Basic application, you can specify logic to respond to these events. You do not need to write handlers for events that are not needed by your container application. The event handler for a specific event can freely call other methods of Telewindows2 Toolkit for ActiveX and procedures in G2.

G2Connected

The G2Connected event signals that Telewindows2 Toolkit established a connection with G2.

Syntax

    G2Connected()

G2Disconnected

 The G2Disconnected event signals that Telewindows2 Toolkit is no longer connected to G2.

Syntax

    G2Disconnected()

G2StateChange

 The G2StateChange event informs you that the G2 state has changed and what the new state is.

Syntax

    G2StateChange(NewState As G2States)
    Argument Description
    NewState As G2States
    		Returns the state of the connected G2, which is one of the following constants:
    • Const g2Reset = 0
    • Const g2Paused = 1
    • Const g2Running =2
    • Const g2Resumed =3

GotMessage

 The GotMessage event informs you that a message has arrived from G2 and contains the text of the message.

Syntax

    GotMessage(Message As String)
    Argument Description
    Message As String
    		The text of the message.

UserLoggedIn

 The UserLoggedIn event informs you that a user has logged in.

Syntax

    UserLoggedIn()

UserLoggedOut

 The UserLoggedOut event informs you that a user has logged out.

Syntax

    UserLoggedOut()

UserModeChanged

 The UserModeChanged event informs you that the user mode has changed and what the new user mode is.

Syntax

    UserModeChanged(NewUserMode As String)
    Argument Description
    NewUserMode As String
    		Returns the new user mode of G2.
